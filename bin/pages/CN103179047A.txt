    <div class="abstract">本发明提供一种业务的资源调度方法、装置及系统，属于核心网技术领域。其中，该业务的资源调度方法包括：策略服务器接收到深度包检测设备上报的业务数据流的业务信息；所述策略服务器根据所述业务信息制定相应的服务质量QoS控制策略；所述策略服务器将所述QoS控制策略通过SGSN传递给GREAN网络，以便GREAN网络根据所述QoS控制策略对所述业务数据流进行无线资源调度。本发明的技术方案能够在不影响用户的网络感知的情况下，通过高精度的业务识别对数据业务实施差异化的无线资源调度。</div>

    <p>业务的资源调度方法、装置及系统</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及核心网技术领域，特别是指一种业务的资源调度方法、装置及系统。</p>
    <p>背景技术</p>
    <p>[0002]	数据业务作为一种新兴的业务已经渗透到人们的工作和生活的各个领域，成为手机的一项重要业务，且最近几年呈现“井喷”式的高速增长。数据业务的高速增长，特别是以M(Instant Messenger，即时消息)为代表的小包突发型业务，由于现网GERAN(GSM EDGERadio Access Network，增强型数据速率全球移动通讯系统无线接入网)无法区分业务等级，对所有等级的业务采用了资源公平的调度机制，使得这些小包突发型业务与大带宽需求业务占用了同等的信道资源，从而导致整体信道数据传送效率低下，间接的挤压其他高带宽需求业务的信道需求，对2G网络资源和2G网络质量感知带来了巨大的冲击和压力。</p>
    <p>[0003]	为此，业界相继提出各种差异化数据业务资源管控的方法。目前业界主要有基于3GPP PCC架构的资源管控和非标准3GPP架构硬件DPI的流量管控两种解决方案。</p>
    <p>[0004]	其中，基于3GPP PCC架构的资源管控解决方案中，PCC架构中主要有两大功能实体，其中PCRF(Policy and Charging Rule Function,策略和计费控制单元)功能实体包含策略控制决策和基于流计费控制的功能，向PCEF(Policy and Charging EnforcementFunction,策略和计费执行单元)提供关于业务数据流检测、门控、基于QoS (Quality ofService，服务质量)和基于流计费(除信用控制外)的网络控制功能。PCEF功能实体位于GGSN(Gateway GPRS Support Node,网关通用分组无线服务网络支持节点)网关,主要包含业务数据流的检测、策略执行和基于流的计费功能。基于3GPP PCC架构的资源管控信令流程如图1所不:</p>
    <p>[0005]	1、UE (用户)完成F1DP (Packet Data Protocol,分组数据协议)激活过程,接入网络同时开始业务数据的交互；</p>
    <p>[0006]	2、内置于GGSN(PCEF)中的软件DPI (Deep Packet Inspection,深度包检测)功能实体对用户业务数据流进行识别检测，提取该业务数据流的特征信息，并对这些信息进行分析，获知该业务数据流对应的业务信息，如业务类型等；</p>
    <p>[0007]	3、软件DPI功能实体通过PECF向PCRF上报该业务数据流的业务信息；</p>
    <p>[0008]	4、PCRF依据后台策略数据库制定该业务的QoS控制策略，如，对于&#38914;类业务生成低调度优先级QoS的PCC策略；</p>
    <p>[0009]	5、PCRF将业务的QoS控制策略信息下发给PCEF ；</p>
    <p>[0010]	6、PCEF(GGSN)通过 update PDP context Request (升级 PDP 上下文请求)消息将QoS控制策略信息传递给SGSN (Serving GPRS Support Node，服务通用分组无线服务网络支持节点)；</p>
    <p>[0011]	7、SGSN通过PFC流程将业务流QoS信息传递给GREAN ；</p>
    <p>[0012]	8、同时，SGSN与手机终端完成PDP上下文的QoS信息更新；</p>
    <p>[0013]	9、SGSN 向 GGSN 返回 Update PDP Context Response (升级 PDP 上下文响应)消息确定QoS更新完成；</p>
    <p>[0014]	10、GREAN根据接收到的QoS信息对用户业务数据流进行差异化的无线信道资源调度，从而实现对低价值业务的资源管控。</p>
    <p>[0015]	非标准3GPP架构硬件DPI流量管控解决方案中，主要由外置硬件DPI和策略服务器2个功能模块组成。其中，外置硬件DPI组要负责对业务数据包的深度报文识别和对策略服务器下发的策略进行执行；策略服务器主要是负责策略的管理和下发；其资源管控流程如图2所:</p>
    <p>[0016]	1、首先由后台操作人员在策略服务器上定制策略，策略服务器将该策略下发至外置硬件DPI模块，如对IM业务流实现20%劣化控制，即丢弃20%的数据报文；</p>
    <p>[0017]	2、用户完成激活并开始业务数据流交互；</p>
    <p>[0018]	3、外置硬件DPI对业务数据流进行逐报文的检测和识别；</p>
    <p>[0019]	4、外置硬件DPI对检测和识别到并符合控制规则的报文按照规则进行丢弃，从而一定程度上实现对该业务无线资源占用缓解。</p>
    <p>[0020]	现有技术中，基于3GPP PCC架构的资源管控解决方案识别数据业务的准确性比较低，无法对数据业务的特征进行精确识别；非标准3GPP架构硬件DPI的流量管控解决方案只能执行简单的数据报文丢弃类的流量控制策略，虽然对特定业务报文的丢弃在一定程度上能减少无线资源的占用，但其却不能从根本上优化资源的调度，且硬丢包的行为会重挫用户的网络感知。</p>
    <p>发明内容</p>
    <p>[0021]	本发明要解决的技术问题是提供一种业务的资源调度方法、装置及系统，能够在不影响用户的网络感知的情况下，通过高精度的业务识别对数据业务实施差异化的无线资源调度。</p>
    <p>[0022]	为解决上述技术问题，本发明的实施例提供技术方案如下:</p>
    <p>[0023]	一方面，提供一种业务的资源调度方法，包括:</p>
    <p>[0024]	策略服务器接收到深度包检测设备上报的业务数据流的业务信息；</p>
    <p>[0025]	所述策略服务器根据所述业务信息制定相应的服务质量QoS控制策略；</p>
    <p>[0026]	所述策略服务器将所述QoS控制策略通过SGSN传递给GREAN网络，以便GREAN网络根据所述QoS控制策略对所述业务数据流进行无线资源调度。</p>
    <p>[0027]	进一步地，所述策略服务器接收到深度包检测设备上报的业务数据流的业务信息之前还包括:</p>
    <p>[0028]	所述策略服务器下发用以对业务数据流进行识别的业务识别规则至所述深度包检测设备。</p>
    <p>[0029]	进一步地，所述策略服务器将所述QoS控制策略通过SGSN传递给GREAN网络包括:</p>
    <p>[0030]	所述策略服务器通过伪造的GGSN的请求报文将所述控制策略发送给SGSN，以便SGSN将所述控制策略发送GREAN网络。</p>
    <p>[0031]	进一步地，所述策略服务器将所述QoS控制策略通过SGSN传递给GREAN网络之后还包括:[0032]	所述深度包检测设备检测到SGSN发送给GGSN的对应所述请求报文的响应报文；</p>
    <p>[0033]	所述深度包检测设备丢弃所述响应报文，以避免GGSN接收到所述响应报文。</p>
    <p>[0034]	本发明实施例还提供了一种业务的资源调度装置，包括:</p>
    <p>[0035]	接收模块，用于接收到深度包检测设备上报的业务数据流的业务信息；</p>
    <p>[0036]	处理模块，用于根据所述业务信息制定相应的服务质量QoS控制策略；</p>
    <p>[0037]	发送模块，用于将所述QoS控制策略通过SGSN传递给GREAN网络，以便GREAN网络根据所述QoS控制策略对所述业务数据流进行无线资源调度。</p>
    <p>[0038]	进一步地，所述发送模块还用于下发用以对业务数据流进行识别的业务识别规则至所述深度包检测设备。</p>
    <p>[0039]	进一步地，所述发送模块具体用于通过伪造的GGSN的请求报文将所述控制策略发送给SGSN，以便SGSN将所述控制策略发送GREAN网络。</p>
    <p>[0040]	本发明实施例还提供了一种业务的资源调度系统，包括:</p>
    <p>[0041]	深度包检测设备，用于对业务数据流进行检测，获取业务数据流的业务信息，并将所述业务信息发送给策略服务器；</p>
    <p>[0042]	所述策略服务器，用于根据所述业务信息制定相应的服务质量QoS控制策略，将所述QoS控制策略发送给SGSN ；</p>
    <p>[0043]	所述SGSN，用于将所述QoS控制策略发送给GREAN网络，以便GREAN网络根据所述QoS控制策略对所述业务数据流进行无线资源调度。</p>
    <p>[0044]	进一步地，所述策略服务器还用于下发用以对业务数据流进行识别的业务识别规则至所述深度包检测设备；</p>
    <p>[0045]	所述深度包检测设备具体用于根据所述业务识别规则对业务数据流进行检测，获取业务数据流的业务信息。</p>
    <p>[0046]	进一步地，所述策略服务器具体用于通过伪造的GGSN的请求报文将所述控制策略发送给SGSN ；</p>
    <p>[0047]	所述SGSN具体用于在接收到所述请求报文后，将所述QoS控制策略发送给GREAN网络。</p>
    <p>[0048]	进一步地，SGSN还用于向GGSN返回对应所述请求报文的响应报文；</p>
    <p>[0049]	所述深度包检测设备还用于检测并丢弃所述响应报文，以避免GGSN接收到所述响应报文。</p>
    <p>[0050]	本发明的实施例具有以下有益效果:</p>
    <p>[0051]	上述方案中，策略服务器根据DPI设备上报的业务信息制定QoS控制策略，并伪造GGSN的请求报文将QoS控制策略传递给SGSN，以便SGSN将QoS控制策略发送给GREAN网络，之后GREAN网络可以根据该QoS控制策略对业务数据流进行无线资源调度。本发明提高了业务识别的精度，同时弥补了现有技术中无法将QoS控制策略传递给GERAN网络的不足，使得GERAN网络能依据QoS控制策略进行差异化的无线资源调度。</p>

    <p>[0075]	SGSN53，用于将QoS控制策略发送给GREAN网络55，以便GREAN网络55根据QoS控制策略对业务数据流进行无线资源调度。</p>
    <p>[0076]	其中，外置的DPI设备51采用硬件的设计架构，具备指纹级别精确的业务识别功能，外置的DPI设备21与策略服务器52、SGSN53及GGSN54连接:外置的DPI设备51接收策略服务器52下发的业务识别规则，并根据该业务识别规则对业务数据流进行检测，将业务识别检测事件报告上报策略服务器52，同时对SGSN53返回的Update PDP ContextResponse信令实施丢弃功能，对其他业务数据包在SGSN和GGSN间做透传。</p>
    <p>[0077]	策略服务器52管理后台业务控制策略详细信息数据和业务识别事件上报规则，包括对不同业务/用户的QoS策略数据、什么事件需要外置DPI设备上报策略服务器等；策略服务器52与外置DPI设备51及SGSN53连接:策略服务器52负责将业务识别规则下发给外置DPI设备51，同时策略服务器52负责向SGSN53发送伪造的Update PDP ContextRequest报文,用于传递QoS控制策略；</p>
    <p>[0078]	SGSN53与策略服务器52、外置DPI设备51及GERAN55连接，SGSN53负责向外置DPI设备51发送和接收业务数据流，当资源管控起作用时，接收策略服务器52伪造GGSN54的 Update PDP Context Request 报文，并将 QoS 控制策略通过 Packects Flow contextProcedures 下发给 GERAN55。</p>
    <p>[0079]	GERAN55具有根据不同的QoS控制策略对业务数据流做不同优先级的无线资源调度，GERAN55 与 SGSN53 连接，GERAN55 通过Packects Flow context Procedures 接收 SGSN53下发的QoS控制策略，并根据QoS控制策略对不同业务数据流实施不同优先级的无线资源调度，从而实现高效的无线资源管控。</p>
    <p>[0080]	下面结合图6进一步说明本发明的业务的资源调度的具体流程，如图6所示，本发明的业务的资源调度流程包括以下步骤:</p>
    <p>[0081]	步骤SI，用户进行PDP激活接入到网络，并开始业务数据流的交互；</p>
    <p>[0082]	步骤S2，DPI设备对业务数据流实施深度报文检测；</p>
    <p>[0083]	在此之前，外置的硬件DPI设备接收策略服务器下发的业务识别规则，根据该业务识别规则DPI设备可以对业务数据流进行高精度的业务识别；</p>
    <p>[0084]	步骤S3，DPI设备获取业务数据流的业务信息，并将业务信息发送给策略服务器；</p>
    <p>[0085]	外置的硬件DPI设备将符合业务识别规则的业务识别事件信息上报给策略服务器；</p>
    <p>[0086]	步骤S4,策略服务器根据DPI设备上报的业务信息制定相应的QoS控制策略；</p>
    <p>[0087]	比如，策略服务器可以根据DPI设备上报的业务数据流的业务类型制定相应的QoS控制策略，如，对于IM类业务生成低调度优先级的QoS控制策略；</p>
    <p>[0088]	步骤S5,策略服务器通过伪造GGSN的Update PDP Context Request报文，将QoS控制策略传递给SGSN ；</p>
    <p>[0089]步骤	S6, SGSN 通过 Packect Flow Context Procedures 流程将 QoS 控制策略传递至 GERAN ；</p>
    <p>[0090]	步骤S7，SGSN与用户终端完成QoS控制策略的更新；</p>
    <p>[0091]步骤	S8、S9，SGSN 向 GGSN 上报更新响应消息 Update PDP Context Response, DPI设备检测识别到该消息并将消息做丢弃处理，以避免GGSN收到未知的响应消息；</p>
    <p>[0092]	步骤S10，与此同时，GERAN根据接收到的用户业务的QoS控制策略对用户的业务数据流实施差异化的无线资源调度，从而实现通过高精度的业务识别让GREAN对业务实施差异化的无线资源调度，并且不会重挫用户的网络感知。</p>
    <p>[0093]	比如，策略服务器对于各种业务生成不同调度优先级的QoS控制策略，GERAN根据接收到的QoS控制策略，在进行无线资源调度时，根据业务所对应的调度优先级为业务分配无线资源。</p>
    <p>[0094]	本发明弥补了现有技术方案一中内置软件DPI设备只能识别简单业务的不足，提高了业务识别的精度，同时无需对GGSN做PCEF升级改造，使整套系统可以更加灵活独立的应用于不同厂家的GPRS核心网中；本发明弥补了现有技术方案二中无法将QoS控制策略传递给无线接入网GERAN的不足，使得无线接入网GERAN能依据业务的QoS控制策略进行差异化的资源调度。</p>
    <p>[0095]	此说明书中所描述的许多功能部件都被称为模块，以便更加特别地强调其实现方式的独立性。</p>
    <p>[0096]	本发明实施例中，模块可以用软件实现，以便由各种类型的处理器执行。举例来说，一个标识的可执行代码模块可以包括计算机指令的一个或多个物理或者逻辑块，举例来说，其可以被构建为对象、过程或函数。尽管如此，所标识模块的可执行代码无需物理地位于一起，而是可以包括存储在不同位里上的不同的指令，当这些指令逻辑上结合在一起时，其构成模块并且实现该模块的规定目的。</p>
    <p>[0097]	实际上，可执行代码模块可以是单条指令或者是许多条指令，并且甚至可以分布在多个不同的代码段上，分布在不同程序当中，以及跨越多个存储器设备分布。同样地，操作数据可以在模块内被识别，并且可以依照任何适当的形式实现并且被组织在任何适当类型的数据结构内。所述操作数据可以作为单个数据集被收集，或者可以分布在不同位置上(包括在不同存储设备上)，并且至少部分地可以仅作为电子信号存在于系统或网络上。</p>
    <p>[0098]	在模块可以利用软件实现时，考虑到现有硬件工艺的水平，所以可以以软件实现的模块，在不考虑成本的情况下，本领域技术人员都可以搭建对应的硬件电路来实现对应的功能，所述硬件电路包括常规的超大规模集成(VLSI)电路或者门阵列以及诸如逻辑芯片、晶体管之类的现有半导体或者是其它分立的元件。模块还可以用可编程硬件设备，诸如现场可编程门阵列、可编程阵列逻辑、可编程逻辑设备等实现。</p>
    <p>[0099]	在本发明各方法实施例中，所述各步骤的序号并不能用于限定各步骤的先后顺序，对于本领域普通技术人员来讲，在不付出创造性劳动的前提下，对各步骤的先后变化也在本发明的保护范围之内。</p>
    <p>[0100]	以上所述是本发明的优选实施方式，应当指出，对于本技术领域的普通技术人员来说，在不脱离本发明所述原理的前提下，还可以作出若干改进和润饰，这些改进和润饰也应视为本发明的保护范围。</p>
