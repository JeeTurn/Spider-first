    <div class="abstract">本发明提供一种视频传输方法、系统及视频服务器，方法包括：接收移动终端的视频请求消息，所述视频请求消息中包括终端信息和无线场强信息；根据所述视频请求消息，获取与所述终端信息相匹配的第一视频码流；根据所述无线场强信息对所述第一视频码流进行压缩，获取第二视频码流；将所述第二视频码流发送给所述移动终端。本发明技术方案基于无线场强来进一步优化视频码流的大小，降低了移动终端在移动过程中冗余视频流对无线带宽的占用，通过降低所传输的视频码流的大小提高了视频在移动终端播放的流畅度。</div>

    <p>视频传输方法、系统及视频服务器</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及通信技术，尤其涉及一种视频传输方法、系统及视频服务器。 背景技术</p>
    <p>[0002]	随着互联网技术的发展和移动通信终端的普及，移动互联网得到了更大地发展空 间和市场，而无线视频技术更是得到了前所未有的发展机遇。无线视频技术是指移动终端 通过无线网络接收视频码流的技术。目前无线视频传输通常是由视频服务平台根据移动终 端的硬件类型，对视频采集端采集到的视频码流进行相应编码和压缩后通过无线网络传输 到移动终端的。</p>
    <p>[0003]	由于视频传输的数据量很大，即使在有线网络下，视频传输效果也不是很理想。而 由于无线网络带宽能力有限，相对有线网络，无线网络的带宽低且稳定性较差，尤其是在移 动终端移动的过程中，网络状况会不断发生变化，视频的播放还会因为网络状态的变化而 产生延时、抖动等情况，因此，如何在网络带宽有限的条件下尽可能提高视频播放的流畅度 成为无线视频技术面临地极大挑战。</p>
    <p>发明内容</p>
    <p>[0004]	本发明提供一种视频传输方法、系统及视频服务器，用以提高无线视频播放的流 畅度。</p>
    <p>[0005]	本发明提供一种视频传输方法，包括：</p>
    <p>[0006]	接收移动终端的视频请求消息，所述视频请求消息中包括终端信息和无线场强信 息；</p>
    <p>[0007]	根据所述视频请求消息，获取与所述终端信息相匹配的第一视频码流；</p>
    <p>[0008]	根据所述无线场强信息对所述第一视频码流进行压缩，获取第二视频码流；</p>
    <p>[0009]	将所述第二视频码流发送给所述移动终端。</p>
    <p>[0010]	本发明提供一种视频服务器，包括：</p>
    <p>[0011]	接收模块，用于接收移动终端的视频请求消息，所述视频请求消息中包括终端信 息和无线场强信息；</p>
    <p>[0012]	第一获取模块，用于根据所述视频请求消息，获取与所述终端信息相匹配的第一 视频码流；</p>
    <p>[0013]	第二获取模块，用于根据所述无线场强信息对所述第一视频码流进行压缩，获取 第二视频码流；</p>
    <p>[0014]	发送模块，用于将所述第二视频码流发送给所述移动终端。</p>
    <p>[0015]	本发明提供一种视频传输系统，包括本发明提供的任一视频服务器和移动终端；</p>
    <p>[0016]	所述移动终端，用于向所述视频服务器发送视频请求消息，所述视频请求消息中 包括终端信息和无线场强信息，并接收所述视频服务器发送的第二视频码流。</p>
    <p>[0017]	本发明的视频传输方法、系统及视频服务器，采用移动终端上报无线网络的无线场强信息，视频服务器根据无线场强信息对获取的视频码流进行压缩并传输的技术方案， 基于无线场强来进一步优化视频码流的大小，降低了移动终端在移动过程中冗余视频流对 无线带宽的占用，通过降低所传输的视频码流的大小提高了视频在移动终端播放的流畅度。</p>

    <p>具体实施方式</p>
    <p>[0027]	为使本发明实施例的目的、技术方案和优点更加清楚，下面将结合本发明实施例 中的附图，对本发明实施例中的技术方案进行清楚、完整地描述，显然，所描述的实施例是 本发明一部分实施例，而不是全部的实施例。基于本发明中的实施例，本领域普通技术人员 在没有作出创造性劳动前提下所获得的所有其他实施例，都属于本发明保护的范围。</p>
    <p>[0028]	图1为本发明各实施例所基于的视频传输系统的结构示意图。如图1所示，本实 施例的视频传输系统包括：视频采集端11、视频服务器12和移动终端13。</p>
    <p>[0029]	其中，视频采集端11主要包括视频采集设备和视频编码控制单元，用于负责视频 的采集、视频的压缩编码和视频的传输；其中视频采集设备可以为网络摄像头、数字摄像头 等，视频编码控制单元主要用于对视频采集设备采集到的视频信息进行压缩编码，并将经 压缩编码后形成的视频码流提供给视频服务器12。视频服务器12主要负责对移动终端13 进行认证鉴权、控制视频采集端11的接入、以及负责对视频码流的优化和提供流媒体服务 等。移动终端13作为观看端，可以是手机、无线上网本等无线通讯设备。</p>
    <p>[0030]	其中，视频采集端11可以通过有线方式接入通讯网络，例如视频采集端11可以 通过非对称数字用户环路（Asymmetric Digital Subscriber Line ；简称为：ADSL)方式与 视频服务器12连接；视频采集端11也可以通过无线方式接入通讯网络，例如视频采集端 11 可以采用宽带码分多址(Wideband Code Division Multiple Access ；简称为：WCDMA)、 时分同步石马分多址(Time Division-Synchronous Code Division Multiple Access ； 简称为：TD-SCDMA)或码分多址 2000 (Code Division Multiple Access 2000;简称为: CDMA2000)等方式接入通讯网络。移动终端13通过无线方式接入通讯网络。[0031]	其中，在本发明各实施例中视频服务器12的主要功能是控制视频采集端11采集 视频，从视频采集端11获取视频码流，对获取到的视频码流做进一步优化处理以流媒体的 形式通过无线传输方式提供给移动终端13以供移动终端13侧的客户观看。其中，优化处 理是指根据移动终端13提供的无线场强信息对获取到的视频码流做进一步压缩，以根据 无线场强信息降低视频码流的大小，提高视频在移动终端13播放的流畅度。</p>
    <p>[0032]	本发明以下各实施例将基于图1所示的视频传输系统对本发明技术方案做详细 说明。</p>
    <p>[0033]	实施例一</p>
    <p>[0034]	图2为本发明实施例一提供的视频传输方法的流程图。本实施例的执行主体为视 频服务器，如图2所示，本实施例的视频传输方法包括：</p>
    <p>[0035]	步骤201、接收移动终端的视频请求消息，所述视频请求消息中包括终端信息和无 线场强信息；</p>
    <p>[0036]	具体地，移动终端向视频服务器发起视频请求消息，该视频请求消息表明某客户 要观看某个视频，因此，在视频请求消息中携带有可标识客户身份的终端信息和所要观看 的视频信息。通常终端信息包括移动终端的标识、移动终端的屏幕分辨率、移动终端所支持 的编码格式以及其他硬件能力等信息。视频信息主要包括表征视频的视频标识。另外，在 本实施例中视频请求消息还包括无线场强信息，该无线场强信息是指移动终端所在无线网 络的场强大小，用于供视频服务器据此对视频码流做进一步优化处理。</p>
    <p>[0037]	当视频服务器接收到移动终端的视频请求消息时，首先根据视频请求消息中的相 关信息（例如终端信息）对移动终端进行身份认证和鉴权，以判断移动终端是否为合法用 户。当移动终端通过身份认证和鉴权之后，继续执行步骤202。由于对移动终端进行身份认 证和鉴权并不是本发明各实施例的重点，因此，对于此本发明各实施例均不进行详细描述， 且关于身份认证和鉴权的具体过程可以采用现有技术实现。</p>
    <p>[0038]	步骤202、根据视频请求消息，获取与终端信息相匹配的第一视频码流；</p>
    <p>[0039]	具体地，视频服务器对移动终端的身份认证和鉴权结束之后，根据视频请求消息 中的相关信息（例如视频信息等）获取与终端信息相匹配的第一视频码流。其中，第一视 频码流是由视频采集端根据视频服务器的控制指令采集并经压缩编码后发送给视频服务 器的。</p>
    <p>[0040]	步骤203、根据无线场强信息对第一视频码流进行压缩，获取第二视频码流；</p>
    <p>[0041]	在本实施例中，视频服务器获取到第一视频码流之后，根据无线场强信息对第一 视频码流做进一步压缩，以减小视频码流，提高无线网络带宽的利用率。其中，本实施例中 的“压缩”主要是指由视频服务器根据无线场强信息对第一视频码流进行压缩编码，以减少 第一视频码流的数据量，使获取的第二视频码流适合无线网络状况的技术手段。通常，第二 视频码流的数据量小于或等于第一视频码流的数据量。例如：当移动终端的无线场强较大 时，说明无线网络状况较佳，所提供的带宽较大，则视频服务器可以不压缩或轻度压缩第一 视频码流；当无线场强较小时，说明无线网络状况较差，所提供的带宽有限，因此，视频服务 器可以较大幅度地对第一视频码流做进一步压缩，以使获取的第二视频码流的大小与无线 网络的状况相适应。</p>
    <p>[0042]	步骤204、将第二视频码流发送给移动终端。[0043]	视频服务器将根据无线场强再次压缩后的视频码流发送给移动终端，以供移动终 端播放该视频。其中，由于通过进一步压缩视频码流减少了视频码流所包含的冗余信息，因 此，使得无线带宽在单位时间可以传输更多的视频码流，保证了视频码流在移动终端播放 的流程图。</p>
    <p>[0044]	本实施例的视频传输方法，由视频服务器根据无线场强的大小对视频码流做进一 步压缩处理，降低视频码流的冗余信息，提高了无线带宽的利用率，保证了视频码流在移动 终端播放的流畅度。</p>
    <p>[0045]	实施例二</p>
    <p>[0046]	图3为本发明实施例二提供的视频传输方法的流程图。本实施例可基于实施例一 实现，相同之处不再赘述，如图3所示，本实施例的方法包括：</p>
    <p>[0047]	步骤301、接收移动终端的视频请求消息，所述视频请求消息中包括终端信息和无 线场强信息；</p>
    <p>[0048]	步骤302、根据预先存储的第一参数映射表，将终端信息映射为视频采集端的第一 采集参数；</p>
    <p>[0049]	具体地，在视频服务器上预先存储有第一参数映射表，在该第一参数映射表中存 储有移动终端的信息和视频采集端对应的各视频码流信息的对应关系，例如移动终端所支 持的编码格式和/或屏幕分辨率与需要视频采集端获取的视频码流编码格式的映射关系。 视频服务器可以将移动终端所支持的编码格式和屏幕分辨率映射为视频采集端采集视频 的第一采集参数，该第一采集参数主要是指控制视频采集端采集视频时所需的采集参数， 例如：帧频、分辨率、采集后的编码格式、压缩指标等。</p>
    <p>[0050]	步骤303、向视频采集端发送携带第一采集参数的第一采集控制指令，以供视频采 集端采集并根据第一采集参数生成第一视频码流；</p>
    <p>[0051]	视频服务器获取到第一采集参数后即可控制视频采集端进行视频采集，具体通过 向视频采集端发送控制指令控制视频采集端开始视频采集。其中，第一采集控制指令中可 以包括第一采集参数，也可以不包括第一采集参数。当第一采集控制指令中不包括第一采 集参数时，视频服务器可以通过另外的交互流程将第一采集参数发送给视频采集端。为节 约交互流程，本实施例优选第一采集控制指令中携带第一采集参数的实施方式。</p>
    <p>[0052]	步骤304、接收视频采集端发送的第一视频码流；</p>

    <p>[0109]	本实施例的视频服务器同样可用于执行本发明实施例提供的视频传输方法的流 程，在根据无线场强信息对视频码流做进一步压缩，以降低视频码流冗余信息提高播放的 流畅度的同时，通过格式转换模块可支持多个移动终端请求同一视频，通过对视频采集端 提供的视频码流进行编码提供满足各个移动终端需求的视频码流，提高视频码流的利用 率；进一步通过更新模块实现对无线场强的实时更新，可以保证对视频码流的压缩与无线 网络状态最相适应，在保证播放流畅度的同时保证播放的清晰度。[0110]	实施例八</p>
    <p>[0111]	本发明实施例八提供一种视频传输系统，该视频传输系统的结构可参见图1所 示，具体包括：视频采集端11、视频服务器12和移动终端13。其中，视频服务器12可以采 用本发明实施例提供的视频服务器，其具体结构和工作原理请参见上述实施例的描述，在 此不再赘述。</p>
    <p>[0112]	移动终端13，用于向视频服务器12发送视频请求消息，所述视频请求消息中包括 终端信息和无线场强信息，并在视频服务器12生成第二视频码流时接收并播放第二视频 码流。</p>
    <p>[0113]	视频采集端11，用于根据视频服务器12的采集控制指令进行视频采集，并根据采 集参数生成第一视频码流，并向视频服务器12提供第一视频码流，以供视频服务器12根据 第一视频码流获取提供给移动终端13的第二视频码流。</p>
    <p>[0114]	本实施例的视频传输系统可用于执行本发明实施例提供的视频传输方法的流程， 由视频服务器根据无线场强的大小对视频码流做进一步压缩处理，降低视频码流的冗余信 息，提高了无线带宽的利用率，保证了视频码流在移动终端播放的流畅度。</p>
