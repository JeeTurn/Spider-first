    <div class="abstract">本发明涉及一种嵌入到移动终端的金融IC卡空中圈存方法和系统，属于移动支付领域。本发明所述的方法以POS终端方式接入银行现有系统；通过空中圈存服务器及短信中心平台与现有银行系统进行签到、签退的方式获取圈存流程中PIN的转加密保护密钥和通信报文MAC计算密钥；定时执行签退和签到，使工作密钥在每次签到时动态更新。采用本发明所述的方法和系统解决了不用新增或改造银行现有任何系统及服务，即可实现对嵌入到移动终端的金融IC卡进行圈存的问题，既不依赖于移动终端的植入改造，又免除了银行对原有系统的改造和升级，同时保证了金融IC卡交易的高安全性。</div>

    <p>一种嵌入到移动终端的金融IC卡空中圈存方法和系统</p>
    <p>技术领域</p>
    <p>[0001]	本发明属于移动支付、IC卡、金融领域，具体涉及一种嵌入到移动终端的金融IC卡空中圈存方法和系统。</p>
    <p>背景技术</p>
    <p>[0002]	近些年移动支付发展迅速，市场上已经出现若干移动支付产品和空中圈存系统。移动支付也称为手机支付，就是允许用户使用其移动终端(通常是手机)对所消费的商品或服务进行账务支付的一种服务方式。单位或个人通过移动设备、互联网或者近距离传感直接或间接向银行金融机构发送支付指令产生货币支付与资金转移行为，从而实现移动支付功能。移动支付将终端设备、互联网、应用提供商以及金融机构相融合，为用户提供货币支付、缴费等金融业务。移动支付主要分为近场支付和远程支付两种，所谓近场支付，就是用手机刷卡的方式坐车、买东西等，很便利。远程支付是指:通过发送支付指令(如网银、电话银行、手机支付等)或借助支付工具(如通过邮寄、汇款)进行的支付方式，如掌中付推出的掌中电商，掌中充值，掌中视频等属于远程支付。空中圈存是指通过操作手机UTK/STK(SIM Tool KU，用户识别应用发展工具)菜单或者手机客户端，使用OTA或者BIP等无线方式与后台支付系统进行通信，完成对手机智能卡内钱包的充值、查询、消费等功能。通俗来说就是通过手机打电话、短信、OTA菜单或定制化的相关手机客户端软件等方式进行圈存，即时将银行卡账户金额划转到手机卡钱包的过程。</p>
    <p>[0003]目前，大部分移动支付是电子钱包/电子存折产品，运营方也大多是电信运营商</p>
    <p>和城市--^通公司。随着我国金融IC卡的迁移，将会有更多的金融IC卡(Integrated</p>
    <p>Circuit，集成电路)被植入到移动终端中作为电子现金使用，既有移动小额支付功能，又有借/贷记卡功能。银行方面，要使用电子现金实现空中圈存，需要对已有系统进行改造，且目前银行的移动支付产品多是基于智能终端的软件应用或对移动终端的模块进行改造，是借助移动终端来实现圈存功能的。</p>
    <p>发明内容</p>
    <p>[0004]	针对现有技术中存在的缺陷，本发明的目的是提供一种嵌入到移动终端的金融IC卡空中圈存方法和系统。该方法和系统能够解决不用新增或改造银行现有任何系统及服务，即可实现对嵌入到移动终端的金融IC卡进行圈存的问题，既不依赖于移动终端的植入改造，又免除了银行对原有系统的任何改造和升级，同时保证了金融IC卡交易的高安全性。</p>
    <p>[0005]	为达到以上目的，本发明采用的技术方案是:一种嵌入到移动终端的金融IC卡空中圈存方法，包括以下步骤:</p>
    <p>[0006]	以POS终端方式接入银行现有系统；</p>
    <p>[0007]	通过空中圈存服务器及短信中心平台与现有银行系统进行签到、签退的方式获取圈存流程中PIN的 转加密保护密钥和通信报文MAC计算密钥；[0008]	定时执行签退和签到，使工作密钥在每次签到时动态更新。</p>
    <p>[0009]	进一步，所述的签到/签退步骤是由空中圈存服务器发起并与银行系统进行通&#943;目。</p>
    <p>[0010]	所述的签到完成空中圈存服务器与银行系统之间的同步工作，主要完成数据同步工作，同步数据包括:工作密钥，批次号，交易日期，交易时间等重要信息。</p>
    <p>[0011]	所述的签退完成系统从正常工作状态到已签退状态的转换，签退后系统需要重新签到才能工作，空中圈存服务器每日凌晨某时刻自动执行签退和再签到，这样每次通过签到动态更换工作密钥、批次号等信息，保证了交易的高安全性。</p>
    <p>[0012]	进一步，空中圈存服务器启动时，就会自动启动签到流程，如果签到不成功，空中圈存服务器稍后继续执行签到任务，直到签到成功；</p>
    <p>[0013]	以下情况空中圈存服务器会重新签到，重新签到时，先执行签退操作，再执行签到:</p>
    <p>[0014]	I)从银行系统返回报文应答码值为PIN格式错或MAC校验错；</p>
    <p>[0015]	2)空中圈存服务器校验银行响应报文MAC值出错。</p>
    <p>[0016]	进一步，圈存流程中，用户在移动终端选择对应的STK菜单、输入圈存金额和绑定的银行账户的PIN密码，以短信方式发出圈存请求，一笔圈存交易共包含IC卡与空中圈存服务器的三次短信交互，空中圈存服务器与银行系统的一次数据通信。</p>
    <p>[0017]	进一步，圈存流程中，当空中圈存服务器接收到卡片圈存确认短信时，下发确认响应，圈存交易结束；若卡片上发的确认短信中卡片执行结果为失败，空中圈存服务器还需对相应交易向银行系统发起冲正操作，银行接收到冲正请求后，将已扣款的圈存金额回退到用户账户中，保证用户资金安全。</p>
    <p>[0018]	一种嵌入到移动终端的金融IC卡空中圈存系统，包括以下装置:</p>
    <p>[0019]	嵌入到移动终端使用的非接触支付金融IC卡及其STK应用单元:用于从STK菜单发起空中圈存请求、校验圈存响应中发卡行响应数据及对发卡行脚本进行认证；</p>
    <p>[0020]	短信中心平台:用于提供无线空中网络完成短信的接收和转发；</p>
    <p>[0021]	空中圈存服务器:用于以POS终端方式接入银行系统，接收卡片终端上发的请求，与银行系统交互完成业务处理，完成签到、签退、圈存及冲正操作；</p>
    <p>[0022]	银行系统:对IC卡请求数据进行认证和校验，与空中圈存服务器进行交互，完成签到、签退、圈存、冲正等业务，也是整个圈存系统的资金来源方。</p>
    <p>[0023]	本发明的效果在于:采用本发明所述的方法和系统，解决了不用新增或改造银行现有任何系统及服务，即可实现对嵌入到移动终端的金融IC卡进行圈存的问题，既不依赖于移动终端的植入改造，又免除了银行对原有系统的任何改造和升级，同时保证了金融IC卡交易的高安全性。</p>

    <p>[0035]	通过空中圈存服务器及短信中心平台与现有银行系统进行签到、签退的方式获取圈存流程中PIN (Personal Identification Number,个人识别码)的转加密保护密钥和通信报文MAC计算密钥；</p>
    <p>[0036]	定时执行签退和签到，使工作密钥在每次签到时动态更新。</p>
    <p>[0037]	一、签到/签退流程</p>
    <p>[0038]	如图2所示，图2是展示空中圈存系统的状态转换示意图，系统只有处在正常工作状态下才能够处理圈存请求。签到/签退流程是由空中圈存服务器发起的、与银行系统进行通信的流程。</p>
    <p>[0039]	签到完成空中圈存服务器与银行系统之间的同步工作，主要完成数据同步工作，同步数据包括:工作密钥，批次号，交易日期，交易时间等重要信息。</p>
    <p>[0040]	签退完成系统从正常工作状态到已签退状态的转换，签退后系统需要重新签到才能工作，空中圈存服务器每日凌晨某时刻自动执行签退和再签到，这样每次通过签到动态更换工作密钥、批次号等信息，保证了交易的高安全性。</p>
    <p>[0041]	空中圈存服务器启动时，就会自动启动签到流程，如果签到不成功，空中圈存服务器稍后继续执行签到任务，直到签到成功。</p>
    <p>[0042]	以下情况空中圈存服务器会重新签到，重新签到时，先执行签退操作，再执行签到:</p>
    <p>[0043]	I)从银行系统返回报文应答码值为PIN格式错或MAC (Message AuthenticationCode,消息权威码)校验错；</p>
    <p>[0044]	2)空中圈存服务器校验银行响应报文MAC值出错。</p>
    <p>[0045]	二、圈存流程</p>
    <p>[0046]	圈存是将用户银行账户中的资金转移至移动终端中金融IC卡中的过程。用户在移动终端选择对应的STK菜单、输入圈存金额和绑定的银行账户的PIN密码，以短信方式发出圈存请求，一笔圈存交易共包含IC卡与空中圈存服务器的三次短信交互，空中圈存服务器与银行系统的一次数据通信。</p>
    <p>[0047]	如图3所示，通过IC卡及其STK应用单元与空中圈存服务器的交互完成基于借贷记应用的IC卡钱包的圈存，本实施例中，圈存流程中的各节点交互详细步骤如下:</p>
    <p>[0048]	I)用户从STK菜单选择应用目录，STK应用向卡片底层发起SELECT PPSE(Proximity Payment Systems Environment,近距离支付系统环境)指令；</p>
    <p>[0049]	2) STK应用向卡片底层发出选择应用AID (Application Identif ier，应用标识符)的指令，卡片底层按借贷记规定的格式响应；</p>
    <p>[0050]	3)卡片向空中圈存服务器发送圈存准备上行短信，申请具体的TOOL数据(Processing Options Data Object List,处理选项数据对象列表);</p>
    <p>[0051]	4)空中圈存服务器按照短信通信协议返回TOOL数据给卡片，PDOL数据中包含签到响应中从银行系统获取的交易日期、交易时间等信息；</p>
    <p>[0052]	5)卡片应用初始化操作。在应用初始化处理中，STK应用发送GET PROCESSINGOPTIONS命令给卡片通知卡片交易处理开始。在命令中，终端提供给卡片在处理选项数据对象列表(PDOL)中请求的数据元。命令相应信息包括:AIP和AFL，AIP (ApplicationInterchange Profile,应用交互特征)列出了交易在处理过程中执行的功能；AFL(Application File Locator,应用文件定位器)列出交易需要读出的数据存放的短文件标识符、记录号、记录个数以及脱机数据认证需要的静态签名数据的存放位置；</p>
    <p>[0053]	6) STK 应用读数据(Read Record 指令)；</p>
    <p>[0054]	7)STK 应用进行脱机数据认证，产生 ARQC (Authorization Request Cryptogram,授权请求密文)，表示请求联机授权；</p>
    <p>[0055]	8)卡片组织圈存请求上行短信发往空中圈存服务器；</p>
    <p>[0056]	9)空中圈存服务器在工作密钥保护下，组织基于PBOC (People' s Bank ofChina,中国人民银行)的借贷记小额支付圈存请求报文给银行系统，银行系统完成扣款，并下发 ARPC (Authorization Response Cryptogram,授权响应密文)；</p>
    <p>[0057]	10)卡片收到下行短信后，对ARPC进行外部认证，一次交易只执行一次外部认证；</p>
    <p>[0058]	11)卡片执行第二条Generate AC指令(Generate AC指令在一次交易中最多执行两次)，此次执行，生成的应用密文类型为一交易证书(TC, Transaction Certif icate),表示批准交易；</p>
    <p>[0059]	12) STK应用发送I3UT DATA指令修改卡内余额；</p>
    <p>[0060]	13)卡片上发圈存确认短信，通知卡片的处理结果给空中圈存服务器；</p>
    <p>[0061]	14)空中圈存服务器下发圈存确认响应短信,一笔圈存交易结束。</p>
    <p>[0062]	三、冲正流程</p>
    <p>[0063]	圈存流程中，当空中圈存服务器接收到卡片圈存确认短信时，下发确认响应，圈存交易结束；若卡片上发的确认短信中卡片执行结果为失败，空中圈存服务器还需对相应交易向银行系统发起冲正操作，银行接收到冲正请求后，将已扣款的圈存金额回退到用户账户中，保证用户资金安全。</p>
    <p>[0064]	根据上述实施例，可以看出，本发明实现了既不依赖于移动终端的植入改造，又免除了银行对原有系统的任何改造和升级，同时保证了金融IC卡交易的高安全性。因此，本发明使用户可以随时随地为移动终端中的金融IC卡充值，避免了去营业厅或者充值POS网点的麻烦；本发明可以无缝接入现有的银行系统，无需升级和改造现有银行系统即可实现移动支付，将推动金融IC卡在移动支付产业的发展。[0065]	本领域技术人员应该明白，本发明所述的方法和系统并不限于具体实施方式中所述的实施例，上面的具体描述只是为了解释本发明的目的，并非用于限制本发明。本领域技术人员根据本发明的技术方案得出其他的实施方式，同样属于本发明的技术创新范围，本发明的保护范围由权利要求及其等同物限定。</p>
