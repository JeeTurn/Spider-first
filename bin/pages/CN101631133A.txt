    <p>一种域名解析系统、设备及方法</p>
    <p>技术领域</p>
    <p>本发明涉及通信技术领域的域名解析技术，尤其涉及一种域名解析系 统、设备及方法。</p>
    <p>背景技术</p>
    <p>随着科技的发展，手机上网成为可能，目前的手机通常采用无线应用协</p>
    <p>议（Wireless Application Protocol, WAP )，通过WAP网关进行上网，用户 不需要知道要访问业务的域名系统（Domain Name System, DNS)的IP地 址，也不需要了解DNS流程，就可以通过统一资源定位符（Universal Resource Locator, URL )访问对应的业务月l务器，例如：多々某体消息业务（Multimedia Message Service, MMS )、下载业务（Download Service, DS )和WAP浏 览业务等。</p>
    <p>但是对于一些新业务，例如视频共享（Video Share, VS)、无线一键 通（Push to talk over Cellular, PoC)、辅助全球定位系统（Assistant Global Positioning System ， AGPS)等，由于业务流程不采用WAP网关代理方式 访问服务器，无法利用WAP网关解析业务服务器的URL，用户需要直接通 过服务提供商（Service Provider, SP)的IP地址访问业务，用户需要记录 SP的IP地址，对业务的后续扩展带来不便。</p>
    <p>发明内容</p>
    <p>有鉴于此，实有必要提供一种使得方便业务后续扩展的域名解析方法， 以解决非WAP业务不能进行域名解析域的问题。</p>
    <p>同时，提供一种域名解析系统，来解决数据业务不能通过WAP网关代 理方式访问服务器解析域名的问题。</p>
    <p>5同时，提供一种DNS代理服务器，来解决数据业务不能通过WAP网 关代理方式访问服务器解析域名的问题。</p>
    <p>同时，提供一种移动终端，来解决数据业务不能通过WAP网关代理方 式访问服务器解析域名的问题。</p>
    <p>一种域名解析方法，包括：</p>
    <p>接收移动终端发送的域名解析请求；</p>
    <p>向域名系统DNS或电子号码域名系统ENUM DNS查询所述域名解析请 求查询的域名所对应的IP地址；</p>
    <p>向所述移动终端下发与所述域名解析请求对应的IP地址。 一种域名解析方法，包括：</p>
    <p>向域名系统DNS代理服务器发送域名解析请求；</p>
    <p>接收由所述DNS代理服务器下发的与所述域名解析请求对应的IP地址。</p>
    <p>一种域名解析系统，包括：</p>
    <p>GGSN，用于根据移动终端的激活请求，向所述移动终端下发所述DNS 代理服务器的地址信息；</p>
    <p>DNS代理服务器，用于接收所述移动终端发送的域名解析请求，向域 名系统DNS或电子号码域名系统ENUM DNS查询所述域名解析请求查询的 域名所对应的IP地址，及向所述移动终端下发所述与所述域名解析请求对 应的IP地址。</p>
    <p>一种域名系统DNS代理服务器，包括：</p>
    <p>域名解析请求接收模块，用于接收移动终端发送的域名解析请求； 域名查询模块，用于向域名系统DNS或电子号码域名系统ENUM DNS</p>
    <p>查询所述域名解析请求查询的域名所对应的IP地址；</p>
    <p>域名地址下发模块，用于向所述移动终端下发所述与所述域名解析请求</p>
    <p>对应的IP地址。</p>
    <p>域名地址下发^t块，用于向所述移动终端下发所述与所述域名解析请求对应的IP地址。</p>
    <p>一种移动终端，包括：</p>
    <p>域名解析请求模块，用于向域名系统DNS代理服务器发送域名解析请</p>
    <p>求；</p>
    <p>IP地址接收模块，用于接收由所述DNS代理服务器下发的与所述域名 解析请求对应的IP地址；</p>
    <p>数据通信连接模块，用于根据所述IP地址进行数据通信。 与现有技术相比，本发明实施例通过DNS代理服务器接收域名解析请 求，并通过DNS代理月良务器向域名系统DNS或电子号码域名系统ENUM DNS查询所述域名解析请求查询的域名所对应的IP地址，下发给移动终端， 使得移动终端可以不需要记录SP的IP地址就可以通过URL来访问业务， 使得非WAP业务也可以通过URL来访问，有利于新业务的推广及域名解析 系统的完善。</p>

    <p>请参看图1 ，本发明实施例域名解析系统包括：网关GPRS ( General Packet Radio Service,通用分组无线业务）支持节点GGSN104,用于才艮据移 动终端102的激活请求，向移动终端（Mobile Station, MS) 102下发DNS 代理服务器108的IP (Internet Protocol,网际协议）地址；DNS代理服务器 108,用于接收移动终端102发送的域名解析请求，并向域名系统DNSllO或电子号码域名系统ENUM ( Electronic Numbering ) DNS 112查询域名的 IP地址，同时，向移动终端102下发域名的IP地址。</p>
    <p>移动终端102，用于向GGSN104发起PDP ( Packet Data Protocol,分组 数据协议）激活请求，并向DNS代理服务器108请求解析域名，得到域名 IP地址后，根据该IP地址与SP进行通信。其中，DNS代理服务器的地址 可以从GGSN返回的PDP激活响应中获取，也可以在移动终端中直接配置。 本实施例中，移动终端102可以是PC (Personal Computer,个人电脑）、 STB (Set-top Box,才几顶盒）、NB (Notebook Computer,笔i己本电脑）、 HS( Handset,手才几）、GP( Game Player,游戏才几）或ODD( Optical Disc Drive, 光碟机）等具备通信功能的移动终端设备，本实施例中以手机为例进行说明。</p>
    <p>GGSN104用于接收移动终端102发起的分组数据协议PDP激活请求后， 向移动终端返回激活响应信息，本实施例中在激活响应信息中携带DNS代 理服务器108的地址信息。</p>
    <p>该系统还包括：数据服务器106，用于存储移动终端102的IP地址与 终端标识的对应关系，本实施例中，终端标识为手机号码。数据服务器106 可以是AAA ( Authentication, Authorization, and Accounting, i人i正4受斗又i十费) 服务器或Radius服务器。</p>
    <p>DNS代理服务器108，用于接收移动终端102发送的域名解析请求，向 DNSllO查询或ENUM DNS112查询域名解初&#12316;清求查询的域名所对应的IP 地址，及向移动终端102下发与域名解析请求对应的IP地址。DNS代理服 务器108，还可以判断域名解析请求对应的移动终端102的IP地址是否是归 属域地内的业务，若是归属地内的业务，根据移动终端的IP地址向数据服 务器106查询移动终端102的手机号码或E-mail地址等终端标识信息。</p>
    <p>DNSllO,用于才艮据域名解析请求返回对应的IP地址，DNSllO中存储 有域名与IP地址的对应关系表，或DNSllO可访问的存储器中存储有域名 与IP地址对应关系表。ENUMDNS112,用于根据移动终端102的终端标识， 查询用户域名解析请求，如：URL (Universal Resource Locator,统一资源定位符）所对应的IP地址。</p>
    <p>请结合参看图2，本实施例中MS102包括：激活请求模块202，响应接 收模块204、域名解析请求模块206、 IP地址接收模块208和数据通信连接 模块210。</p>
    <p>激活请求模块202，用于向GGSN104发起分组数据协议PDP激活请求。</p>
    <p>响应接收模块204,用于接收GGSN104返回的激活响应，该激活响应 中携带DNS代理服务器108的IP地址。</p>
    <p>域名解析请求模块206,用于根据DNS代理服务器108的IP地址，向 DNS代理服务器108发送有GGSN104激活后的域名解析请求，如URL地 址，其对应的业务如：纟见频共享Video Share、无线一键通PoC或辅助全J求 定位系统AGPS业务等。</p>

    <p>激活请求接收模块302,用于接收移动终端102发起的PDP激活请求。</p>
    <p>激活响应下发冲莫块304,用于向移动终端102返回激活响应，该激活响 应中携带DNS代理服务器108的地址。本实施例中，在GGSN104上设置需 要路由的DNS代理服务器108的IP地址，移动终端102发起激活请求时， 直接提供给移动终端102。当然，GGSN104也可以从其它设备获取需要路由 的DNS代理服务器108的IP地址。</p>

    <p>终端标识查询模块406，用于在域名解析请求对应的业务是归属地内的 业务时，查询所述移动终端102的终端标识信息，如手机号码或E-mail地 址。查询到所述终端标识信息后发送给域名查询模块408。</p>

    <p>本发明实施例通过DNS代理服务器108接收域名解析请求，并通过DNS 代理服务器108向域名系统DNS110或电子号码域名系统ENUM DNS112 查询所述域名解析请求查询的域名所对应的IP地址，下发给移动终端102， 使得移动终端102可以不需要记录SP的IP地址就可以通过URL来访问新 业务，使用起来非常方便，有利于新业务的推广及域名解析系统的完善。并 且通过ENUM DNS112 4艮据终端102的终端标识信息来确定新业务URL对 应的IP地址，使得不是该区域用户的终端可以访问归属地的业务，方便业 务的扩展。</p>

    <p>步骤506，移动终端根据DNS代理服务器的IP地址，向DNS代理服务 器发送域名解析请求，如URL对应的IP地址解析请求。本实施例中，域名 解析请求是GGSN激活后的域名请求，在其他实施例中，可以是DNS代理 服务器直接接收移动终端的域名解析请求。</p>

    <p>步骤512，数据服务器向DNS代理服务器返回查询到的终端标识信息。</p>
    <p>步骤514, DNS代理服务器向DNS或ENUM DNS查询域名解析请求对 应的IP地址，例如URL对应的IP地址。其中，ENUM DNS查询URL对应 的IP地址时，需要根据移动终端的终端标识来查询。</p>
    <p>步骤516, DNS代理服务器接收DNS或ENUM DNS返回的查询到的域 名解析请求的IP地址。</p>
    <p>步骤518, DNS代理服务器向移动终端下发SP对应的IP地址。包括向 所述移动终端下发所述与所述域名解析请求对应的^^网IP地址或私网IP地 址。</p>
    <p>步骤520,移动终端根据SP的IP地址与SP之间建立通信连接。</p>
    <p>本发明实施例通过DNS代理服务器接收由GGSN激活后的域名解析请 求，并通过DNS代理服务器向域名系统DNS或ENUM DNS查询所述域名 解析请求查询的域名所对应的IP地址，下发给移动终端，使得移动终端可 以不需要记录SP的IP地址就可以通过URL来访问新业务，使用起来非常 方便，有利于新业务的推广及域名解析系统的完善。并且通过ENUM DNS 根据终端的终端标识信息来确定新业务URL对应的IP地址，使得不是该区 域用户的终端可以访问归属地的业务，方便业务的扩展。</p>
