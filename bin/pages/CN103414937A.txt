    <p>一种基于数字电视中间件的服务信息数据共享方法</p>
    <p>技术领域</p>
    <p>[0001]	本发明属于数字电视技术领域，特别涉及一种基于数字电视中间件的服务信息数据共享方法。</p>
    <p>背景技术</p>
    <p>[0002]	模拟电视信号带宽大致能传输几十个频道的电视节目，而数字化的电视信号，使同等的带宽上须承载更多的节目流。目前，数字电视能看到几百个左右的电视频道，这样，一个既能方便用户频道的检索，又能方便广电运营商的频道编排、部署和管理，功能强大、使用方便的电视节目信息管理系统就显得至关重要。</p>
    <p>[0003]	数字电视中间件的出现，为广电运营商的业务提供一个相对统一的管理平台，使得跨域的业务互通变得更加简单。数字电视中间件技术可以实现从众多的电视频道中筛选出用户喜爱的电视频道，或者使用户方便地对众多的电视频道进行分类和浏览等功能。</p>
    <p>[0004]	通过对底层硬件资源的抽象，提供对上层各种应用的支持，同时，中间件平台负责屏蔽底层硬件差异性，并将底层资源模块抽象为上层API接口供第三方业务模块调用。而终端平台是整个数字电视平台呈现的载体，它负责接收前端播发的各种数据和指令，并进行解析和呈现。在中间件平台里，终端平台在负责完成基本的音视频播放的同时，也负责接收前端广播的应用，并根据用户的操作运行这些应用，使得业务的开展成为可能。数字电视中间件的功能日益增强，由原来的单纯看电视演变为使用电视，使用电视上网浏览，点播影片，查询信息，玩游戏等，越来越多的电视平台的应用被开发。这就要求电视中间件具备能够启动，管理，控制应用的管理系统。</p>
    <p>[0005]	现有技术中数字电视对主要功能的实现有两种方式:其一是，主应用是运行在javaVM上的java应用，实现搜台，切台等电视观看操作的功能，并且管理其他应用的启动，运行，退出等；其二是，主应用是HTML+JS的网页应用，同样实现搜台，切台等电视观看操作的功能，并且管理其他应用的启动，运行和退出等。两种不用形式的应用实现相同的功能。所以该方法的实现使数字电视中间件具备这样的功能。</p>
    <p>发明内容</p>
    <p>[0006]	本发明的目的是提供一种数字电视中间件的节目信息管理方法，主要是对繁多的节目进行管理，使用户更容易找到自己喜欢的节目，也同时为广电运营商提供方便。使节目数据既可以为java应用使用，也可以为html+js网页应用所使用，做到数据共享，同步。用户可以通过搜台操作搜索电视节目，并且保持到存储器，中间件会将搜索的节目信息转换成一个JSON(JavaScript Object Notation)文件(是一种轻量级的数据交换格式。它基于JavaScript的一个子集，方便JS调用解析)，在中间件浏览器引擎下，主应用(html+javascript实现)通过js接口进行调台，音量加减,启动退出应用等操作；用户也可以不用搜台，直接通过终端的双向网络从运营商的服务器上下载JSON文件，中间件系统通过该方法将JSON文件进行解析，在内存中生产相应的数据结构，并写入到存储器中，这样也可以在中间件javaVM系统中,供java应用进行数据读取，修改，存储等，以便达到调台，音量加减，启动退出应用等操作。</p>
    <p>[0007]	本发明的技术方案是，一种基于数字电视中间件的服务信息数据共享方法，包括以下步骤:</p>
    <p>[0008]	其一，采用包括手动、自动或全频道方式进行频道搜索；通过OC下载NIT、</p>
    <p>[0009]	PAT、PMT、SDT或BAT的表；解析，处理服务信息，并保存到存储器中；将获取的节目信息根据数据结构由JavaVM通过java应用读取存储器的节目信息数据；通过运行中的JAVA应用，实现对观看电视的切台、音量加减的操作；</p>
    <p>[0010]	其二,运营商服务器节目管理系统采用html+javascript实现的主应用通过ajax请求数据文件经网络下载节目信息json文件,Browser通过javascript读取json文件数据；通过运行中的HTML应用，实现对观看电视的切台、音量加减的操作；</p>
    <p>[0011]	所述的数字电视中间件将所述的获取的节目信息转换成json文件保存到存储器，所述的主应用通过js接口访问节目信息数据，实现在同一平台下，java应用和html对服务信息的共享和融合。</p>
    <p>[0012]	本发明重点是服务信息(service inforrmation),即SI信息的转换，其一是根据内存中SI数据，调用中间件的接口函数create_json_file (bool need)生产 JSON (JavaScript Object Notation)文件:即将内存中的 TSInfo, ServiceInfo,Channellnfo, Bounquetlnfo, PrivateData 等数据，通过 sidatabase 的 API 得到，然后按照JSON文件的格式生成JSON文件供JavaScript调用；其二是根据网络请求服务器得到的JSON 文件，调用中间件的接 口函数 json_pase (const json_char*jsonf iIe)解析 JSON 文件，并且按照设定好的结构更新到内存中，同时写到本地存储器里。</p>
    <p>[0013]	本发明使得该数字电视中间件具有既能支持HTML+JS方式的应用通过JSON文件方式存取，服务信息(service inforrmation),即SI信息,又能支持java应用方式存取SI信息，并且二者可以同时使用，做到数据共享，数据同步。</p>
