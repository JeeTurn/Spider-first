    <div class="abstract">本发明公开了一种控制信息的解析方法，获取承载DCI的PDCCH所在资源的编号，使用获取的资源的编号对取模参数进行取模计算；根据取模计算的余数，判断所述资源编号对应的资源上承载的DCI的格式，所述DCI的格式包括正常格式和节省比特格式，所述节省比特格式的DCI的长度小于正常格式的DCI的长度，使用判断出的正常格式或节省比特格式的DCI的长度对PDCCH进行盲检测，得到所述DCI的各个字段。因节省比特格式的DCI的长度比正常格式的DCI的长度小，基站对每个资源上除了回退格式只下发一种格式的DCI，所以没有增加DCI长度的种类，实现了在不增加盲检测次数的同时，又增强了物理下行控制信道。</div>

    <p>控制信息的解析方法及终端</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及通讯技术领域，具体涉及�种控制信息的解析方法及&#32066;端。</p>
    <p>背景技术</p>
    <p>[0002]在长期演进(LTE,Long	Term Evolution)系统中，用户设备(UE,User Equipment)接收或发送业务数据之前，需要获知演进基站(eNB，Evolved NodeB)配置给该用户设备的下行控制信息(DCI, Downlink Control Information),该DCI通过物理下行控制信道(PDCCH,Physical Downlink Control channel)承载。姆个F1DCCH的大小称为聚合级别(AL,Aggregation Level),可以由1、2、4或者8个连续的控制信道单元(CCE,Control ChannelElement,)聚合而成。</p>
    <p>[0003]	UE通过传输模式的配置知道自己需要检测哪两种DCI格式(�种正常格式，�种回退格式)，但UE不知道承载发送给自己的DCI的HXXH的聚合级别，也不知道承载发送给自己的DCI的PDCCH的CCE起始位置，于是UE用已经知道的两种DCI的长度去对搜索空间做逐一盲检测，如果UE检测到循环冗余校验(CRC, Cyclic Redundancy Check)是用自己的无线网络临时标识符(RNTI, Radio Network Temporary Identifier)加扰,那么UE就知道这个DCI是发给自己的，也就得到了相应的DCI格式的所有字段，如调制方式，从而进一步解出DCI内容。这就是所谓的“盲检测”过程。UE寻找承载其HXXH的可能CCE位置称为搜索空间。搜索空间可分为公共搜索空间(CSS, Common search space)和用户设备专用搜索空间(USS 或 DSS, UE-specific/Dedicated search space), CSS 用于承载发给一组UE (如DCI 3/3A)或所有UE的公共控制信息，USS用于承载发给某�个UE的DCI信息。对于不同的格式的DCI在不同的空间里搜索。</p>
    <p>[0004]	现在的DCI有十几种格式，对于一种长度(size)的DCI,则UE可以一次盲检测出来，如果两种格式DCI的size &#8212;祥，则不会&#22679;加盲检测次数，如果有不同的长度的DCI，则需要多次盲检测。在第八版本(Rel-8)中，CSS需要盲检测DCI format IC和DCI format0/1A/3/3A (这4种格式的DCI的size相同)，也就是需要盲检测两种size的DCI。在DSS中，由于DCI formatl/lB/lD/2/2A不会同时出现，当UE的传输模式确定之后，只会盲检其中�种格式，因而在DSS中需要盲检测回退格式和format 1/1B/1D/2/2A中的�种，也是盲检两种size的DCI。Rel-8的最大盲检测为44次。在Rel-1O中增加了两种新的格式2C和4，2C通过传输模式来区分，不会增加盲检测次数，format4则是�种新的用于上行多输入多输出(UL MIM0)格式，拥有新的size，因其只在DSS传输，故对于使用UL MMO的UE其DSS空间的盲检测次数增加16次。</p>
    <p>[0005]	本发明的发明人发现，在现有技术中，PDCCH上承载的DCI有限，如果增加I3DCCH上承载的DCI数量，则会增加format size的种类，进而导致盲检测次数的增加。</p>

    <p>[0032]	在现有技术中，要增强HXXH，也就是在HXXH上多承载�些DCI，可以通过压缩DCI格式来实现，但压缩DCI格式的方法增加了 format size的种类，进而导致盲检测次数的增加。</p>
    <p>[0033]	本发明实施例提供一种控制信息的解析方法，可以在不&#22679;加盲检测次数的同&#21515;，又增强了物理下行控制信道(PDCCH)。本发明实施例还提供相应的终端。以下分别进行详细说明。</p>
    <p>[0034]	&#21443;阅图1，本发明实施例提供的控制信息的解析方法的一实施例包括:</p>
    <p>[0035]	101、获取承载DCI的PDCCH所在资源的编号，所述资源为PDCCH的载体。</p>
