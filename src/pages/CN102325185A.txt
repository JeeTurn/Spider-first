    <p>[0001]	本发明涉及一种基于互联网的移动通讯终端数据筛选下载方法，尤其是指一种基于二维码识别的安卓系统手机软件下载方法。</p>
    <p>背景技术</p>
    <p>[0002]	随着技术的飞速发展，近些年来智能移动终端，特别是智能手机由于其系统（诸如常见的安卓系统）软件上可扩展性愈发得到大众的喜爱。对应的，为了方便智能手机使用者能方便获取最新软件资讯及下载，多采用移动通讯终端的上网功能，通过智能手机输入互联网网址或在手机中预装带互联网浏览功能的软件，而后最终通过访问互联网方式， 至远端对应地址的页面寻找软件资源资讯并供给使用者按喜好下载。然而此数据筛选获取方法需要手机用户访问互联网进行上网一访问远端服务器一浏览一筛选一下载等步骤方可完成一次软件的获取，过程繁杂且对操作者有一定的能力要求，此外该方法不便于运营商直接向客户推广软件的商业需求，而数据的筛选必须通过远程网络来实现也造成使用者在使用过程容易受到无线网的访问速度限制无法及时获取软件信息，产生软件获取失败的问题，进一步致使供应商的软件推广达不到预期效果。</p>
    <p>发明内容</p>
    <p>[0003]	本发明的目的在于克服了上述缺陷，提供一种基于二维码识别且可直接获取手机软件下载的方法。</p>
    <p>[0004]	本发明的目的是这样实现的：一种基于二维码识别的安卓系统手机软件下载方法，它包括：</p>
    <p>[0005]	A)、安装拍照软件及本地客户端的步骤，所述客户端向手机系统注册一固有类型的地址，所述客户端由拍照软件获取图片信息；</p>
    <p>[0006]	B)、获取含软件地址的二维码图片的步骤；</p>
    <p>[0007]	拍照软件通过拍照或读取图片方式获取含软件地址的二维码图片；</p>
    <p>[0008]	C)、解析二维码图片，从而获得地址的步骤；</p>
    <p>[0009]	拍照软件对获取的二维码图片进行解析，从而获得其内包含的地址；</p>
    <p>[0010]	D)、判断该地址的类型是否被注册的步骤，若已注册则继续，否则获取地址的类型</p>
    <p>注册；</p>
    <p>[0011]	拍照软件判断该地址的类型是否在手机系统中已被注册，若已注册则继续，否则获取地址的类型注册；</p>
    <p>[0012]	上述步骤中，获取地址的类型注册具体步骤包括，</p>
    <p>[0013]	dl)、访问互联网，以下载获取客户端；</p>
    <p>[0014]	d2)、客户端是否获取成功，是则继续步骤，否则跳出步骤结束；</p>
    <p>[0015]	d3)、请求手机系统在本地安装客户端；</p>
    <p>[0016]	d4)、客户端是否安装成功，是则继续步骤，否则跳出步骤结束。[0017]	E)、根据手机的安卓系统指定的htent消息，将地址发送给安卓手机系统；</p>
    <p>[0018]	拍照软件根据获取到的地址访问手机系统；</p>
    <p>[0019]	F)、手机系统根据地址的类型激活本地客户端，并将地址送入客户端；</p>
    <p>[0020]	G)、本地客户端对送入地址进行处理解析得到要获取的软件在服务器中的信息；</p>
    <p>[0021]	所述解析是通过手机系统API处理地址，对其进行字符串切分得到参数id的值。 所述字符串切分包括步骤，</p>
    <p>[0022]	gl)、遍历查找“问号”在地址中的位置；</p>
    <p>[0023]	g2)、根据该“问号”位置获得其后的所有字符串；</p>
    <p>[0024]	g3)、对此时字符串以“等号”切分，获得“id”和一个数值的两个字符串；</p>
    <p>[0025]	g4)、上述数值即为地址中id的值。</p>
    <p>[0026]	H)、本地客户端将该软件信息传送给服务器；</p>
    <p>[0027]	I)、服务器根据接收的软件信息至数据库中进行软件应用信息匹配，若未找到匹配内容则返回消息给客户端，客户端提示匹配不成功，否则继续步骤；</p>
    <p>[0028]	J)、服务器返回匹配所得的软件应用信息至客户端；</p>
    <p>[0029]	K)、客户端显示接收的软件应用信息。</p>
    <p>[0030]	相比于常见的手机软件下载方法，本发明的有益效果在于无需用户主动访问互联网搜寻所需软件并安装，而是直接通过二维码方式，用户只需对含有软件地址的二维码拍照，而后通过本地客户端对地址码进行解析获取对应软件信息后再向远端服务器直接调取软件的应用信息，减少了原有获取方法中手机端与服务器端反复访问的步骤，有效提高软件的获取速度及成功率，为运营商有效提高用户使用体验。</p>

    <p>具体实施方式</p>
    <p>[0033]	为详细说明本发明的技术内容、构造特征、所实现目的及效果，以下结合实施方式并配合附图详予说明。</p>
    <p>[0034]	请参阅图1，本发明涉及一种基于二维码识别的安卓系统手机软件下载方法，它包括步骤：</p>
    <p>[0035]	A)、安装拍照软件及本地客户端，客户端安装时会向手机系统注册一固有类型的地址，此外其会在后续步骤中由拍照软件获取图片信息；</p>
    <p>[0036]	如，在安卓（Android)系统中，注册一种类型地址，地址格式：iandroid:// detail ？ id = {应用在服务器的唯一编号}，例：</p>
    <p>[0037]地址：iandroid://detail	？ id = 1319</p>
    <p>[0038]	格式：scheme:"host ？ key = value</p>
    <p>[0039]	1) scheme ：协议名称，地址中为 “iandroid”，等同于 http://www. baidu. com 中的 "http"</p>
    <p>[0040]	2)host ：主机名，地址中为 “detail”，等同于 http：//www. baidu. com 中的"www.baidu. com，，</p>
    <p>[0041]	3) ？：指定问号后面的字符都为参数（key = value)</p>
    <p>[0042]	4)key = value :键值对形式的参数。</p>
    <p>[0043]	B)、拍照软件通过拍照或读取图片方式获取含软件地址的二维码图片；</p>
    <p>[0044]	C)、拍照软件对获取的二维码图片进行解析，从而获得其内包含的地址；</p>
    <p>[0045]	D)、拍照软件判断该地址的类型是否在手机系统中已被注册，若已注册则继续，否则获取地址的类型注册，地址的类型注册通常通过安装本地客户端来向手机系统注册，获取地址的类型注册具体步骤包括，</p>
    <p>[0046]	dl)、访问互联网，以下载获取客户端；</p>
    <p>[0047]	d2)、客户端是否获取成功，是则继续步骤，否则跳至步骤L结束；</p>
    <p>[0048]	d3)、拍照软件请求手机系统在本地安装客户端；</p>
    <p>[0049]	d4)、客户端是否安装成功，是则继续步骤，否则跳至步骤L结束。</p>
    <p>[0050]	E)、拍照软件根据获取到的地址访问手机系统；</p>
    <p>[0051]	F)、手机系统根据地址的类型激活本地客户端，并将地址送入客户端；</p>
    <p>[0052]	G)、本地客户端对送入地址进行处理解析得到对应软件信息（id)；</p>
    <p>[0053]	本步骤中的解析是客户端通过手机系统API处理地址，对其进行字符串切分从而得到参数id的值的，具体的说地址字符串切分是指：</p>
    <p>[0054]	目的：通过切割获取地址中的键值参数（参数id及对应的值)</p>
    <p>[0055]	方法流程：</p>
    <p>[0056]	gl)、遍历查找“？”（问号)在地址中的位置；</p>
    <p>[0057]	g2)、根据该问号位置获得其后的所有字符串，如上述实施例中地址dandroid:// detail ？ id = 1319，即取得字符串"id = 1319” ；</p>
    <p>[0058]	g3)、同理对“id = 1319”字符串以“=”（等号）切分，获得“id”和“ 1319”两个</p>
    <p>字符串；</p>
    <p>[0059]	g4)、得到地址中id的值，1319。</p>
    <p>[0060]	H)、本地客户端通过互联网协议（如HTTP)将该软件信息（id)传送给服务器；</p>
    <p>[0061]	I)、服务器根据接收的软件信息（id)至数据库中进行软件应用信息匹配，若未找到匹配内容则返回消息给客户端，客户端提示匹配不成功（即提示诸如“未找到相关内容” 等信息），否则继续步骤；</p>
    <p>[0062]	J)、服务器返回匹配所得的软件应用信息至客户端；</p>
    <p>[0063]	K)、客户端显示接收的软件应用信息；</p>
    <p>[0064]	L)、结束。</p>
    <p>[0065]	以上所述仅为本发明的实施例，并非因此限制本发明的专利范围，凡是利用本发明说明书及附图内容所作的等效结构或等效流程变换，或直接或间接运用在其他相关的技术领域，均同理包括在本发明的专利保护范围内。</p>
