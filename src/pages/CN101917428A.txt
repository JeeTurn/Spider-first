    <p>家庭媒体视频播放的方法、系统及移动通信终端</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及媒体视频播放技术，尤其涉及家庭媒体视频播放的方法、系统及移动 通信终端。</p>
    <p>背景技术</p>
    <p>[0002]	目前，在家庭媒体视频技术中，数字电视已经越来越普及，为了进行电视节目的播 放，需要为每台电视设置一个机顶盒。具体地，进行播放的流程包括：机顶盒通过无线方式 接收遥控器发送的播放请求，该播放请求中包含请求播放的节目信息；然后机顶盒从视频 服务器获取与该节目信息对应的传送流（TS，Transport Stream)，对TS进行解码后传送给 电视机进行播放。所述视频服务器也就是提供电视节目的服务器。</p>
    <p>[0003]	如果手机等移动通信终端能够通过机顶盒播放电视节目，则可以在不产生额外费 用的情况下，实现由移动通信终端进行电视节目的播放。然而，现有的家庭媒体视频播放方 案中，没有手机等移动通信终端通过机顶盒播放视频节目的技术。</p>
    <p>发明内容</p>
    <p>[0004]	本发明提供了一种家庭媒体视频播放的方法，该方法能够在不产生额外费用的情 况下，实现由移动通信终端播放电视节目。</p>
    <p>[0005]	本发明提供了一种家庭媒体视频播放的系统，该系统能够在不产生额外费用的情 况下，实现由移动通信终端播放电视节目。</p>
    <p>[0006]	本发明提供了一种移动通信终端，该移动通信终端能够在不产生额外费用的情况 下，进行电视节目的播放。</p>
    <p>[0007]	&#8212;种家庭媒体视频播放的方法，该方法中，移动通信终端和机顶盒连接上无线相 容性认证（WIFI，Wireless Fidelity)设备，该方法还包括：</p>
    <p>[0008]	移动通信终端通过WIFI设备向机顶盒发送播放请求，该播放请求中包含移动通 信终端的网际协议（IP，Internet Protocol)地址和端口，还包含请求播放的节目信息；</p>
    <p>[0009]	机顶盒接收所述播放请求后，从视频服务器获取与所述节目信息对应的TS，通过 WIFI设备发送到与所述IP地址和端口对应的移动通信终端；</p>
    <p>[0010]	移动通信终端对接收到的TS进行解码，并播放。</p>
    <p>[0011 ] 一种家庭媒体视频播放的系统，该系统包括移动通信终端、WIFI设备和机顶盒，移 动通信终端和机顶盒连接上WIFI设备；</p>
    <p>[0012]	所述移动通信终端，通过WIFI设备向机顶盒发送播放请求，该播放请求中包含移 动通信终端的IP地址和端口，还包含请求播放的节目信息；通过WIFI设备接收机顶盒发送 的TS，进行解码，并播放；</p>
    <p>[0013]	所述机顶盒，通过WIFI设备接收移动通信终端发送的所述播放请求，从视频服务 器获取与所述节目信息对应的TS，通过WIFI设备发送到与所述IP地址和端口对应的移动 通信终端。[0014]	一种移动通信终端，该移动通信终端与WIFI设备连接，其包括第二无线收发模块 和解码播放模块；</p>
    <p>[0015]	所述第二无线收发模块，用于通过WIFI设备向机顶盒发送播放请求，该播放请求 中包含移动通信终端的网际协议IP地址和端口，还包含请求播放的节目信息；并通过WIFI 设备接收机顶盒发送的TS，将TS传送给解码播放模块；</p>
    <p>[0016]	所述解码播放模块，用于接收第二无线收发模块发送的TS，进行解码，并播放。</p>
    <p>[0017]	从上述方案可以看出，本发明将移动通信终端和机顶盒连接上WIFI设备，之后， 移动通信终端通过WIFI设备向机顶盒请求相应节目的TS，对TS进行解码并播放。由移动 通信终端从机顶盒获取TS进行电视节目的播放，不会产生额外费用，从而，实现了在不产 生额外费用的情况下，由移动通信终端播放电视节目。</p>
    <p>附图说明</p>
    <p>[0018]	图1为本发明家庭媒体视频播放的方法示意性流程图；</p>
    <p>[0019]	图2为本发明家庭媒体视频播放的系统结构示意图。</p>
    <p>具体实施方式</p>
    <p>[0020]	为使本发明的目的、技术方案和优点更加清楚明白，下面结合实施例和附图，对本 发明进一步详细说明。</p>
    <p>[0021]	本发明将移动通信终端和机顶盒连接上WIFI设备，构成由移动通信终端、机顶盒 和WIFI设备组成的局域网；然后，移动通信终端通过WIFI设备向机顶盒请求相应节目的 TS进行播放。由于移动通信终端从机顶盒获取TS进行电视节目的播放，不会产生额外费 用，从而，实现了在不产生额外费用的情况下，由移动通信终端播放电视节目。并且，由于电 视机不易搬动，限定了观看电视节目的位置；而采用本发明方案，可通过移动通信终端在家 庭中任意位置观看电视节目，这样，可灵活变动观看电视节目的位置。</p>
    <p>[0022]	参见图1，为本发明家庭媒体视频播放的方法示意性流程图，该方法中，预先将移 动通信终端和机顶盒连接上WIFI设备，所述WIFI设备具体为无线路由器或接入点（AP， Access Point)设备等，这样，便构成由移动通信终端、机顶盒和WIFI设备组成的局域网。 根据WIFI设备的特点，在组成局域网之后，移动通信终端可以通过WIFI设备获知机顶盒的 IP地址和端口，机顶盒可以通过WIFI设备获知移动通信终端的IP地址和端口。</p>
    <p>[0023]	图1所示的流程包括以下步骤：</p>
    <p>[0024]	步骤101，移动通信终端通过WIFI设备向机顶盒发送播放请求，该播放请求中包 含移动通信终端的IP地址和端口，还包含请求播放的节目信息。</p>
    <p>[0025]	组成局域网之后，可选地，移动通信终端、WIFI设备以及机顶盒三者之间的通信可 以采用通用即插即用(UPnP, Universal Plug and Play)协议实现。此时，所述播放请求 承载于UPnP协议中的建立统一资源定位器连接（set URL content)信令中发送，所述播放 请求中包含的IP地址和端口承载于UPnP协议中的描述协议（SDP，Session Description Protocol)消息中，会话描述协议(SDP, Session Description Protocol)消息包含在 set URL content 信令中。</p>
    <p>[0026]	移动通信终端通过WIFI设备向机顶盒发送播放请求具体为：移动通信终端将播放请求以及机顶盒的IP地址和端口发送给WIFI设备，WIFI设备再根据IP地址和端口将 播放请求发送给对应的机顶盒。</p>
    <p>[0027]	在本步骤之前，移动通信终端需要获取节目列表，获取方式例如为：移动通信终端 从公网获取节目列表，该节目列表包含节目信息。然后，移动通信终端从节目列表选择请求 播放的节目信息，将请求播放的节目信息包含在播放请求中。</p>
    <p>[0028]	步骤102，机顶盒接收所述播放请求后，从视频服务器获取与播放请求中包含的节 目信息对应的TS，通过WIFI设备发送到与播放请求中包含的IP地址和端口对应的移动通</p>
    <p>信终端。</p>
    <p>[0029]	机顶盒通过WIFI设备将TS发送到移动通信终端具体包括：机顶盒将TS以及移动 通信终端的IP地址和端口发送给WIFI，WIFI接收后，将TS传送给与该IP地址和端口对应 的移动通信终端。</p>
    <p>[0030]	步骤103，移动通信终端对接收到的TS进行解码，并播放。</p>
    <p>[0031]	本步骤中移动通信终端对TS进行解码的方法与机顶盒对接收的TS进行解码的方 法相同，例如，可以通过在移动通信终端安装与机顶盒相同的解码软件实现；这里不过多赘 述。</p>
    <p>[0032]	参见图2，为本发明家庭媒体视频播放的系统结构示意图，该系统包括移动通信终 端210、WIFI设备220和机顶盒230，移动通信终端210和机顶盒230连接上WIFI设备220， 构成由移动通信终端210、机顶盒230和WIFI设备220组成的局域网。根据WIFI设备220 的特点，在组成局域网之后，移动通信终端210可以通过WIFI设备220获知机顶盒230的</p>
    <p>IP地址和端口，机顶盒230可以通过WIFI设备220获知移动通信终端210的IP地址和端□。</p>
    <p>[0033]	所述移动通信终端210，通过WIFI设备220向机顶盒230发送播放请求，该播放请 求中包含移动通信终端210的网际协议IP地址和端口，还包含请求播放的节目信息；通过 WIFI设备220接收机顶盒230发送的TS，进行解码，并播放。移动通信终端210通过WIFI 设备220向机顶盒230发送播放请求具体为：移动通信终端210将播放请求以及机顶盒230 的IP地址和端口发送给WIFI设备220，WIFI设备220再根据IP地址和端口将播放请求发 送给对应的机顶盒230。</p>
    <p>[0034]	其中，WIFI设备220可以是单独的设备，也可以集成到机顶盒230中。</p>
    <p>[0035]	所述机顶盒230，通过WIFI设备220接收移动通信终端210发送的所述播放请求， 从视频服务器获取与所述节目信息对应的TS，通过WIFI设备220发送到与所述IP地址和 端口对应的移动通信终端210。机顶盒230通过WIFI设备220将TS发送到移动通信终端 210具体包括：机顶盒230将TS以及移动通信终端210的IP地址和端口发送给WIFI，WIFI 接收后，将TS传送给与该IP地址和端口对应的移动通信终端210。</p>
    <p>[0036]	可选地，所述机顶盒230中包括第一无线收发模块231和TS获取模块232 ；</p>
    <p>[0037]	所述第一无线收发模块231，用于通过WIFI设备220接收移动通信终端210发 送的所述播放请求，传送给TS获取模块232 ；还用于接收TS获取模块232发送的TS，通过 WIFI设备220发送到与所述IP地址和端口对应的移动通信终端210 ；</p>
    <p>[0038]	所述TS获取模块232，用于接收第一无线收发模块231传送的播放请求后，从视频 服务器获取与所述节目信息对应的TS，传送给第一无线收发模块231。[0039]	可选地，所述移动通信终端210包括第二无线收发模块212和解码播放模块213 ；</p>
    <p>[0040]	所述第二无线收发模块212，用于通过WIFI设备220向所述第一无线收发模块 231发送播放请求，并通过WIFI设备220接收所述第一无线收发模块231发送的TS，将TS 传送给解码播放模块213 ；</p>
    <p>[0041]	所述解码播放模块213，用于接收第二无线收发模块212发送的TS，进行解码，并 播放。机顶盒230接收遥控器发送的播放请求后，也会从视频服务器获取相应的TS，并对 TS进行解码，然后发送给电视机进行播放。解码播放模块213对TS进行解码的方案与机顶 盒230对TS进行解码的方案相同，这里不赘述。</p>
    <p>[0042]	可选地，移动通信终端210、WIFI设备220以及机顶盒230三者之间的通信可以采 用UPnP协议实现。相应地，所述播放请求承载于UPnP协议中的set URL content信令中 发送，所述播放请求中包含的IP地址和端口承载于UPnP协议中的SDP消息中；SDP消息包 含在set URL content信令中。</p>
    <p>[0043]	所述WIFI设备220为无线路由器或AP设备等。</p>
    <p>[0044]	移动通信终端210向机顶盒230请求电视节目前，需要获取节目列表；可选地，所 述移动通信终端210还包括节目信息请求模块211，用于公网获取节目列表，该节目列表包 含节目信息。然后，第二无线收发模块212便可从节目列表选择请求播放的节目信息，将请 求播放的节目信息包含在发送给机顶盒230的播放请求中。</p>
    <p>[0045]	本发明所述的移动通信终端包括手机、个人数字助理（PDA，Personal Digital Assistant)等。</p>
    <p>[0046]	本发明构成由移动通信终端、机顶盒和WIFI设备组成的局域网，移动通信终端通 过WIFI设备向机顶盒请求相应电视节目的TS进行解码并播放。移动通信终端从机顶盒获 取TS进行电视节目的播放，不会产生额外费用，实现了在不产生额外费用的情况下通过移 动通信终端播放电视节目。并且，由于移动通信终端可灵活移动位置，可以在家庭中任意位 置观看电视节目，克服了电视机不易搬动，限定了观看电视节目位置的缺陷。</p>
