    <p>网络服务生成系统及方法</p>
    <p>技术领域</p>
    <p>本发明是关于一种生成网络服务（web service)的系统及方法。 背景技术</p>
    <p>Web service是应用程序集成的平台，该平台是一套完全基于XML (extensible markup language,可扩展性标志语言)、XSD ( XML schema definition, XML构架定义）等独立于平台、独立于软件供应商的标准， 其定义了应用程序如何在web上实现互操作性，用户可以用任何喜欢的 编程语言，在任何喜欢的平台上写web service,只要其他用户可以通过 web service标准对这些服务进行查询和访问。</p>
    <p>通常，在web service平台上所集成的应用程序的设计过程中，涉及 对数据源的访问时，针对不同类型的数据源，如关系型数据库（IBM DB2、 IBM UDB、 Oracle、 SQL Server、 SyBase、 Informix) 、 XML数 据库、XML文件、网页等，需要编写特定的访问程序，如访问数据库 时，才艮据凄t据库的应用禾呈序接口 ( application programming interface, API) 函数实现程序访问。当访问的数据源较多，且数据源的类型不同时，这 种方法较为复杂。</p>
    <p>发明内容</p>
    <p>鉴于以上内容，有必要提供一种生成网络服务（web service)的系 统，以XML查询（以下简称XQuery)i吾言标准作为 一个中间平台，通过 XQuery语言扩展对异种数据源进行访问。</p>
    <p>此外，还有必要提供一种生成网络（web service )的方法，以XQuery</p>
    <p>i吾言标准作为一个中间平台，通过XQuery i吾言扩展对异种数据源进行 访问。</p>
    <p>该web service生成系统包括网络服务器，所述网络服务器包括：定 义模块，用于根据数据源的类型定义XQuery文件，在所述XQuery文 件中声明查询参数及数据源地址；分析模块，用于分析所述XQuery文 件，提取该XQuery文件中声明的查询参数；执行模块，用于根据所提 取的查询参数解析所述XQuery文件，从该XQuery文件中声明的数据 源抓取所需要的数据，产生可执行的程序源文件；及网络服务描述语言 文件(web service description language, WSDL)生成才莫块，用于才艮才居所 述程序源文件生成相应的WSDL文件。</p>
    <p>该web service生成方法包括以下步骤：（A)根据数据源的类型定 义XQuery文件，在所述XQuery文件中声明查询参数及数据源地址； (B )分析所述XQuery文件，提取该XQuery文件中声明的查询参数； (C )根据所提取的查询参数解析所述XQuery文件，从该XQuery文件 中声明的数据源抓取所需要的数据，产生可执行的程序源文件；及（D) 根据所述程序源文件生成相应的网络服务描述语言文件。</p>
    <p>本发明所提供的web service生成系统及方法可通过XQuery语言扩 展对异种数据源进行访问。</p>
