    <p>数字证书生成方法、用户密钥获取方法、移动终端及设备</p>
    <p>技术领域</p>
    <p>[0001] 本发明涉及通信领域，尤其涉及一种数字证书生成方法、用户密钥获取方法、移动</p>
    <p>终端及网络设备。背景技术</p>
    <p>[0002] 移动终端设备在电子商务安全环节需要保障的安全范围包括：身份认证、机密性、完整性、抗抵赖。公钥基础设施（Public Key Infrastructure,PKI)技术通过数字签名和数字信封等，可以实现身份认证、机密性、完整性、抗抵赖的保障。由于移动终端设备分为SIM卡和移动终端，而现阶段的SIM卡或者不具备智能芯片不能实现PKI功能，或者具备智能芯片，可以实现PKI功能，却由于机卡通信协议、手机操作系统的问题不能实现机卡通信，因而移动终端不能实现对SIM卡内PKI功能的调用。目前移动终端一般采用基于文件方式的PKI功能，即通过文件存储方式存储用户密钥，调用PKI功能时从该文件种获取用户密钥，然而通过文件存储方式存储用户密钥存在用户密钥容易丢失或被恶意窃取的风险。</p>
    <p>发明内容</p>
    <p>[0003] 鉴于现有技术存在的问题，本发明实施例提出一种数字证书生成方法、用户密钥获取方法移动终端及网络设备，可以保障用户私钥的安全性，降低用户私钥的丢失或被窃取的安全风险。</p>
    <p>[0004]     鉴于现有技术的缺陷，本发明实施例提出一种数字证书生成方法，包括：</p>
    <p>[0005] 接收移动终端发送的申请数字证书的请求信息，该请求信息携带有用户标识及初</p>
    <p>始计数器数值；</p>
    <p>[0006]     登记所述用户标识，并为该用户生成相应的用户公钥和用户私钥；</p>
    <p>[0007] 根据所述用户标识及初始计数器数值生成动态密钥，并利用该动态密钥加密所述</p>
    <p>用户私钥，生成加密后的用户私钥；</p>
    <p>[0008] 根据所述用户标识及所述用户公钥生成用户公钥数字证书。</p>
    <p>[0009]     相应地，本发明实施例还提出一种用户密钥获取方法，包括：</p>
    <p>[0010]      向网络侧发送用户密钥获取请求，该用户密钥获取请求中携带有用户标识；</p>

    <p>5具体实施方式</p>
    <p>[0032] 本发明实施例提出一种数字证书生成方法、用户密钥获取方法移动终端及网络设</p>
    <p>备，可以保障用户私钥的安全性，降低用户私钥的丢失或被窃取的安全风险。</p>
    <p>[0033]      图1是本发明实施例中数字证书生成方法第一实施例流程图，本实施例中数字证</p>
    <p>书生成方法在证书授权中心（Certif icationAuthority， CA)中执行，具体包括以下步骤：</p>
    <p>[0034] 步骤101，接收移动终端发送的申请数字证书的请求信息，该请求信息携带有用户</p>
    <p>标识及初始计数器数值；</p>
    <p>[0035] 此处，具体实施时，该请求信息可以是移动通信网络的网络设备转发的，请求信息 中的用户标识可以是移动终端的国际移动装备辨识码（InternationalMobile Equipment Identity number, IMEI)或国际移云力用户识另U码(InternationalMobile Subscriber Identity, IMSI)，初始计数器数值可以是用户身份识别模块（Subscriber Identity Module, SIM)卡的保存的4字节计数器的初始值。</p>
    <p>[0036]     步骤102，登记所述用户标识，并为该用户生成相应的用户公钥和用户私钥； [0037]     此处，登记移动终端用户标识，并生成记录，并使用RSA、 ECC等公钥算法生成密钥 对，即用户公钥和用户私钥。</p>
    <p>[0038] 步骤103，根据所述用户标识及初始计数器数值生成动态密钥，并利用该动态密钥 加密所述用户私钥，生成加密后的用户私钥；</p>
    <p>[0039] 此处，具体实施时，根据所述用户标识及初始计数器数值生成动态密钥，具体的算 法为：</p>
    <p>[0040]    HOTP (K， C) = Truncate (&#34217;C-SHA-1 (K， C))，其中： [0041]     K为种子，K = MD5(MEI+IMSI);</p>
    <p>[0042]      C为用户身份识别模块的计数器的数值，具体为4字节的计数器的计数器数值； [0043]     Truncate (HMAC-SHA-l (K， C))为动态密钥生成算法，所述动态密钥生成算法具体 为：</p>
    <p>[0044]     HS =&#34217;C-SHA-1 (K， C);</p>
    <p>[0045]     此处，使用Hash信息验证码（Hash Message Authentication Codes,&#34217;C)安全 散列算法（Secure Hash Algorithm, SHA)，即HMAC-SHA，计算得到20字节的摘要数据HS。 [0046]     Offset = Low4Bit(HS[19]);</p>
    <p>[0047]     此处，摘要数据HS由第19位开始的偏移量Offset,对HS数组最后一个字节，取低 4位，得到一个0x0-0xf的值，这个值作为一个偏移量。Offset是一个0_15之间的数，保证 在20字节的数组中根据Offset去定位取四字节时是不会发生越位溢出异常的。 [0048]     HS[Offset] = HS[Offset]&amp;0x8f ;</p>
    <p>[0049] 此处，根据Offset定位，在HS数组中取4字节，并且第一个字节去符号位，且第一 字节做最高位。</p>
    <p>[0050]    Digit = 4BytetoInt(HS[Offset]);</p>
    <p>[0051]     此处，把偏移量Offset处的连续4个字节转换成整型。</p>
    <p>[0052]    Digit = Digit% 1000000。</p>
    <p>[0053] 此处，对上面产生的HS进行截取运算，取一百万模，得到显示的6个数字字符，该 6个数字字符为动态密钥。此时Digit是一个ulong型的长整型，为了取得6位数字，则要取1000000的模（8个数字就取100000000的模）；有的数字取模后，可能真实的数字小于 6位，如5629009876% 1000000 = 9876，此时需要前端补0，最终显示"009876"。以下是 Truncate (&#34217;C-SHA-1 (K， C))的一个实例： [0054]     HS是"Byte Value"，20个字节； [0055]     Offset = 0x5a&amp;0x0f = 0x0a ; [0056]     HS[IO] = HS[10]&amp;0x7f = 0x50 ;</p>
    <p>[0057]    Digit = 0xHS[10]HS[ll]HS[12]HS[13] = 0x50ef7f19 = 1357872921 ; [0058]    Digit = 1357872921% 1000000 = 872921。 [0059]     动态密钥就是872921。</p>
    <p>[0060] 使用该动态密钥与公钥算法加密所述用户私钥，生成加密后的用户私钥。</p>
    <p>[0061]     步骤104，根据所述用户标识及所述用户公钥生成用户公钥数字证书；</p>
    <p>[0062]     此处，用户标识可以是IMSI或MEI，也可以是IMSI或IMEI唯一对应的伪码，用于</p>
    <p>证明申请者的身份。CA中心根据用户MEI、IMSI、或其伪码，以及用户公钥签发用户公钥数</p>
    <p>字证书。</p>
    <p>[0063]     步骤105，向所述移动终端发送申请数字证书成功的通知信息。</p>
    <p>[0064] 此处，此步骤为可选步骤，CA中心签发用户公钥数字证书后，可以通过网络设备通</p>

    <p>中心或移动通信网络的网络设备生成动态密钥的方式相同，在此不再详述。移动终端获取</p>
    <p>解密后的用户私钥后，将本地计数器的数值按照与CA中心或移动通信网络的网络设备中</p>
    <p>相同的计数规则进行计数，生成新的计数器数值，保持移动终端本地计数器中的计数器数</p>
    <p>值与CA中心或移动通信网络的网络设备的计数器中的计数器数值相一致，当移动终端再</p>
    <p>次接收到CA中心或移动通信网络的网络设备下发的加密后的用户私钥时，利用该新的计</p>
    <p>数器数值重新生成新的动态密钥。使用新的动态密钥该对用户私钥进行重新解密。</p>
    <p>[0184]     图7是本发明实施例中移动终端和CA中心交互过程示意图，具体包括以下步骤：</p>
    <p>[0185] 步骤701，移动终端向移动通信网络的网络设备发送申请数字证书的请求信息，该</p>
    <p>请求信息携带有用户标识及初始计数器数值，该网络设备将请求信息转发到CA中心；</p>
    <p>[0186]     步骤702， CA中心接收移动终端发送的申请数字证书的请求信息；</p>
    <p>[0187]     步骤703， CA中心登记所述用户标识，并为该用户生成相应的用户公钥和用户私</p>
    <p>钥；</p>
    <p>[0188] 步骤704， CA中心根据所述用户标识及初始计数器数值生成动态密钥，并利用该 动态密钥加密所述用户私钥，生成加密后的用户私钥；</p>
    <p>[0189]     步骤705， CA中心根据所述用户标识及所述用户公钥生成用户公钥数字证书； [0190]     步骤706， CA中心通过移动通信网络的网络设备向所述移动终端发送申请数字证 书成功的通知信息；</p>
    <p>[0191] 步骤707，移动终端向网络侧发送用户密钥获取请求，该用户密钥获取请求中携带 有用户标识，该网络设备将用户密钥获取请求转发到CA中心；</p>
    <p>[0192] 步骤708，移动终端根据所述用户标识从所述网络侧查询获取利用动态密钥加密 后的用户私钥，并将该加密后的用户私钥缓存在移动终端内存中；</p>
    <p>[0193] 步骤709，移动终端根据所述用户标识及初始计数器数值生成动态密钥，并利用该动态密钥解密所述用户私钥，获取解密后的用户私钥，并将该解密后的用户私钥缓存在内 存中</p>
    <p>[0194] 步骤710，移动终端根据所述用户私钥进行数字签名或解密使用用户公钥加密的 数据；</p>
    <p>[0195] 步骤711，移动终端完成所述数字签名或解密使用用户公钥加密的数据后，清除所 述用户私钥和所述动态密钥。</p>
    <p>[0196] 图8是本发明实施例中移动终端和移动通信网络的网络设备交互过程示意图，具 体包括以下步骤：</p>
    <p>[0197] 步骤801，移动终端向移动通信网络的网络设备发送申请数字证书的请求信息，该 请求信息携带有用户标识及初始计数器数值；</p>
    <p>[0198]     步骤802，网络设备接收移动终端发送的申请数字证书的请求信息；</p>
    <p>[0199] 步骤803，网络设备登记所述用户标识，并为该用户生成相应的用户公钥和用户私</p>
    <p>钥；</p>
    <p>[0200] 步骤804，网络设备根据所述用户标识及初始计数器数值生成动态密钥，并利用该 动态密钥加密所述用户私钥，生成加密后的用户私钥；</p>
    <p>[0201]     步骤805，网络设备向证书授权中心发送所述用户标识信息与所述用户公钥； [0202]     步骤806， CA中心根据所述用户标识信息与所述用户公钥生成用户公钥数字证 书，并返回所述用户公钥数字证书；</p>
    <p>[0203]     步骤807，网络设备接收所述证书授权中心下发的用户公钥数字证书； [0204]     步骤808，网络设备向所述移动终端发送申请数字证书成功的通知信息； [0205] 步骤809，移动终端向网络侧发送用户密钥获取请求，该用户密钥获取请求中携带 有用户标识；</p>
    <p>[0206] 步骤810，移动终端根据所述用户标识从所述网络侧查询获取利用动态密钥加密 后的用户私钥，并将该加密后的用户私钥缓存在移动终端内存中；</p>
    <p>[0207] 步骤811，移动终端根据所述用户标识及初始计数器数值生成动态密钥，并利用该 动态密钥解密所述用户私钥，获取解密后的用户私钥，并将该解密后的用户私钥缓存在内 存中</p>
    <p>[0208] 步骤812，移动终端根据所述用户私钥进行数字签名或解密使用用户公钥加密的 数据；</p>
    <p>[0209] 步骤813，移动终端完成所述数字签名或解密使用用户公钥加密的数据后，清除所 述用户私钥和所述动态密钥。</p>
    <p>[0210] 综上所述，网络设备接收移动终端发送的申请数字证书的请求信息，为该用户生 成相应的用户公钥和用户私钥，根据请求信息中的用户标识及初始计数器数值生成动态密 钥，并利用该动态密钥加密所述用户私钥，生成加密后的用户私钥；移动终端需要使用用户 密钥时，向网络侧发送用户密钥获取请求，根据所述用户标识从所述网络侧查询获取利用 动态密钥加密后的用户私钥，并将该加密后的用户私钥缓存在移动终端内存中，，并利用该 生成的动态密钥解密所述用户私钥，获取解密后的用户私钥，并将该解密后的用户私钥缓 存在内存中，可是实现移动终端未使用用户私钥时，由网络设备保存用户私钥，在移动终端 需要使用用户私钥时，仅在内存中缓存用户私钥，避免用户私钥以文件存储方式存储在移动终端内，从而有效可以保障用户私钥的安全性，降低用户私钥的丢失或被窃取的安全风 险。</p>
