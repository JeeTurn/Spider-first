    <p>一种实现快速业务聚合的方法及通用业务平台</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及软件开发领域，尤其涉及一种实现快速业务聚合的方法及通用业务平台。</p>
    <p>背景技术</p>
    <p>[0002]	随着信息技术的高速发展，二十一世纪已经成为信息的时代，信息的瞬息万变促使生活、经济、社会建设也随之迅速变化。在这个与时间赛跑的时代，企业的IT化程度，以及如何能对信息的变化做出迅速、敏捷、准确的响应已经成为很多企业得以生存和发展的关键所在。</p>
    <p>[0003]	传统的企业运作方式，一般是由业务人员与客户进行沟通得到业务需求，再将业务需求传达给企业IT人员，IT人员则根据对需求的理解和分析，将其转化为可执行的程序。最后，企业IT人员再将解决方案交给业务人员，经业务人员审核后，推出给最终客户。 在这样的企业运作流程中，除去IT人员软件开发的过程，客户和业务人员的沟通，业务人员和IT人员的沟通成为制约企业开发效率的关键。谁能在同样的市场需求下在最短的时间内做出最快速正确的反应，谁就是赢家。</p>
    <p>[0004]	企业面对快速的业务变化，如何能充分利用已有的各种IT资源及服务，快速、简捷进行业务的开发成为企业迫切需要解决的问题。如果有一种平台能为业务人员所应用， 并提供快速简单的方法创建场景应用，问题便能得到很好的解决。</p>
    <p>发明内容</p>
    <p>[0005]	有鉴于此，本发明的主要目的在于提供一种实现快速业务聚合的方法及通用业务平台，能够实现数据资源的快速聚合。</p>

    <p>[0053]	Web服务部署模块132，用于根据需求对解析后的数据业务进行服务化处理，将数据业务发布成SOAP形式或REST形式的Web服务。</p>

    <p>[0059]	基于上述通用业务平台，本发明还提供一种实现快速业务聚合的方法，图2是本发明实现快速业务聚合的方法的流程示意图，如图2所示，该方法包括以下步骤：</p>
    <p>[0060]	步骤201，资源接入单元接收不同类型的数据资源，并对数据资源进行适配处理；</p>
    <p>[0061]	具体的，资源接入单元与各种不同类型的本地文件、和/或不同厂商的关系数据库、和 /或简单对象访问协议（SOAP，Simple Object Access Protocol)形式的Web Service 资源、和/或表述性状态转移（REST，Representational State Transfer)形式的Web Service资源等数据源连接，从数据源接收各种不同类型的数据资源；其中，本地文件包括 xml文件、excel文件、txt文件等，关系数据库包括MySql数据库、Oracle数据库、DB2数据库等；资源接入单元中包括各种资源适配器，这些资源适配器用于对数据资源进行适配处理；资源接入单元包括以下资源适配器：REST-WS(Itepresentational State Transfer-web service)资源适配器，通过可交互的用户界面设置REST形式web服务（Web service)的 url (统一资源定位符，Uniform Resource Locator)，连接 REST 形式的 Webservice 数据资源；和/或SOAP-WS资源适配器，通过可交互的用户界面设置SOAP形式Wfeb service的url 和端点（endpoint)，连接SOAP形式的W^eb service数据资源；和/或Excel格式的本地文件资源适配器，通过可交互的用户界面设置Excel文件的基本格式，连接Excel文件资源； 和/或MySql数据库资源适配器，通过可交互的用户界面设置要连接的MySql数据库的相应字段，连接MySql数据库资源；和/或Oracle数据库资源适配器，通过可交互的用户界面设置要连接的Oracle数据库的相应字段，连接Oracle数据库资源；和/或FTP资源适配器，通过可交互的用户界面设置FTP资源的服务器信息及用户验证信息，连接FTP资源；和 /或HTTP资源适配器，通过可交互的用户界面设置HTTP资源的url，连接HTTP资源；和/ 或简易信息聚合（RSS，Really Simple Syndication)资源适配器，通过可交互的用户界面设置要订阅的RSS资源的url，连接RSS资源；资源接入单元将适配处理后的数据资源提供给业务生成单元进行调用。</p>
    <p>[0062]	步骤202，业务生成单元利用聚合操作符对经过适配处理的数据资源进行聚合操作，构建出新的数据业务；</p>
    <p>[0063]	具体的，业务生成单元与资源接入单元连接，可以调用资源接入单元提供的各种数据资源；业务生成单元利用轻量级的数据聚合语言以及数据转换、过滤、连接、混合等聚合操作符，对资源接入单元提供的经过适配处理的各种数据资源进行聚合操作，最后构建出一个新的综合化的数据业务；这里，业务生成单元还可以提供可视化用户编辑界面，能够接收用户利用统一的XML语言描述多来源的数据对象，接收用户利用可视化拖拽或编辑脚本语言的方式进行的操作。</p>
    <p>[0064]	步骤203，业务执行引擎单元对新的数据业务进行解析和执行；</p>
    <p>[0065]	具体的，业务执行引擎单元能够提供运行新的数据业务的环境，能够提供运行的支持；业务执行引擎单元中，解析执行模块对业务生成单元生成的新的数据业务进行解析，并根据解析出的数据执行该数据业务；此外，Web服务部署模块根据需求对解析后的数据业务进行服务化处理，将数据业务发布成SOAP形式或REST形式的Web服务；资源适配器接口模块可以将从资源接入单元获取到的数据资源，转化为特定格式后存储在临时数据库中，资源管理模块对该临时数据库和所述Web服务进行管理，主要包括数据库中数据资源的更新以及Web服务的启动和停止等Java业务集成（JBI)接口模块可以将业务执行引擎单元变成JBI接口规范的服务引擎，能够集成到符合JBI规范的企业服务总线（ESB， Enterprise Service Bus)产品中。</p>
    <p>[0066]	图3是本发明资源接入单元中资源适配器的加载方法的流程示意图，如图3所示， 该方法包括以下步骤：</p>
    <p>[0067]	步骤301，通过Java插件框架（JPF，Java Plug-in Framework)进入程序入口类， 程序入口类按照JPF规则实现程序初始化工作，读取所有资源适配器的配置文件。</p>
    <p>[0068]	步骤302，程序初始化。</p>
    <p>[0069]	步骤303，程序入口类根据读取的资源适配器的配置文件加载所有的资源适配器。</p>
    <p>[0070]	步骤304，初始化临时数据库，临时数据库用于存储数据转化过程中生成的中间临时数据。</p>

    <p>[0157]	本发明实施例中，轻量级的数据聚合语言遵循XML规范，提出一个标准的基于XML khema的文档结构描述，从命名空间、时间元素、数据资源和聚合描述符、连接线、输入接口五方面对数据资源进行描述，对聚合描述符进行定义，能够提供完整的通过脚本描述语言生成新的数据业务的方式，具体的：</p>
    <p>[0158]	命名空间（Name Space Group)：是指不同的元素或属性的定义空间。该语言中用命名空间的接口规范了整个文档的五个元素的命名空间http ： //www. bupt. edu. cn/， 并为其他元素的子元素或属性的定义提供了定义命名空间的接口，如ftp的目录元素 directory 定义在 http ： //www. bupt. edu. cn/service/ftp,文件的目录兀素 directory 定义在 http://www. bupt. edu. cn/service/file ；</p>
    <p>[0159]	时间元素（Date And Time Group)：用于定义工程创建时间以及最近修改时间的描述方式，是对数据源的数据资源进行提取，并保存到数据库中，根据时间信息进行增删改的重要聚合描述符；</p>
    <p>[0160]	数据资源和聚合描述符（Node Group)：提供了描述数据资源和聚合描述符的共同元素，同时也对其在命名空间内的自描述信息提供了接口 ；数据资源和聚合描述符的共同信息包括业务生成UI模块中显示时的位置信息，在通用数据库中存储时的输入数据资源和输出数据资源；</p>
    <p>[0161]	连接线（Arrow Group)：不同数据资源和聚合描述符之间通过连接线来确定其输入与输出关系，从而达到数据的聚合；</p>
    <p>[0162]	输入接口（Project Data Group)：对数据资源的输入配置进行描述；文档以接口的方式将其独立出来，方便修改和重用。</p>

    <p>[0178]	实施例</p>
    <p>[0179]	以煤矿企业为例，实现一个定位有害气体浓度超过安全界限地点并在煤矿地图显示的原型系统，该系统的底层模拟数据源来自真实的煤矿企业的瓦斯，一氧化碳等传感器为例，具体过程包括以下步骤：</p>
    <p>[0180]	数据资源接入及数据资源聚合过程：</p>
    <p>[0181]	步骤1，每个工作面的传感器接入同一网关中，在网关中，创建REST服务，通过不同URL表示不同传感器，向外部提供访问其中的传感器信息的restful接口，传感器的地理位置信息存储在本地数据库中，通过传感器的ID，能获得传感器的地理位置信息。</p>
    <p>[0182]	步骤2，在可视化编辑区域内，放置“REST资源”节点，通过用户交互配置REST资源适配器，输入传感器节点URL，设置数据交换格式JS0N/XML ；REST资源适配器发送请求， 获取指定传感器节点数据资源并进行解析后，提供数据属性（“传感器ID”、“传感器类型”、 “传感器浓度”、“时间”）供选择，本实施例中，选择“传感器ID” “传感器类型” “传感器浓度”三个属性后，完成REST资源适配器的配置；按照上述方法，可以配置多个区域的对应的“REST资源”节点。</p>
    <p>[0183]	步骤3，在可视化编辑区域内，放置“有序合并的聚合操作符”节点。设置“有序合并的聚合操作符”节点，按照传感器类型进行有序合并。将所有的REST资源节点输出连接至“有序合并的聚合操作符”节点的输入，REST资源节点通过适配器获取数据资源，并存储到临时数据库中；“有序合并的聚合操作符”节点利用对应SQL操作临时表，将所有的数据资源有序合并于该节点中。</p>
    <p>[0184]	步骤4，在可视化编辑区域内，放置“筛选符”节点，连接“有序合并的聚合操作符” 节点的输出；设置筛选条件：瓦斯浓度数据大于0. 保留，去除小于0. 的值，以及一氧化碳浓度数据大于0. 2%保留，去除小于0. 2%的值；筛选符节点自动生成SELECT SQL语句，并处理临时数据库的数据资源。</p>
    <p>[0185]	步骤5，在可视化编辑区域内，放置“数据库”节点，配置数据库的URL、用户名密码、选择传感器的地点信息表，确认后，数据库节点发送请求，获取指定传感器节点的数据资源并进行解析，选择“传感器ID”和“地点”属性。</p>
    <p>[0186]	步骤6，在可视化编辑区域内，放置“聚合”节点，将上述“数据库”节点和“筛选符” 节点的输出连接至“聚合”节点输入；设置“聚合”条件，“数据库”节点的传感器ID等于“筛选符”节点的传感器ID，该节点根据设置条件，调用对应的JOIN SQL语句，在临时数据库中保存属性是“传感器ID” “传感器类型”、“传感器浓度”、“地点”的数据信息。</p>
    <p>[0187]	步骤7，保存脚本，运行脚本，用户可以预览得到实时的瓦斯浓度数据大于0. 1% 或一氧化碳浓度数据大于0. 2%的传感器信息及其地理位置信息。</p>
    <p>[0188]	数据资源的服务化过程：</p>
    <p>[0189]	步骤1，定义REST访问接口 ：REST访问接口的所有操作定义一个基本地址，REST 访问接口对应总共五个操作：getALL操作是得到用户的服务列表，访问方式为HTTP GET, 返回参数为服务名列表；getByServiceName操作是根据服务名运行服务操作，访问URI为基本地址+参数service name，访问方式为HTTP GET，返回参数以JSON格式返回服务运行结果和运行服务的状态；addService操作是注册服务，访问URI也是基本地址+参数 service name，访问方式为HTTP POST方式，返回参数是增加服务操作返回的结果状态信息；updateService操作是更新服务，访问URI也是基本地址+参数service name，访问方式为HTTP PUT方式，返回的参数为更新服务结果状态信息；deleteService操作是删除服务，访问URI也是基本地址+参数service name，访问方式为HTTP DELETE，返回参数为删除服务是否成功标志；这里，REST访问接口各操作返回参数根据操作结果不同。</p>
    <p>[0190]	步骤2，定义SOAP访问接口 ：S0AP方式接口操作与REST接口操作含义一样，参数也相同；区别是SOAP不能设置操作访问的路径和访问方式，SOAP请求都是以HTTP POST方式访问，在请求中要注明访问的操作，并且其中SOAP的所有操作都以XML格式返回消息。</p>
    <p>[0191]	步骤3，利用上述定义的基本地址+adc^ervice+servicename注册上述煤矿数据聚合脚本为rest和soap服务。</p>
