    <div class="abstract">本发明公开了一种网页内容下载方法，包括：设置目标下载内容和下载模式；根据所述设置的目标下载内容获取下载列表；满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。本发明还相应地公开了一种网页内容下载系统。本发明通过提前下载用户设定内容，给用户浏览阅读体验带来流畅感觉，而不受限于当前的网络环境，从而能够提高用户体验，并且，本发明可以提供不同模式的下载方式，满足不同需求的用户，从而进一步提高用户体验。</div>

    <p>一种网页内容下载方法及系统</p>
    <p>技术领域</p>
    <p>[0001 ] 本发明涉及数据下载技术，尤其涉及一种网页内容下载方法及系统。</p>
    <p>背景技术</p>
    <p>[0002]	随着无线网络通信技术的不断发展，越来越多的移动终端用户开始通过WIFI网络、GPRS网络、3G网络浏览网页内容，如新闻、网络电子书等，但是，由于移动终端活动性较强，在移动网络环境不好时，手机浏览体验很槽糕，例如，一些上班族每天上班时会在地铁上用手机看新闻，但由于地铁内网络信号不好，经常会出现不能打开或者很慢才能打开的情况；另外，在没有网络环境时，用户无法进行手机阅读，例如，一些iTouch只能在有WIFI的环境下（例如在家，在图书馆，咖啡厅等）才能上网，一旦切换到其他没有WIFI地点时，就无法继续浏览网页。 </p>
    <p>[0003]	总之，由于公交车、地铁等环境下移动网络信号不稳定，在这些场景下通过移动终端打开新的链接时，很容易因为断网、网速慢等原因无法及时获取网页内容，从而使用户处于等待状态，影响用户体验。</p>
    <p>发明内容</p>
    <p>[0004]	有鉴于此，本发明的主要目的在于提供一种网页内容下载方法及系统，能够灵活下载网页内容，提高用户体验。</p>
    <p>[0005]	为达到上述目的，本发明的技术方案是这样实现的：</p>
    <p>[0006]	一种网页内容下载方法，包括：</p>
    <p>[0007]	设置目标下载内容和下载模式；</p>
    <p>[0008]	根据所述设置的目标下载内容获取下载列表；</p>
    <p>[0009]	满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。</p>
    <p>[0010]	所述设置的目标下载内容为以下一种或多种：站点、频道、链接；</p>
    <p>[0011]	所述设置的下载模式包括以下一种或多种：WIFI模式、定时模式、用户触发模式。</p>
    <p>[0012]	该方法还包括：设置最大迭代次数，</p>
    <p>[0013]	所述根据设置的目标下载内容获取下载列表为：</p>
    <p>[0014]	进行多次迭代，将设置的目标下载内容所对应的网页链接，以及所述设置的目标下载内容所对应的网页链接的多级子级链接地址添加到下载列表中，直到迭代次数不小于最大迭代次数，获得最终的下载列表。</p>
    <p>[0015]	该方法还包括：</p>
    <p>[0016]	用户请求下载列表中的链接地址对应的网页内容时，所述网页内容已经下载完成，则直接向用户呈现所述网页内容；所述网页内容正在下载，则下载完成后向用户呈现所述网页内容；所述网页内容还没有下载，则暂停下载下载列表中其他链接地址的网页内容，下载所述网页内容，下载完成并向用户呈现所述网页内容后，继续下载下载列表中其他链接地址的网页内容，[0017]	用户请求非下载列表中的链接地址对应的网页内容时，暂停下载列表中链接地址的下载任务，下载用户请求的网页内容，下载完成并向用户呈现内容后，继续下载下载列表中链接地址的网页内容。</p>
    <p>[0018]	该方法还包括：当下载网页内容的流量不小于预先设置的最大下载流量时，停止下载。</p>
    <p>[0019]	一种网页内容下载系统，包括：用户设置模块和下载模块；其中，</p>
    <p>[0020]	所述用户设置模块，用于根据用户操作设置目标下载内容和下载模式；</p>
    <p>[0021 ] 所述下载模块，用于根据所述在用户设置模块设置的目标下载内容获取下载列表；以及在满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。</p>
    <p>[0022]	所述在用户设置模块设置的目标下载内容为以下一种或多种：站点、频道、链接； </p>
    <p>[0023]	所述在用户设置模块设置的下载模式包括以下一种或多种：WIFI模式、定时模式、用户触发模式。</p>
    <p>[0024]	所述下载模块根据设置的目标下载内容获取下载列表为：</p>
    <p>[0025]	进行多次迭代，将设置的目标下载内容所对应的网页链接，以及所述设置的目标下载内容所对应的网页链接的多级子级链接地址添加到下载列表中，直到迭代次数不小于最大迭代次数，获得最终的下载列表。</p>
    <p>[0026]	所述下载模块，还用于在用户所请求链接地址的网页内容没有下载时，暂停下载下载列表中链接地址的网页内容，下载用户请求的网页内容，下载完成后，继续下载下载列表中链接地址的网页内容。</p>
    <p>[0027]	该系统还包括内容呈现模块，用于根据用户需求向用户呈现下载模块下载的网页内容。</p>
    <p>[0028]	本发明网页内容下载方法及系统，设置目标下载内容和下载模式；根据所述设置的目标下载内容获取下载列表；满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。本发明通过提前下载用户设定内容，给用户浏览阅读体验带来流畅感觉，而不受限于当前的网络环境，从而能够提高用户体验，并且，本发明可以提供不同模式的下载方式，满足不同需求的用户，从而进一步提高用户体验。</p>

    <p>[0031]	图3为本发明定时模式的下载流程示意图；</p>
    <p>[0032]	图4为本发明用户触发模式的下载流程示意图；</p>

    <p>具体实施方式</p>
    <p>[0035]	本发明的基本思想是：设置目标下载内容和下载模式；根据所述设置的目标下载内容获取下载列表；满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。</p>
    <p>[0036]	图I为本发明网页内容下载方法流程示意图，如图I所示，该方法包括：[0037]	步骤101 :设置目标下载内容和下载模式。</p>
    <p>[0038]	步骤102 :根据所述设置的目标下载内容获取下载列表。</p>
    <p>[0039]	步骤103 :满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。</p>
    <p>[0040]	步骤104 :向用户需求呈现下载的网页内容。</p>
    <p>[0041]	步骤101中，例如可以设置目标下载内容为腾讯网、新浪网、凤凰网等站点，这些站点的内容会在不同下载模式下完成后台下载。当然，用户也可以设置目标下载内容为站点的某些频道和/或链接，例如选择手机腾讯网中的某几个新闻链接，然后再下载，这样，只需要下载用户指定的新闻链接，下载量相对会少一些。</p>
    <p>[0042]	步骤101中，例如可以设置下载模式为以下一种或多种：WIFI模式、定时模式、用户触发模式。 </p>
    <p>[0043]其中：</p>
    <p>[0044]	WIFI模式的下载流程如图2所示，该模式下，移动终端处于WIFI连接状态时，自动检测用户设置的目标下载内容是否已经完成，如果完成则不再下载，如果还未开始下载或者有未完成的下载列表任务，则继续下载；</p>
    <p>[0045]	定时模式的下载流程如图3所示，该模式下，在到达指定的时间时（例如早上8点），移动终端会自动更新用户设置的目标下载内容，此时如果有WIFI会连接WIFI，如果没有WIFI则会使用GRPS或3G网等来下载；</p>
    <p>[0046]	用户触发模式的下载流程如图4所示，该模式下，移动终端提供触发入口给用户，只要用户触发了这个下载模式，就会记录用户选择的链接地址并开始在后台下载。</p>
    <p>[0047]	步骤102具体可以为：进行多次迭代，将设置的目标下载内容所对应的网页链接，以及所述设置的目标下载内容所对应的网页链接的多级子级链接地址添加到下载列表中，直到迭代次数不小于最大迭代次数，获得最终的下载列表。具体的，除了将步骤101中设置的站点、频道和/或链接所对应的所有网页链接地址存储到下载列表，还需要对这些网页链接地址进行迭代，将更多相关的网页链接地址存储到下载列表。以设置下载某站点内容为例，根据用户设置的要下载的网址，存储该网址的所有网页链接地址（记为LinkSet_0)到下载列表（例如要下载的链接地址是手机腾讯网3g. qq. com, LinkSetA_0的内容就是3Gqq.com上的所有链接地址）；然后，遍历解析LinkSet_0里每个网页链接的内容，获取每个链接里存在的二级链接地址，如果二级链接地址标签是下页、下一页、下图、下一图、余下全文等，则将该二级链接地址存储到下载列表中，这些链接地址记为LinkSubSet_l ;再重复遍历解析LinkSubSet_l的网页内容，判断每个链接里的存在的子链接地址，对于标签是下页、下一页、下图、下一图、余下全文等的子链接地址，将该子链接地址存储到下载列表</p>
    <p>中，这些子链接地址记为LinkSubSet_2......;重复迭代，直到LinkSubSet_N的列表为空，</p>
    <p>或者迭代次数超过预先设置的最大迭代次数MAXTimes (例如5次，10次等)，则迭代终止。这样，移动终端除了会下载用户设定的站点所有链接的内容，还会下载链接内容中还有下页、下一页等的内容，给用户带来更好的新闻阅读体验。</p>
    <p>[0048]	例如，链接地址和标签在HTML的表现形式为：</p>
    <p>[0049]	&lt;ahref = " /p ? i_url = http % 3A % 2F% 2Finfo. 3g. qq. com% 2Fg % 2Fs %3Faid % 3Dtemplate % 26tid % 3Dsports_20llsjs % 26icfa % 3Dhome&#8212;act % 26s&#8212;it %3D1% 26sid% 3DAaEBXTslCysaLjGlcEJf jvl&#8212;" &gt; 世锦赛-丁俊晖 12 平&#183; 21:30 决战 &lt;/a&gt;[0050] 则表示链接地址是i_url后面的字符串，标签是世锦赛-丁俊晖12平.21:30决战。</p>
    <p>[0051 ] 再例如，链接地址和标签在HTML的表现形式为：</p>
    <p>[0052]	&lt;ahref = " /p ? i_url = http % 3A % 2F % 2Finfo50. 3g. qq. com % 2Fg %2Fs % 3Fsid % 3DAaEBXTslCysaLjGlcEJfjvl_ % 26aid % 3Dsports_ss % 26id %3Dsports_20110430000456 % 26pos % 3Dhome_C % 26icfa % 3Dhome_abcd % 26s_it %3D I " &gt;9球北京赛-潘晓婷7-9爆冷出局&lt;/a&gt;</p>
    <p>[0053]	则表示链接地址是i_url后面的字符串，标签是9球北京赛-潘晓婷7_9爆冷出局。 [0054]	需要说明的是，根据下载列表下载网页内容的过程可以是连续的，也可以是非连续的，换言之，终端可以一次性下载所有需要下载的网页内容，也可以分多次下载需要下载的网页内容。</p>
    <p>[0055]	另外，用户还可以进一步设置最大下载流量，当下载网页内容的流量不小于所述最大下载流量时，停止下载，从而可以避免下载费用超出用户预期。</p>
    <p>[0056]	图5为本发明实施例一种获取下载列表的详细流程示意图，如图5所示，该流程包括：</p>
    <p>[0057]	步骤501 :遍历下载站点中的所有链接地址，存储为LS0。</p>
    <p>[0058]	步骤502 :将LSO的所有链接地址添加到下载列表LS (LinkSet)。</p>
    <p>[0059]	步骤503 :遍历LSO中每个链接地址里所有存在标签是下页、下一页、下图、下一图、余下全文等的子级链接地址，存储为SUBLS。</p>
    <p>[0060]	步骤504 :判断SUBLS是否为空以及迭代次数是否小于预先设置的最大迭代次数，如果SUBLS不为空且迭代次数小于最大迭代次数，则继续进行迭代，LSO = SUBLS，返回步骤502 ；SUBLS为空或者迭代次数不小于最大迭代次数，则流程结束，当前LS中的所有链接地址内容即为需要下载的内容。</p>
    <p>[0061]	步骤103中，一般通过后台下载的方式下载网页内容，在WIFI模式下，只要有WIFI连接上的网络环境，后台下载就会启动，下载LinkSet列表中链接地址的内容直至下载完成；在定时模式下，时间点一到，后台下载就会启动，下载LinkSet列表中链接地址的内容直至下载完成，此模式会优先选择WIFI环境下载；在用户触发模式下，用户触发后，后台下载就会启动，下载LinkSet列表中链接地址的内容直至下载完成。</p>
    <p>[0062]	需要说明的是，用户在阅读网页时，如果点击了在LinkSet列表中的链接地址，比如link_l，移动终端会先判断link_l的网页内容是否已经被下载完成，如果已经完成，则马上呈现这个已经下载到本地上的网页内容；如果正在下载中，则继续下载，完成后呈现；如果还没下载，则暂停LinkSet中其他链接地址的网页内容下载任务（如果有），启动link_l的下载，下载完成后呈现内容。用户点击非LinkSet列表中的链接地址时，移动终端暂停LinkSet中链接的下载任务，启动用户所点击链接地址的网页内容的下载，下载完成并呈现网页内容后，继续LinkSet中链接地址的网页内容的下载任务。</p>
    <p>[0063]	本发明还相应地提出一种网页内容下载系统，图6为本发明网页内容下载系统的结构示意图，如图6所示，该网页内容下载系统包括：用户设置模块和下载模块；其中，</p>
    <p>[0064]	所述用户设置模块，用于根据用户操作设置目标下载内容和下载模式；[0065]	所述下载模块，用于根据所述在用户设置模块设置的目标下载内容获取下载列表；以及在满足所设置的下载模式的条件时，根据所述下载列表下载网页内容。</p>
    <p>[0066]	所述在用户设置模块设置的目标下载内容为以下一种或多种：站点、频道、链接；</p>
    <p>[0067]	所述在用户设置模块设置的下载模式包括以下一种或多种：WIFI模式、定时模式、用户触发模式。</p>
    <p>[0068]	所述下载模块根据设置的目标下载内容获取下载列表为：</p>
    <p>[0069]	进行多次迭代，将设置的目标下载内容所对应的网页链接，以及所述设置的目标下载内容所对应的网页链接的多级子级链接地址添加到下载列表中，直到迭代次数不小于最大迭代次数，获得最终的下载列表。</p>
    <p>[0070]	所述下载模块，还用于在用户所请求链接地址的网页内容没有下载时，暂停下载 下载列表中链接地址的网页内容，下载用户请求的网页内容，下载完成后，继续下载下载列表中链接地址的网页内容。</p>
    <p>[0071]	如图6所示，该系统还包括内容呈现模块，用于根据用户需求向用户呈现下载模块下载的网页内容。</p>
    <p>[0072]	需要说明的是，本发明可以应用于手机浏览、新闻阅读APP、微博类产品、手机邮箱等场景。本发明根据用户的设定在网络环境好的时候提前下载用户设定内容，下载完成后用户在任何网络环境或者没有网络环境下都可以流畅浏览，从而给用户浏览阅读体验带来流畅感觉，而不受限于当前的网络环境，并且，本发明能够提供不同模式的下载方式，满足不同需求的用户。</p>
