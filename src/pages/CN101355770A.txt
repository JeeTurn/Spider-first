    <div class="abstract">本发明公开了一种虚拟扩展用户识别模块SIM卡容量的方法及系统，移动电话将存储卡的唯一身份信息发送至存储卡应用服务器；存储卡应用服务器确定接收的存储卡的唯一身份信息与自身预先存储的存储卡信息一致时，将该存储卡的唯一身份信息通过移动电话发送至SIM卡；SIM卡如果确定自身存储的SIM卡绑定列表信息中不包含该存储卡的唯一身份信息，通过移动电话以及OTA服务器向存储卡应用服务器发送绑定请求信息；存储卡应用服务器接收OTA服务器发送的绑定请求信息，将存储卡存储容量虚拟为SIM卡扩展存储容量。应用本发明，降低用户使用运营商多媒体服务应用的成本，同时方便运营商快速高效部署推广增值业务、并且可保证下载的应用的存储容量以及通信速率。</div>

    <p>一种虚拟扩展用户识别模块卡容量的方法及系统</p>
    <p>技术领域</p>
    <p>本发明涉及存储器技术，特别涉及一种虚拟扩展用户识别模块（SIM， Subscriber Identification Module )卡容量的方法及系统。</p>
    <p>背景技术</p>
    <p>SIM卡在移动通信中，最初主要是作为个人身份认证与识别信息的载 体，也附带存储一些个人资料，如电话本、短消息等， 一般与移动电话结合 使用，实现移动通信。</p>
    <p>随着SIM卡技术以及电子商务的发展，将电子商务和SIM卡技术相结 合，扩展用户移动通信的业务，实现多媒体服务应用，是移动通信发展的趋 势。例如，通过移动电话上网浏览信息、发送邮件、下单购物、下载音乐、 电影、在线观看视频。既提高了用户个性化和多样化的需求，也有利于运营 商增值业务快速推广。</p>
    <p>现有SIM卡容量一般较小、通信速率较低，而上述多媒体服务需要较 大的存储容量以及较高的通信速率，因而，针对现有小容量SIM卡以及低 通信速率的问题，依据现有技术提出了一种改进SIM卡的方案，即采用大 容量SIM卡。目前，改进后的大容量SIM卡的存储容量可以是传统SIM卡 的存储容量的一千多倍，并通过通用串行总线（USB， Universal Serial Bus ) ll接口或多媒体卡（MMC, MultiMediaCard)接口与移动电话建立高速通 信通道进行通信。大容量SIM卡可以支持多种新型应用，并可以部署运营 商提供的有针对性的定制和个性化服务，满足人们对多媒体业务的需求；同 时，由于提高了存储容量和通信速率，因而，还可以在该SIM卡上集成先 进的安全功能，例如，非对称加密功能、以及防篡改功能，对下载的应用进</p>
    <p>行安全加密，实现安全通信。</p>
    <p>图1为现有技术中大容量SIM卡执行应用下载的系统结构示意图，参</p>
    <p>见图1，该系统包括：大容量SIM卡、移动电话、以及空中接口 （OTA， Over The Air)服务器,其中，</p>
    <p>大容量SIM卡，通过USBll接口或MMC接口与移动电话建立高速通 信通道，用于对接入的用户进行认证以及对下载的应用进行管理和保护，预 先存储有SIM卡应用工具包（STK， SIM Tool Kit)应用列表信息，即绑定 列表信息，STK应用列表对应用户定制的各类应用信息，</p>
    <p>根据i人证后用户选择的STK应用列表的应用，向移动电话发送下载应 用请求，接收移动电话发送的应用文件，进行存储，并对存储的应用和多媒 体文件进行管理和保护；</p>
    <p>移动电话，用于接收大容量SIM卡发送的下载应用请求，通过空中接 口发送至OTA服务器；</p>
    <p>OTA服务器，用于接收移动电话发送的下栽应用请求，将下载应用请 求对应的应用文件发送至移动电话。</p>
    <p>实际应用中，STK应用列表中存储的STK应用遵循欧洲电信标准学会 (ETSI, European Telecommunications Standards Institute )通信标准(TS， Telecommunications Standards) 102 223协i义头见范，即全5求移动通信系统 (GSM 11.14, Global System for Mobile communications 11.14)的十办i义^L范， 由运营商预置并可进行更新。STK应用列表信息可以通过移动电话以菜单的 形式显示给用户。</p>
    <p>运营商将STK应用存储到SIM卡上并更新具体包括：首先，SIM卡商 根据运营商要求，在对SIM卡进行个人化操作的时候将STK应用预置在SIM 卡上；其次，对于已预置在SIM卡上的STK应用，如果运营商更新了OTA 服务器上的STK应用，则OTA服务器发送远程文件管理（RFM, Remote File Management)，将运营商更新的STK应用下载到SIM卡上并更新，以 &lt;更于</p>
    <p>但是在现有改进的大容量SIM卡方案中，对SIM卡本身作了巨大的改 动，特别是改变了 SIM卡操作系统和接口，增加了 SIM结构的复杂性；而 且，为了能够兼容大容量SIM卡，移动电话也需要作相应改进，这样，如 果用户需要应用多媒体服务，则首先需要选择购买大容量SIM卡以更换现 有的SIM卡，同时，还需要购买支持移动电话和SIM卡之间高速通信接口 的移动电话以更换现有的移动电话，增加了用户应用多4某体服务应用的成 本；而如果采用现有小容量SIM卡，由于其存储容量小、通信速率低，限 制了用户下载速率以及对下载的应用的存储，因而，上述的两种技术方案都 对用户的使用造成不便，不利于多媒体服务应用以及运营商增值业务的开 展。</p>
    <p>发明内容</p>
    <p>有鉴于此，本发明的一个主要目的在于提供一种虚拟扩展用户识别模块 卡容量的系统，降低用户应用多媒体服务应用的成本、且可保证下载的应用 的存储容量以及通信速率。</p>
    <p>本发明的另 一个主要目的在于提供一种虚拟扩展用户识别模块卡容量 的方法，降低用户应用多媒体服务应用的成本、且可保证下载的应用的存储 容量以及通信速率。</p>
    <p>为达到上述目的，本发明提供了一种虚拟扩展用户识别模块卡容量的系 统，该系统包括：移动电话、空中接口OTA服务器、SIM卡，该系统还包 括：存储卡和存储卡应用服务器，其中，</p>
    <p>移动电话，用于将插入的存储卡的唯一身份信息发送至存储卡应用服务器， 并接收存储卡应用服务器返回的存储卡的唯一身份信息，发送至SIM卡；接收 来自SIM卡的绑定请求信息，并发送至OTA服务器；</p>
    <p>OTA服务器，用于将接收的绑定请求信息发送至存储卡应用服务器；</p>
    <p>SIM卡，用于接收来自移动电话的存储卡的唯一身份信息，如果确定自身 存储的绑定列表信息中不包含该存储卡的唯一 身份信息，向移动电话发送绑定</p>
    <p>请求信息；</p>
    <p>存储卡应用服务器，用于确定来自移动电话的存储卡的唯一身份信息与自 身预先存储的存储卡信息一致时，将接收到的存储卡的唯一身份信息返回移动</p>
    <p>电话；在接收到OTA服务器发送的绑定请求信息后，绑定存储卡，将存储卡存 储容量虚拟为SIM卡的扩展存储容量。</p>
    <p>所述SIM卡进一步用于根据用户确定的绑定列表信息中的应用，向移动电 话发送启动应用请求信息，相应地，</p>

    <p>存储卡应用服务器，进一步根据接收的启动应用请求信息查询存储的存储 卡应用信息，获取应用下载资源信息，并发送给OTA服务器；</p>
    <p>OTA服务器，进一步用于根据接收的应用下载资源信息，进行下载，并将 下载的应用文件发送至移动电话；</p>
    <p>存储卡，进一步用于接收应用文件并存储。</p>

    <p>本步骤中，存储卡应用服务器接收删除应用请求信息，更新存储的存储</p>
    <p>卡内应用文件管理信息，向OTA服务器发送删除应用命令信息。</p>
    <p>步骤604, OTA服务器接收删除应用命令信息，发送至移动电话； 步骤605,移动电话接收删除应用命令信息，发送至存储卡； 步骤606，存储卡接收删除应用命令信息，删除该应用文件。 由上述实施例可见，本发明提供的一种虚拟扩展用户识别模块卡容量的</p>
    <p>方法及系统，通过将存储卡与SIM卡进行绑定，存储卡通过高速的存储卡 接口与移动电话进行通信，将原应存储在大容量SIM卡上的多种新型应用</p>
    <p>文件通过存储卡接口存储到存储卡上；并利用STK应用以及存储卡应用服 务器对存储卡存储的应用文件进行管理和保护。这样，通过将存储卡存储容 量虚拟为SIM卡存储容量，并通过存储卡接口下载应用文件，实现了现有 大容量SIM卡的功能，又无需改动现有SIM卡操作系统和移动电话结构， 降低了用户应用多媒体服务应用的成本、节约了移动电话设备商生产成本、 且可保证下载的应用的存储容量以及通信速率。实现方便，兼容性好，有利 于运营商业务应用的快速有效推广。</p>
