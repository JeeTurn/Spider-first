    <p>使用打印机和移动设备来打印电子文档的方法和系统</p>
    <p>[0001]	相关申请的交叉引用</p>
    <p>[0002]	本申请包含的主题可能涉及2011年9月30日提交的序列号为13/249,701、题为 “Wireless Printing from a Device Outside the Network”(代理人案号为37682/025001 ;客户号为BIL-1031)且转让给本申请的受让人的美国申请的主题。</p>
    <p>技术领域</p>
    <p>[0003]	本发明涉及打印技术，并且具体地涉及使用打印机和移动设备来打印电子文档的方法和系统。</p>
    <p>背景技术</p>
    <p>[0004]	为了在 传统网络环境中的打印机上打印来自移动设备的电子文档(ED)，附接至该网络中的打印服务器的所有打印机将势必作为可用打印设备显露给想要打印该文档的移动设备。在大的企业环境中，这将显露跨全局的作为对于移动设备的可用打印设备的许多打印机。从安全性的角度而言，该信息以此方式的显露是不实际的。</p>
    <p>发明内容</p>
    <p>[0005]	总体而言，在一个方面中，本发明涉及一种用于使用打印机和移动设备来打印电子文档(ED)的方法。该方法包括:通过打印机接收对于打印的请求；响应于该请求，通过打印机生成唯一标识符(UID)，其中移动设备在第一网络上将与UID和ED相关联的打印作业发送到打印服务器；通过打印机将WD发送到打印服务器；通过打印机从对发送WD作出响应的打印服务器接收打印作业；以及基于打印作业，通过打印机打印ED的硬拷贝。</p>
    <p>[0006]	总体而言,在一个方面中，本发明涉及一种用于使用打印机和移动设备来打印电子文档(ED)的方法。该方法包括:通过移动设备获得由打印机生成的唯一标识符(UID);在获得UID之后，由移动设备从移动设备的用户接收ED的选择；以及通过移动设备在第一网络上将与ED和UID相关联的打印作业发送到打印服务器，其中响应于打印作业与从打印机接收到的UID匹配，打印服务器将打印作业发送到打印机；以及其中打印机基于打印作业来生成ED的硬拷贝。</p>
    <p>[0007]	总体而言，在一个方面中，本发明涉及一种用于使用打印机和移动设备来打印电子文档(ED)的方法。该方法包括:通过打印服务器在第一网络上从移动设备获得与唯一标识符(UID)和ED相关联的打印作业，其中移动设备从打印机获得WD ;通过打印服务器在第二网络上从打印机获得WD，其中打印机响应于对于打印的请求而生成WD ;通过打印服务器将来自打印机的UID与来自移动设备的打印作业进行匹配；以及响应于匹配，通过打印服务器在第二网络上将打印作业发送到打印机，其中打印机基于打印作业来生成ED的硬拷贝。</p>
    <p>[0008]	总体而言，在一个方面中，本发明涉及一种存储用于打印电子文档(ED)的指令的非暂时性计算机可读介质(CRM)。该指令包括用于以下的功能性:使用打印机接收对于打印的请求；响应于该请求，使用打印机生成唯一标识符(UID)，其中移动设备在第一网络上将与UID和ED相关联的打印作业发送到打印服务器；使用打印机将WD发送到打印服务器；使用打印机从对发送WD作出响应的打印服务器接收打印作业；以及基于打印作业，使用打印机打印ED的硬拷贝。</p>
    <p>[0009]	总体而言，在一个方面中，本发明涉及一种存储用于打印电子文档(ED)的指令的非暂时性计算机可读介质(CRM)。该指令包括用于以下的功能性:使用打印服务器在第一网络上从移动设备获得与唯一标识符(UID)和ED相关联的打印作业；使用打印服务器在第二网络上从打印机获得WD，其中移动设备从打印机获得WD，并且其中打印机响应于对于打印的请求而生成nD ;使用打印服务器将来自打印机的UID与来自移动设备的打印作业匹配；以及响应于nD与打印作业匹配，使用打印服务器在第二网络上将打印作业发送到打印机，其中打印机基于打印作业来生成ED的硬拷贝。</p>

    <p>[0029]	在步骤204中，通过打印机生成包括WD的图形码。图形码例如可以为包括WD的一维条码、二维条码(例如，快速响应(QR)码)、数据字形、图像、明文等。</p>
    <p>[0030]	在步骤206中，打印机 在打印机的屏幕上显示图形码。如果打印机没有屏幕，则打印机可以在介质(例如，纸张、透明胶片、微缩胶卷、微缩平片等)上打印图形码。</p>
    <p>[0031]	在步骤208中，打印机将WD发送到打印服务器。在本发明的一个或更多个实施例中，UID标识打印服务器的位置。在本发明的一个或更多个实施例中，打印服务器的位置对于打印机而言是已知的。打印机可以在企业安全网络上将UID发送到打印服务器。WD可以作为对于与UID对应的打印作业的请求的一部分来被发送到打印服务器。步骤206可以在生成UID之后的任意时间处执行。换言之，打印机可以在生成、显示和/或打印图形码之前(即，在步骤204或步骤206之前)将WD发送到打印服务器。</p>
    <p>[0032]	在步骤210中，打印机从打印服务器接收与UID对应的打印作业。打印机可以在企业安全网络上接收打印作业。</p>
    <p>[0033]	在步骤212中，基于打印作业来打印ED的硬拷贝。硬拷贝为ED的一个或更多个物理地打印的页面。可以根据打印作业中的信息打印硬拷贝。例如，打印作业可以指明硬拷贝利用特定字体、大小、页边空白、字符等来打印。</p>
    <p>[0034]	图2B示出根据本发明的一个或更多个实施例的流程图。可以通过系统(100)(例如，移动设备M (102))执行图2B中的处理的一个或更多个步骤来打印ED。图2B所示步骤的顺序在本发明的各实施例中可以不同，并且步骤中的一个或更多个步骤可以并行执行和/或可以是任选的。而且，可以重复图2B中的一个或更多个步骤。相应地，本发明的范围不应认为限于图2B所示步骤的特定布置。</p>
    <p>[0035]	在步骤215中，移动设备发起打印应用的执行。移动设备可以响应于用户对于打印的请求来发起打印应用的执行。在本发明的一个或更多个实施例中，打印应用的执行激活移动设备的扫描仪(例如，相机、条码激光器等)。</p>
    <p>[0036]	在步骤217中，通过移动设备获得唯一标识符(UID)。例如，移动设备可以通过对包括WD且在打印机的屏幕上显示或者通过打印机打印在介质上的图形码进行拍照来获得nD。然后可以(例如通过打印应用)从所得到的图像提取nD。作为另一示例，移动设备可以使用其条码激光器扫描图形码并提取UID。作为又一示例，可以将UID从打印机无线地(例如，经由红外、蓝牙等)传送到移动设备。</p>
    <p>[0037]	在步骤219中，移动设备接收对用于打印的ED的选择。在本发明的一个或更多个实施例中，打印应用包括允许移动设备的用户选择用于打印的ED的一个或更多个GUI。ED可以存储在移动设备上。ED可以存储在对于移动设备而言可访问的远程位置处。</p>
    <p> [0038]	在步骤221中，通过移动设备生成与UID和ED相关联的打印作业。打印作业可以通过在移动设备上执行的打印应用生成。例如，打印作业可以包括标识UID的打印票据。作为另一示例，打印作业可以包括用于打印的ED的内容。作为又一示例，打印作业可以标识存储ED的远程位置。</p>
    <p>[0039]	在步骤223中，移动设备将打印作业发送到打印服务器。在本发明的一个或更多个实施例中，UID标识打印服务器的位置。在本发明的一个或更多个实施例中，打印服务器的位置对于打印应用而言是已知的。打印作业可以经由例如公共蜂窝或无线网络发送到打印服务器。通过加密或其它类似手段，可以确保打印作业在公共网络上传输期间的安全。</p>
    <p>[0040]	图2C示出根据本发明的一个或更多个实施例的流程图。可以通过系统(100)(例如，打印服务器(118))执行图2C中的处理的一个或更多个步骤来打印ED。图2C所示步骤的顺序在本发明的各实施例之间可以不同，并且步骤中的一个或更多个步骤可以并行执行和/或可以是任选的。而且，可以重复图2C中的一个或更多个步骤。相应地，本发明的范围不应被认为限于图2C所示步骤的特定布置。</p>
    <p>[0041]	在步骤S230中，打印服务器从移动设备接收打印作业。打印作业可以在蜂窝网络上接收。而且，打印作业与将被打印的ED和唯一标识符(UID)二者相关联。例如，打印作业可以包括标识WD的打印票据。作为另一示例，打印作业可以包括用于打印的ED的内容。作为另一示例，打印作业可以标识存储ED的远程位置。</p>
    <p>[0042]	在步骤232中，打印服务器从打印机接收WD。UID可以是从打印机接收到的打印请求的一部分。而且，打印服务器可以在企业安全网络(即，与用于将打印作业传送给打印服务器的网络不同的网络)上从打印机接收nD。在本发明的一个或更多个实施例中，步骤232可以在步骤230之前执行。换言之，在打印服务器从移动设备接收打印作业之前，打印服务器可以从打印机接收WD。</p>
    <p>[0043]	在步骤234中，打印服务器将从打印机接收到的UID与从移动设备接收到的对应的打印作业进行匹配。在本发明的一个或更多个实施例中，匹配需要解析从打印机接收到的请求和从移动设备接收到的打印作业以识别和比较nD。本领域技术人员在受益于本详细描述之后将认识到，打印服务器可以存储任意数目的打印作业，并且可能需要解析所有打印作业以便找到与从打印机接收到的UID对应的打印作业。类似地，打印服务器可以存储来自任意数目的打印机的任意数目的打印请求。</p>
    <p>[0044]	在步骤236中，如果必要则打印服务器修改与UID对应的打印作业以便可与打印机支持的打印机制(scheme)兼容,并且将(经修改的)打印作业发送到打印机。例如，如果存储在打印服务器中的打印作业包括使用文档标记语言(例如，OOXML)描述/表示的ED，并且如果请求打印机仅支持PDL打印机制，则打印服务器可以从使用文档标记语言描述/表示的ED生成包括各种打印对象(例如,文本对象、图形对象、图像对象)和控制对象(例如,混合和透明度命令、颜色管理信息)的多个PDL命令/指令。如上面论述的，通过打印服务器接收到的请求可以标识由请求打印机支持的打印机制。备选地，打印服务器可以包括记录由各种打印机支持的打印机制的查找表或目录。</p>
    <p>[0045]	在步骤238中，打印服务器将打印作业发送到打印机用于打印。打印服务器可以在企业安全网络上将打印作业发送到打印机。</p>
    <p>[0046]	图3示出了根据本发明的一个或更多个实施例的示例。如图3所示，存在用户(350)、智能电话(302)、打印机(330)和打印服务器(318)。打印服务器(318)和智能电话(302)通过蜂窝网络(305)可操作地连接。打印机(330)和打印服务器(318)通过企业安全网络(320)可操作地连接。打印机(330)包括IXD屏幕，并且智能电话(302)包括相机。</p>
    <p>[0047]	仍参照图3，打印 机(330)从用户(350)接收对于打印的请求。用户(350)可以通过选择打印机(330)的控制面板上的一个或更多个按钮来执行请求。响应于从用户(350)接收到对于打印的请求，打印机(330)生成并在打印机(330)的IXD屏幕上显示包括唯一标识符(UID)的QR码(303)。UID可以是MFP (330)的MAC地址、随机字符串和时间戳的组合。UID也可以标识打印服务器(318)。然后打印机(330)经由企业安全网络(320MfnD(322)作为对于与WD (322)对应的打印作业的请求的一部分发送到打印服务器(318)。</p>
    <p>[0048]	然后用户(350)在智能电话(302)上发起打印应用。打印应用激活智能电话(302)上的相机，允许用户(350)对显示在打印机(330)的IXD屏幕上的QR码(303)进行拍照。打印应用从通过智能电话(302)的相机捕获的QR码(303)中提取WD。然后，打印应用经由一个或更多个⑶I提示用户(350)将一个或更多个文件与UID (例如，myDocument.docx(306))关联。myDocument.docx (306)可以存储在智能电话(302)上或者存储在可由智能电话(302)访问的远程网络位置处(例如，在可经由因特网访问的贮存库中)。然后智能电话(302)的打印应用生成打印作业(304)并将打印作业(304)提交给打印服务器(318)。打印作业(304)包括文件和具有WD的打印票据，以将打印作业识别到打印服务器(318)。以下为用于打印作业(304)的示例性打印票据:</p>
    <p>[0049]	%-12345XiPJL JOB</p>
    <p>[0050]	@PJL C OMMENT</p>
    <p>[0051]	iPJL SET USERNAME="JSmith"</p>
    <p>[0052]	OPJL SET DATESTAMP="20120119〃</p>
    <p>[0053]	OPJL SET PLANESINUSE=3</p>
    <p>[0054]	iPJL SET J0BNAME="MyDocument.docx〃</p>
    <p>[0055]	iPJL SET PAPER=LETTER</p>
    <p>[0056]	OPJL SET MEDIATYPE=PAPER</p>
    <p>[0057]	OPJL SET HOLD=OFF</p>
    <p>[0058]	OPJL SET TRAYCHAIN=YES</p>
    <p>[0059]	OPJL SET EC 0N0M0DE=0FF[0060]	@PJL SET BITSPERPIXEL=4</p>
    <p>[0061]	@PJL SET COPIES=I</p>
    <p>[0062]	ipjL set tpaymapping=off</p>

    <p>[0073]	打印作业(304)可以通过智能电话(302)经由蜂窝网络(305)提交到打印服务器(318)。具体而言，蜂窝网络(305)是与企业安全网络(320)不同的网络，打印机(330)利用该企业安全网络(320)与打印服务器(318)通信。打印服务器(318)可以驻留于云(399)网络架构中。打印服务器(318)将从打印机(330)接收到的WD (322)与从移动设备接收到的打印作业中的UID进行比较。打印服务器(318)经由企业安全网络(320)将具有匹配UID的打印作业(304)发送到打印机(330)。然后打印机(330)根据打印作业中的指令生成myDocument.docx 的硬拷贝(332)。</p>
    <p>[0074]	本发明的实施例可以具有以下优点中的一个或更多个优点:打印到本地打印机的无线移动设备不需要与打印机在相同的网络上；打印机网络信息不必显露给无线设备用户；打印作业物理地绑定到本地打印机和无线网络设备。此外，如果用户没有物理地出现在打印机处来获得nD，则错误的打印作业不会提交到打印机。最后，用户可以在对移动设备上的移动应用和/或打印机用户界面进行最少输入的情况下快速地提交打印作业。</p>

    <p>[0076]	此外，在本发明的一个或更多个实施例中，前述计算机系统(400)的一个或更多个元件可以位于远程位置处并且通过网络连接到其它元件。此外，本发明的实施例可以在具有多个节点的分布式系统上实施，其中本发明的每个部分(例如，移动设备、用户应用、打印服务器、多功能打印机)可以位 于分布式系统内的不同节点上。在本发明的一个实施例中，节点对应于计算机系统。备选地，节点可以对应于具有关联的物理存储器的处理器。节点可以备选地对应于具有共享存储器和/或资源的处理器或处理器的微核。此外，可以在诸如压缩盘(CD)、磁盘、磁带、存储器或任何其它计算机可读存储设备之类的非暂时性计算机可读存储介质上，临时或永久地存储用于执行本发明实施例的计算机可读程序代码形式的软件指令。</p>
