    <p>一种流量控制方法、系统和设备</p>
    <p>技术领域</p>
    <p>[0001] 本发明涉及无线通信技术领域，特别是涉及一种流量控制方法、系统和设备。</p>
    <p>背景技术</p>
    <p>[0002] 伴随网络技术的成熟以及用户使用习惯的逐渐养成，分组域数据业务在近年来呈 现快速发展的趋势。用户通过手机、数据卡或者上网本等各种终端，可以通过无线网络随时 接入互联网，享受无线上网带来的乐趣。</p>
    <p>[0003] 同时，随着3G(The Third-Generation Communication System,第三代通信系 统）业务的推进，越来越多的固网宽带化的应用出现在了分组域数据网络上，例如流媒体， P2P(Peer-to-Peer，点对点）下载等。这使我们面临越来越多的网络拥塞的问题，例如少量 用户滥用网络大量下载导致核心网和无线网络资源拥塞，降低大部分用户的使用感知；或 者部分"垃圾业务"拥堵网络导致运营商重点发展的"有价值业务"使用感知下降。新的业 务承载需求使得运营商需要越来越关注不同用户的差异化服务以及对用户不同业务服务 质量的控制。</p>
    <p>[0004] 现有分组域流量的控制方法主要是基于3GPP(3rd Generation PartnershipProject，第三代合作伙伴计划）的规范。基于这种方案的控制，主要通过以下 过程：在用户激活PDP (Packet Data Protocol,分组数据协议）使用业务时，SGSN(Serving GPRS Support Node, GPRS服务支持节点）/GGSN(GatewayGPRS Support Node，网关GPRS支 持节点）根据用户在HLR(Home LocationRegister，归属位置寄存器）中的地址文件参数对 该PDP制定MBR(Max Bitrate，最大比特率）和GBR(Guaranteed Bit rate,保证比特率），然 后SGSN和GGSN根据产品的配置和管理进行Diffserv的服务质量映射。BSC (BaseStation Controller，基站控制器）/RNC(Radio Network Controller,无线网络控制器）根据接收到 的PDP响应消息进行无线信道的分配和管理，与基站一起完成无线信道的调度，实现基于 用户的流量控制。</p>
    <p>[0005]     在实现本发明的过程中，发明人发现现有技术至少存在如下问题：</p>
    <p>[0006]     系统主要基于静态的用户HLR QoS(Quality of Service,服务质量）参数定制，</p>
    <p>只能对单用户进行流量的控制；由于系统没有深度识别业务的能力，因此无法结合用户信</p>
    <p>息，例如用户号码、业务类别以及地理位置等信息进行有针对性的流量控制，实现用户的差</p>
    <p>异化服务和不同业务服务质量的控制。</p>

    <p>具体实施方式</p>
    <p>[0059] 下面将结合本发明中的附图，对本发明中的技术方案进行清楚、完整的描述，显 然，所描述的实施例是本发明的一部分实施例，而不是全部的实施例。基于本发明中的实施 例，本领域普通技术人员在没有做出创造性劳动的前提下所获得的所有其他实施例，都属 于本发明保护的范围。</p>
    <p>[0060] 3G移动网络的系统结构如图l所示，其中SGSN和GGSN通过Gn接口传递GTP(GPRS Tunneling Protocol, GPRS隧道协议）消息，GTP协议有两种，即GTP-C(Control Plane, 控制平面）和GTP-U(User Plane，用户平面）。GTP-C是信令协议，主要功能是路径管理、 信道管理、位置管理、移动性管理和移动广播与组播业务（Mobi 1 e Broadcast/Multicast Service，MBMS) 。GTP-U主要功能是负责SGSN和GGSN之间IP PDUs信道的建立与数据传送。 GTP-C消息中的Create PDP Context Request禾口 Create PDPContext Request Response 消息包含有用户的详细信息，其中包括可用于分组域流量控制的控制元素，例如：用户号 码，地理位置和业务类型等。由此可知，可以根据用户的相关信息进行流量控制。本发明提 供了 一种利用用户的相关信息进行流量控制的方法，其中将用户的相关信息转化为对应的 IP (Internet Protocol,网络之间互连的协议）地址，根据IP地址获取匹配的流量控制策 略，用于包括基于地理范围和/或基于用户的流量控制策略。</p>
    <p>[0061] 如图2所示，为本发明的实施例提供的一种流量控制方法，如图2所示，包括以下 步骤：</p>
    <p>[0062]     步骤201 、接收DPI (De印Packet Inspection,深度包检测技术）设备发送的用户</p>

    <p>所述预设的对应关系为预设的流量控制策略与用户的IP地址的对应关系；</p>
    <p>[0064] 步骤203、根据所述匹配的流量控制策略和所述用户的流量统计分析，判断是否需要对所述用户进行流量控制；</p>

    <p>[0067] 为了清晰地阐述本发明的技术方案，首先对系统中各个设备的具体功能说明如 下：</p>
    <p>[0068]     SGSN :</p>
    <p>[0069]     SGSN作为GPRS (General Packet Radio Service,通用分组无线服务技术）/ WCDMA (Wideband Code Division Multiple Access,宽带码分多址）核心网分组域设备重 要组成部分，主要完成分组数据包的路由转发、移动性管理、会话管理、逻辑链路管理、鉴权 和加密、话单产生和输出等功能。 [0070]     GGSN :</p>
    <p>[0071 ] GGSN主要是起网关作用，它可以和多种不同的数据网络连接，如ISDN (Integrated Services Digital Network,综合业务数字网）、PSPDN(PacketSwitched Public Data Network,分组交换公用数据网）和LAN (Local AreaNetwork,局域网）等。GGSN可以把 GSM网中的GPRS分组数据包进行协议转换，从而可以把这些分组数据包传送到远端的 TCP (Transmission ControlProtocol，传输控制协议）/IP网络。 [0072]     GTP信令采集服务器：</p>
    <p>[0073] GPRS隧道协议GTP是用于GSN节点之间的接口协议。GTP协议由GTP控制面和用 户面协议组成。GTP协议控制平面完成移动台MS接入分组网络的隧道控制和管理功能，消 息主要执行建立、修改和删除GSN之间隧道功能，同时还负责GSN节点间的移动性管理、位 置管理、路径管理功能。GTP协议用户平面负责数据在隧道中传输，以及传输隧道的部分维 护功能。</p>
    <p>[0074] GTP信令采集服务器通过侦听SGSN和GGSN之间Gn接口的GTP消息进行信令采集， GTP协议有两种，即GTP-C和GTP-U。 GTP-C是信令协议，主要功能是路径管理、信道管理、位 置管理、移动性管理和移动广播与组播业务。GTP-U主要功能是负责SGSN和GGSN之间IP PDUs信道的建立与数据传送。GTP信令采集服务器主要通过解析GTP-C消息中的Create PDPContext Request禾口 Create PDP Context Request Response消息来获取用户的详细 信息，并提炼为后续进行流量控制的元素，根据3GPP 29060协议，在CreatePDP Context Request消息中包含如表1所示的信息：</p>

    <p>[0080]     DPI设备：</p>
    <p>[0081]     DPI设备通过包深度解析的方式，对流量进行逐包分析，并形成详细的基于IP的</p>
    <p>流量信息报告。DPI设备可根据策略和数据中心下发的策略，对检测出的各种业务流量进行</p>
    <p>带宽限制和流量整形，防止有限的网络带宽被低价值的业务挤占。同时DPI设备还可支持</p>
    <p>黑、白名单策略流控和基于时间调度的流控时变策略。策略和数据中心：</p>
    <p>[0082]     策略和数据中心是跨接GTP信令采集设备和DPI设备的桥梁。他将在分组域中体</p>
    <p>现用户属性的号码、地理位置等信息都对应为一些离散IP地址相组成的群组，策略和数据</p>
    <p>中心通过对群组进行流量统计和策略下发来实现对分组域的灵活的流量控制。策略和数据</p>
    <p>中心会根据DPI设备上送的流量统计信息，计算总体流量，并根据预置的策略来通知DPI设</p>
    <p>备进行相应的流控行为。</p>
    <p>[0083] 由上述可知，通过DPI设备可以对接收的数据进行基于IP地址的流量统计分析(其中用户的IP地址和流量统计分析具有对应关系），并将基于IP地址的流量统计分析发送给策略和数据中心；GTP信令采集服务器解析GTP-C消息中的Create PDP ContextRequest禾P Create PDP Context Request Response消息来获取用户的详细信息，例如：用户号码、地理位置，业务类型以及IP地址，GTP信令采集服务器提炼出可以用于进行流量控制的用户的相关信息，即该用户的IP地址和用户的相关信息具有对应关系，GTP信令采集服务器将该对应关系发送给策略和数据中心；策略和数据中心包括进行分组域流量控制的预设的流量控制策略，该预设的流量控制策略按照控制元素分策略进行控制，该控制元素由用户的相关信息组成，即预设的流量控制策略与用户的相关信息具有对应关系。策略和数据中心根据GTP信令采集服务器发送的用户的IP地址和用户的相关信息的对应关系，以及预设的流量控制策略与用户的相关信息的对应关系可以获得用户的IP地址和预设的流量控制策略的对应关系，从而可以根据DPI设备发送的用户的IP地址获取与之匹配的流量控制策略进行流量控制。</p>
    <p>[0084] 本发明实施例提供一种流量控制方法，流量控制策略为将某分组域小区P2P流量控制到1Mbps，如图4所示，包括以下步骤：</p>
    <p>[0085]     步骤401、在小区A下的用户开始发起分组域数据业务请求。</p>
    <p>[0086] 步骤402、SGSN和GGSN进行Create PDP Context Request和Create PDPContextRequest Response消息的交互，用户完成PDP激活过程。</p>
    <p>[0087] 步骤403、 GTP信令采集服务器通过侦听的方式采集到Create PDPContextRequest禾口 Create PDP Context Request Response消息，并进行解析获取用户的相关消息。</p>
    <p>[0088] 其中用户的相关消息包括：用户号码-用户IP地址-用户所在小区ID等。例如：用户号码为13966669999，用户的IP地址为192. 168. 1. 147，用户所在小区A的ID为222，[0089] 步骤404、 GTP信令采集服务器将用户的相关信息发送给策略和数据中心。[0090] 步骤405、用户数据流通过GGSN的Gi接口 ，经过DPI设备，DPI设备对通过的数据包进行深度包解析，形成基于IP地址的流量统计分析，并实时发送给策略和数据中心。[0091] 其中，基于IP地址的流量统计分析包括用户的IP地址和对应的用户数据流的流量统计分析。</p>
    <p>[0092] 例如：此时DPI设备发送给策略和数据中心的基于IP地址的流量统计分析中的IP地址为192. 168. 1. 147。</p>
    <p>[0093] 步骤406、策略和数据中心根据GTP信令采集服务器发送的用户的相关信息，将策略"某分组域小区P2P流量控制到1Mbps"的需求翻译成所有进入该小区的用户的IP地址的P2P流量控制到1Mbps 。</p>
    <p>[0094] 具体地，策略和数据中心包括进行分组域流量控制的预设的流量控制策略，该预设的流量控制策略按照控制元素分策略进行控制，该控制元素由用户的相关信息组成，即预设的流量控制策略与用户的相关信息具有对应关系。策略和数据中心根据GTP信令采集服务器发送的用户的IP地址和用户的相关信息的对应关系，以及预设的流量控制策略与用户的相关信息的对应关系可以获得用户的IP地址和预设的流量控制策略的对应关系，从而可以根据DPI设备发送的用户的IP地址获取与之匹配的流量控制策略进行流量控制。[0095] 优选地，策略和数据中心将这些用户IP地址自动设置为一个群组。[0096] 例如，策略和数据中心接收的用户的相关信息为：用户号码为13966669999，用户的IP地址为192. 168. 1. 147，用户所在小区A的ID为222，则策略和数据中心判断ID为222的小区A属于该进行分组域P2P流量控制到1Mbps的小区，所以将用户的IP地址为192. 168. 1. 147设置为进行P2P流量控制策略的IP地址，并将符合相同条件的IP地址放入一个群组，如群组A。此时，对于IP地址属于该群组A的用户，策略和数据中心都将对其进行P2P流量控制到1Mbps的流量控制策略。</p>
    <p>[0097] 步骤407、策略和数据中心获取与DPI设备发送的用户的IP地址匹配的流量控制策略。</p>
    <p>[0098]     具体地，策略和数据中心获取DPI设备发送的用户的IP地址，判断该IP地址所属的群组；该IP地址所属的群组对应的流量控制策略即为与该IP地址匹配的流量控制策略。[0099] 例如：策略和数据中心获取的DPI设备发送的用户的IP地址为192. 168. 1. 147，判断该IP地址所属的群组为群组A ;该IP地址所属的群组对应的流量控制策略"P2P流量控制到1Mbps"即为与该IP地址匹配的流量控制策略。</p>
    <p>[0100] 步骤408、策略和数据中心根据DPI设备发送的IP地址和与该IP地址匹配的流量控制策略判断是否对该IP地址对应的用户流量进行流量控制。</p>
    <p>[0101] 优选地，根据所述匹配的流量控制策略获取流量门限，并根据所述用户的流量统计分析获取实际流量；判断所述实际流量是否超过流量门限。</p>
    <p>[0102] 当所述实际流量超过流量门限时，向DPI设备发送流量控制策略，使对所述DPI设备对用户的流量进行控制，转到步骤409 ;</p>
    <p>[0103] 当所述实际流量超过流量门限时，不向所述DPI设备发送流量控制策略，转到步骤410。</p>
    <p>[0104]     步骤409、 DPI设备根据策略和数据中心下发的策略进行实时的流量整形，将该小区下的P2P流量控制到1Mbps以内，而其他流量不受影响。[0105]     步骤410、基于GTP消息的分组域流控流程结束。</p>
    <p>[0106] 需要说明的是，将预设的对应关系中对应相同预设的流量控制策略的用尸的IP地址划分为一个群组只是本发明实施例中对应关系的表现形式之一，凡是根据将预设的对应关系获取与用户的IP地址匹配的流量控制策略均属于本发明的保护范围，例如列表等其他形式。</p>

    <p>[0144]     步骤608、 DPI设备根据策略和数据中心下发的策略进行实时的流量整形，将此时 段内的P2P流量控制到1Mbps以内，而其他流量不受影响。 [0145]     步骤609、基于GTP消息的分组域流控流程结束。</p>
    <p>[0146] 优选地，可以设置默认优先判断DPI设备中的IP地址是否属于特殊的流量控制策 略（如：黑白名单，基于时间调度的流量控制策略）中的IP地址，也可以为流量控制策略设 置优先级，例如：设置黑白名单，基于时间调度的流量控制策略高于普通的基于用户或地理 范围的流量控制策略，且黑白名单，基于时间调度的流量控制策略具有相同的优先级，必须 同时进行判断。</p>
    <p>[0147] 优选地，当根据其用户数据，获知该用于属于多个流量控制策略群组时，则根据该</p>
    <p>多个群组对应的流量控制策略共同控制该用户的流量。</p>
    <p>[0148]      本发明实施例提供一种流量控制系统，包括：</p>
    <p>[0149] 策略和数据中心，用于获取用户的流量统计分析，所述用户的流量统计分析中包 括所述用户的IP地址；根据预设的对应关系，获取与所述用户的IP地址匹配的流量控制策 略，所述预设的对应关系为预设的流量控制策略与用户的IP地址的对应关系；根据所述匹 配的流量控制策略和所述用户的流量统计分析，判断是否需要对所述用户进行流量控制； 根据判断结果进行处理。</p>
    <p>[0150] DPI设备，用于向所述策略和数据中心发送用户的流量统计分析，所述用户的流量 统计分析中包括所述用户的IP地址。 [0巧1] 其中，还包括：</p>
    <p>[0152]     GTP信令采集服务，用于获取用户的IP地址和对应的用户的相关信息，并发送给 所述策略和数据中心，使所述策略和数据中心根据所述用户的相关信息，获取用户的IP地 址与预设的流量控制策略的对应关系，具体包括： [0153] 获取GTP消息；</p>
    <p>[0154]     根据所述GTP消息获取用户的IP地址和用户的相关信息的对应关系；[0155] 将所述用户的IP地址和用户的相关信息的对应关系发送给所述策略和数据中 心，使所述策略和数据中心根据所述用户的相关信息，获取用户的IP地址与预设的流量控 制策略的对应关系。</p>
    <p>[0156]     本发明实施例提供一种策略和数据中心700，如图7所示，包括：</p>
    <p>[0157]     接收模块710，用于接收DPI设备发送的用户的流量统计分析，所述用户的流量统</p>
    <p>计分析中包括所述用户的IP地址；</p>
    <p>[0158] 匹配模块720，用于根据预设的对应关系，获取与接收模块710接收的用户的IP地 址匹配的流量控制策略，所述预设的对应关系为预设的流量控制策略与用户的IP地址的 对应关系；</p>
    <p>[0159]     判断模块730，用于根据匹配模块720获取的匹配的流量控制策略和接收模块710 接收的用户的流量统计分析，判断是否需要对所述用户进行流量控制； [0160]     处理模块740，用于根据判断结果进行处理。 [0161]     其中，匹配模块720具体用于：</p>
    <p>[0162]     接收GTP信令采集服务器发送的用户的IP地址和用户的相关信息的对应关系，所 述用户的相关信息包括对用户进行流量控制的控制元素，与预设的流量控制策略对应； [0163]     根据所述用户的相关信息，获取用户的IP地址与预设的流量控制策略的对应关 系；</p>
    <p>[0164] 根据所述用户的IP地址与预设的流量控制策略的对应关系，获取与所述用户的 IP地址相匹配的流量控制策略。</p>
    <p>[0165]     其中，判断模块730还用于：根据所述匹配的流量控制策略获取流量门限，并根据</p>
    <p>所述用户的流量统计分析获取实际流量；判断所述实际流量是否超过流量门限；</p>
    <p>[0166]     处理模块730具体用于：当所述实际流量超过流量门限时，向DPI设备发送流量控</p>
    <p>制策略，使对所述DPI设备对用户的流量进行控制；否则，不向所述DPI设备发送流量控制策略。</p>
    <p>[0167] 其中，所述预设的流量控制策略还包括：黑、白名单策略；和/或根据基于时间调 度的流量控制时变策略；</p>
    <p>[0168] 策略和数据中心700，如图8所示，还包括：存储模块750，与接收模块710和匹配 模块720连接，用于存储所述预设的流量控制策略与用户的IP地址的对应关系，并实时更 新；</p>
    <p>[0169] 所述存储模块还用于：将所述预设的流量控制策略与用户的IP地址的对应关系 中对应相同预设的流量控制策略的用户的IP地址划分为一个群组；所述匹配模块还用于， 通过判断所述用户的IP地址所属的群组获取与所述用户的IP地址匹配的流量控制策略。 [0170] 本发明的实施例对用户的数据进行基于IP地址的流量统计分析，并通过用户的 相关信息，获取与用户的IP地址匹配的流量控制策略，从而对用户进行分组域流量控制， 实现了差异化服务和不同业务服务质量的控制。</p>
    <p>[0171] 通过以上的实施方式的描述，本领域的技术人员可以清楚地了解到本发明可借助 软件加必需的通用硬件平台的方式来实现，当然也可以通过硬件，但很多情况下前者是更 佳的实施方式。基于这样的理解，本发明的技术方案本质上或者说对现有技术做出贡献的 部分可以以软件产品的形式体现出来，该计算机软件产品存储在一个存储介质中，包括若干指令用以使得一台终端设备（可以是手机，个人计算机，服务器，或者网络设备等）执行 本发明各个实施例所述的方法。</p>
