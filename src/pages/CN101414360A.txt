    <div class="abstract">本发明涉及一种组合型条形码，它是将现有的肉眼可识别的带数字、字母、符号或其组合的一维条形码与带有寄件人信息、收件人信息等肉眼不可识别的二维条形码进行适当组合而形成的一种新的条形码。本发明还及涉及采用该条形码的应用系统，它至少包括：所述的条形码生成部分和条形码识别处理部分；所述的条形码生成部分至少包括：编辑模块、条形码产生模块、打印模块；所述的条形码识别处理部分至少包括：输入接口、解码模块、信息处理模块、输出模块。本发明使用灵活方便、快速，而且使用该系统后不但减少了出错的机率，而且大大节省了劳动成本，减少了邮政人员劳动的工作量，使登记的字迹更加清晰可辩，表格更加美观。</div>

    <p>组合型条形码及其应用系统</p>
    <p>技术领域</p>
    <p>本发明涉及条形码及其应用技术领域，尤其是一种组合型条形码及其应用系统。 背景技术</p>
    <p>近年来，条形码技术日趋成熟，并且被广泛地应用在各个领域中。目前常用的条形 码为一维条形码，它是通过一系列"条"、"空"形成有规则的编码来表达丰富的数据 含义。这些条形的编码可被条形码的读取装置和译码器进行识别、转换，使条形码变 换成数据，同时将数据自动录入,这种条形码常用的有39码、EAN码等数十种，图1为 一种现在常用的EAN-128码。采用条形码技术进行科学管理的领域很多，它包括：图书 馆管理、档案管理、考勤管理以及商业、超市、税务、银行业、保险业等行业中。</p>
    <p>此外，由于21世纪90年代初，二维条形码的技术开发成功，使可携带式文档资料 成为了可能，并在某些场合已经获得了应用；图2为一种现在常用的PDF418二维条形 码。</p>
    <p>然而，遗憾的是，迄今为止，在条形码应用几乎是最早的邮政领域，尚未采用二维 条形码来减轻邮政的工作、减少邮件发送的时间。目前，寄挂号信时，邮局通常要将 邮寄的信息录入邮政计算机系统，同时在每个信封上都要贴上一张一维肉眼可识别的 条形码标贴，以便在发生意外时可&#26619;询。此外，在很多大型的公务机关，在写信时， 通常是将这种一维条形码连同收件人信息一起打印在信封上，以便于邮件的接收。对 于邮政局来说，每天要收到大量的挂号信，特别是稍大的城市，每个邮局中每一天都 要收到数千封挂号信，因此需要几个专人对信件进行挑出、然后手工将寄件人、收信 人、挂号等的信息抄写在纸张上，以供收件人签收；这种方式，不仅花费了邮政工作 人员的大量时间，而且，抄写容易出错。 发明内容</p>
    <p>为了解决上述问题，本发明提供一种组合型条形码及其应用系统，采用该发明后，可以大大减轻邮政工作人员的工作量，降低出错的机率。</p>
    <p>本发明中的组合型条形码是采用以下的技术方案来实现的： 组合型条形码，它是将现有的肉眼可识别的带数字、字母、符号或其组合的一维 条形码与带有寄件人信息、收件人信息等肉眼不可识别的二维条形码进行适当组合而</p>
    <p>形成的一种新的条形码。</p>
    <p>上述所述的组合型条形码，其特征在于： 一维条形码与二维条形码呈水平组合， 且互相间无任何重叠，且其相邻接处留有足够的空隙，以被计算机正确地读取。</p>
    <p>上述所述的组合型条形码，其特征在于： 一维条形码与二维条形码呈垂直组合， 且互相间无任何重叠，且其相邻接处留有足够的空隙，以被计算机正确地读取。</p>
    <p>上述所述的组合型条形码，其特征在于：所述的一维条形码可采用：Code39码、 Codabar码、Code25码、ITF25码、Matrix25码、UPC-A码、UPC-E码、EAN-13码、EAN-8 码、中国邮政码、Code-B码、MSI码、、Codell码、Code93码、ISBN码、ISSN码、Codel28 码EAN128码、Code39EMS等一维条码的任一种，且其为肉眼可读取的数字、字母或其 它符号。</p>
    <p>上述所述的组合型条形码，其特征在于：所述的二维条形码可采用：PDF417码、 Data Matrix码，Maxi Code码，Aztec码，QR Code码，Vericode码，Ultracode码， Code 49码，Code 16K码等二维条码的任一种。</p>
    <p>为了实现该组合型条形码的应用，本发明的另一任务是提供一种应用该条形码的 系统，该系统至少包括：所述的条形码生成部分和条形码识别处理部分；所述的条形 码生成部分至少包括：编辑模块、条形码产生模块、打印模块；所述的条形码识别处 理部分至少包括：输入接口、解码模块、信息处理模块、输出模块。</p>
    <p>上述所述的条形码应用系统，其特征在于：所述的条形码生成部分的编辑模块可 为计算机或其它便携式可编辑文字的设备。</p>
    <p>上述所述的条形码应用系统，其特征在于：所述的条形码生成部分的条形码产生 模块在生成条形码时，对于二维码部分，将不同的信息间采用空格符、逗号、引号或 其它可区分的符号之一或其组合进行标识。</p>
    <p>上述所述的条形码应用系统，其特征在于：所述的条形码识别处理部分的信息处理模块、输出模块，可以将每一个二维条形码部分的信息自动分离出，并可将其自动 地输入到预制的表格的相关栏目中。</p>
    <p>该条形码及采用该条形码的系统，具有以下优点：①一维码部分既可人眼识别， 又可计算机识别，因此更加灵活方便；②二维码部分因带有寄件人信息、收件人信息 等内容，因此条形码被扫描入计算机、进行解码、处理后，可以直接生成表格打印输 出，因此更加方便，不但减少了出错的机率，而且将原有的整理与登记工作合为了一 步，大大节省了劳动成本，减少了邮政人员劳动的工作量。③由于录入的数据最终是 通过计算机打印的，因此，字迹更加清晰可辩，表格更加美观。 附图说明</p>
    <p>图1为现在常用的一种一维条形码的示意图；</p>
    <p>图2为现在常用的一种二维条形码的示意图；</p>

    <p>图6为本发明条形码印在带窗口的信封的信纸上的示意图，条形码位于窗口处； 图7为应用本发明条形码的系统的一实施实例的框图； 图8为图7中信息处理模块处理后输出表格的示例图。 具体实施方式</p>
    <p>下面结合附图，详细说明具体实施方式。</p>
    <p>实施实例一及实施实例二为本发明组合型条形码的实施例，实施实例三为应用该 条形码的系统的实施例。 实施实例一：</p>
    <p>请参阅图3和图5，本发明一种组合型条形码，它是将现有的肉眼可识别的数字、 字母、组合形成的一维条形码与带有寄件人信息、收件人信息等肉眼不可识别的二维 条形码进行适当组合而形成的一种新的条形码。该组合型条型码中， 一维条形码采用 的是EAN-128码，除计算机可识别外，人眼肉眼也可辩认；二维条形码采用的是PDF417 码，内眼不能分辩其信息，但计算机读出的信息为："215554,江苏省常熟市尚湖镇大河村（28)沈塘湾陶家888888号，沈晶晶，100088，北京市海淀区蓟门桥西土城路6 号，国家知识产权局专利局收"。该条形码中， 一维码位于左边，二维码位于右边；一 维码与二维码近似呈水平布置， 一维码与二维码之间有计算机正确读取所需的空隙， 一维码与二维码之间无任何重叠部分。</p>
    <p>图5为本实施例中的条形码贴在普通信封上的示意图，该条形码水平方向粘贴于 收件人邮政编码与邮票之间；图中l为普通信封体，2为本条形码的一维部分，2为本 条形码的二维部分。 实施实例二-</p>
    <p>请参阅图4和图6，本发明一种组合型条形码，基本同实施实例一，只不过将一维 码位于上方，二维码位于下方； 一维码与二维码近似呈垂直布置， 一维码与二维码之 间有计算机正确读取所需的空隙， 一维码与二维码之间无任何重叠部分；该条形码中， 一维条形码采用的是EAN-128码,除计算机可识别外，人眼肉眼也可辩认；二维条形码 采用的是PDF417码，内眼不能分辩其信息，但计算机读出的信息为："215554,江苏 省常熟市尚湖镇大河村（28)沈塘湾陶家888888号，沈晶 晶，sjj@8888sjj8888. com, 13815000001， 100088，北京市海淀区蓟门桥西土城路6号， 国家知识产权局专利局收"。</p>
    <p>当然，上述所述的组合型条形码，其一维条形码可采用:Code39码、Codabar码、 Code25码、ITF25码、Matrix25码、UPC-A码、UPC-E码、EAN-13码、EAN-8码、中国 邮政码、Code-B码、MSI码、、Codell码、Code93码、ISBN码、ISSN码、Codel28码 EAN128码、Code39EMS等一维条码的任一种，且为肉眼可读取的数字、字母或其它符 号或其组合。</p>
    <p>而且，上述所述的组合型条形码，其二维条形码可采用：PDF417码、Data Matrix 码，Maxi Code码，Aztec码，QR Code码，Vericode码，Ultracode码，Code 49码， Code 16K码等二维条码的任一种。</p>
    <p>图6为本实施例中的条形码打印在带窗口的信封的信纸上的示意图,所述条形码 位于窗口处；图中l为带窗口的信封体，ll为窗口， 2为本条形码的一维部分，2为本 条形码的二维部分。实施实例三：</p>
    <p>请参阅图7和图8，本发明中应用上述所述条形码的系统，该系统包括：所述的条 形码生成部分9和条形码识别处理部分9;所述的条形码生成部分8包括：编辑模块、 条形码产生模块、打印模块；所述的条形码识别处理部分9包括：输入接口、解码模 块、信息处理模块、输出模块。</p>
    <p>上述所述的条形码应用系统，其特征在于：所述的条形码生成部分8的编辑模块 可为计算机或其它便携式可编辑文字的设备。</p>
    <p>上述所述的条形码应用系统，其特征在于：所述的条形码生成部分8的条形码产 生模块在生成条形码时，对于二维码部分，将不同的信息间采用空格符、逗号、引号 或其它可区分的符号之一或其组合进行标识。</p>
    <p>上述所述的条形码应用系统，其特征在于：所述的条形码识别处理部分9的信息 处理模块、输出模块，可以将每一个二维条形码部分的信息自动分离出，并可将其自 动地输入到预制的表格的相关栏目中，所述的表格可为EXCEL、 WPS等格式的电子表格。</p>
    <p>当然，该系统中还可增加其它扩充功能的部分；条形码生成部分还可增加其它模 块，如加密模块等；同样条形码识别处理部分还可增加其它模块，如解密模块等。</p>
    <p>图8中示意了一种处理输出的结果，分为了：序号、挂号信号码、收件人邮编、 收件人、收件人地址、收件日期、寄件人邮编、寄件人、寄件地址9列内容；每次扫</p>
