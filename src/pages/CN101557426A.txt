    <div class="abstract">本发明提供一种基于Web Service的统一管理接口机，所述统一管理接口机在Web Service设备侧实现，包括相互连接且分别独立运行的超文本传输协议HTTP模块及统一管理接口IMP模块，其中，所述HTTP模块用于基于HTTP协议收发所述Web Service设备与Web Service对端设备之间的可扩展标记语言XML消息；所述IMP模块，用于实现对所述Web Service设备消息的统一管理，构建发送给所述Web Service对端设备的XML消息以及解析所述Web Service对端设备发送的XML消息。本发明统一管理接口机、Web Service组件及方法，可以实现故障快速定位及开发人员对程序安全的可控性。</div>

    <p>基于Web Service的统一管理接口机、Web Service组件及方法 技术领域</p>
    <p>本发明涉及计算机通信技术的应用，尤其是基于Web Service方式的统 一管理接口机、Web Service组件及方法。</p>
    <p>背景技术</p>
    <p>随着短消息增值业务的蓬勃发展，企业竟争也日趋激烈，国内主要运营 商都在积极扩大市场占有份额，成熟的计算机通信技术不断得到运用，Web Service技术在这种背景下产生发展^f吏用，Web Service技术指一个能够使用 XML (Extensible Markup Language,可扩展标记语言）消息并通过网络来访 问的Interface(接口），这个Interface描述了 一组可访问的操:作，这时就向月l 务商提出更高的要求，开发出一套基于Web Service方式的DCS系统迫在眉 睫，常用做法是基于SOAP (Simple Object Access Protocol,简单对象访问 协议）提供一种在Web服务与其它应用程序之间进行通信的机制）开源工 具进行Web Service应用程序的开发。</p>
    <p>优点：采用SOAP开源工具进行web service开发时，应用程序的开发 比较轻;松，所有关于SOAP协议的编解码工作都由SOAP完成，应用程序只 需要关注相关的业务处理即可。</p>
    <p>缺点：故障的定位和解决比较困难，如果问题出现在应用程序中，解决 起来相对容易，但当问题出现在SOAP的开源代码中时，定位故障就显得相 对困难。如果需要解决此问题，需要深入研究SOAP的代码，工作量较大。 且不同的系统需求，需要开发不同的应用接口，增加开发周期，安全性，稳 定性下降。</p>
    <p>因此如何提供一种方法，用于提高DCS系统的扩展性，通用性并为外 部系统提供灵活统一管理的访问接口是目前急待解决的问题。发明内容</p>
    <p>本发明要解决的技术问题是提供一种统一管理接口机、Web Service组 件及方法，以实现故障快速定位。</p>
    <p>为解决上述技术问题，本发明提供一种基于Web Service的统一管理接 口机，所述统一管理接口机在Web Service设备侧实现，包括相互连接且分 别独立运行的超文本传输协议HTTP模块及统一管理接口 IMP模块，其中，</p>
    <p>所述HTTP模块用于基于HTTP协议收发所述Web Service设备与Web Service对端i殳备之间的可扩展标记语言XML消息；</p>
    <p>所述IMP模块，用于实现对所述Web Service设备消息的统一管理，构 建发送给所述Web Service对端设备的XML消息以及解析所述Web Service 对端设备发送的XML消息。</p>
    <p>进一步地，所述IMP模块设置统一管理配置文件，并根据所述统一管 理配置文件构建或解析可扩展标记语言XML消息的，所述统一管理配置文 件以XML文件格式定义，是解析描述Web服务信息的XML语言文件即 WSDL文件后获得的。</p>
    <p>进一步地，所述HTTP模块封装简单对象访问协议SOAP子模块及可扩 展标记语言XML子模块，所迷SOAP子模块用于基于SOAP协议封装消息 及基于HTTP协议实现数据通信；所述IMP模块调用所述XML子模块实现 消息解析功能。</p>
    <p>进一步地，所述HTTP模块封装简单对象访问协议SOAP子模块，用于 基于SOAP协议封装消息及基于HTTP协议实现数据通信；所述IMP模块 封装XML子模块，所述XML子模块独立运行，用于实现消息的构造及解 析功能。</p>
    <p>为解决上述技术问题，本发明还提供一种基于Web Service的统一管理 接口机，所述统一管理接口机包括相互连接且分别独立运行的超文本传输协 议HTTP模块及统一管理接口 IMP模块，其中，</p>
    <p>所述HTTP模块用于接收对端设备发送的消息并转发给所述IMP模块，还用于接收所述IMP模块处理后的消息，并转发给所述对端设备；</p>
    <p>所述IMP模块设置统一管理配置文件，用于接收本端设备发送的消息， 并根据所述统一管理配置文件构建可扩展标记语言XML消息以及转发给所 述HTTP模块；还用于接收所述HTTP模块转发的消息，并根据所述统一管 理配置文件解析所述HTTP模块发送的消息；</p>
    <p>所述统一管理配置文件是根据描述Web服务信息的XML语言文件即 WSDL文件定义的XML格式文件。</p>
    <p>进一步地，所述HTTP模块封装简单对象访问协议SOAP子模块及可扩 展标记语言XML子模块，所述SOAP子模块用于基于SOAP协议封装消息 及基于HTTP协议实现数据通信；所述IMP模块调用所述XML子模块实现 消息解析功能。</p>

    <p>所述HTTP模块用于接收Web Service对端设备发送的消息并转发给所 述IMP模块，还用于接收所述IMP模块处理后的消息，并转发给所述Web Service对端设备；</p>
    <p>所述IMP模块配置统一管理配置文件，用于接收所述Web Service设备 发送的消息，并根据所述统一管理配置文件构建可扩展标记语言XML消息 以及转发给所述HTTP模块；还用于接收所述HTTP模块转发的消息，并根 据所述统一管理配置文件解析所述HTTP模块发送的消息；</p>
    <p>所述统一管理配置文件是根据描述Web服务信息的XML语言文件即 WSDL文件定义的XML格式文件。</p>
    <p>进一步地，所述HTTP模块封装简单对象访问协议SOAP子模块及可扩 展标记语言XML子模块，所述SOAP子模块用于基于SOAP协议封装消息 及基于HTTP协议实现数据通信；所述IMP模块调用所述XML子模块完成 消息的解析功能。进一步地，所述Web Service设备是服务提供者或服务请求者。</p>
    <p>为解决上述技术问题，本发明还提供一种基于统一管理接口机实现Web Service的方法，所述统一管理接口机包括相互连接且分别独立运行的超文 本传输协议HTTP模块及统一管理接口 IMP模块，所述IMP模块配置根据 WSDL文件获取的统一管理配置文件，该方法包括：</p>

    <p>HTTP模块将IMP模块构建的XML消息进行SOAP封装后基于HTTP 协议发送给对端设备；</p>

    <p>所述IMP模块根据统一管理配置文件解析对端设备发送的XML消息。</p>
    <p>相较于现有技术，本发明统一管理接口才几、Web Service组件及方法， 通过合理划分模块功能，可以很好地解决软件安全且测试人员在发现故障时 可以迅速定位问题的关键所在地及时解决故障。本发明根据设置的统一配置 文件实现XML消息的构建和解析，在后续相同系统不同功能需求的web Service服务当中无需再次开发通讯与解析模块，程序员遇到此类问题只需 修改XML文件格式的统一配置文件无需开发相关联模块，极大提高程序使 用效率，降低开发成本，增强系统稳定性，扩展性。本发明在不改变原有网 络的架构的前提下得以实现，灵活性，兼容性，稳定性极强。</p>

    <p>具体实施方式</p>
    <p>如图1所示，本发明Web Service ( web service )系统包4舌服务请求者、服务提供者及服务中间件，可实现下发、查找、绑定等操作，其中，</p>
    <p>服务请求者：用于发现Web服务，然后调用这些服务以创建应用程序， 其次应用程序通过服务描述语言进行描述，其描述信息通过中间件发送。具 体应用时，可由图3中的OSP( Open service platform,业务开放平台）实现。</p>
    <p>服务提供者：提供服务端口及维护注册表使服务始终保持可用状态。具 体应用时，可由DCS ( Data Center for Short Message Service,短信业务数据 中心）实现。</p>
    <p>服务中间件，也即本发明的统一管理接口机：是服务提供者与服务请求 者的中介，请求消息/响应消息的接收/转发中继站。服务中间件可以在服务 提供者及服务请求者侧实现。</p>
    <p>本发明中，服务请求者侧的服务中间件和服务提供者侧的服务中间件互 为对端，服务中间件实现侧的设备称为本端侧设备，以服务提供者侧的服务 中间件为例，服务提供者为本端侧设备，服务请求者为对端侧设备。</p>
    <p>如图2所示，本发明所说的统一管理接口机包括相互连接且分别独立运 行的HTTP模块（即超文本传输协议模块）及IMP模块，详细信息如下：</p>
    <p>HTTP模块用于基于HTTP协议收发本端设备与对端设备之间的消息， 封装有相连接的简单对象访问协议子模块（简称SOAP子模块）及可扩展标 记语言子模块（简称XML子模块），其中，</p>
    <p>SOAP子模块用于处理网络通讯之间数据通信的数据包的高速交换及基 于SOAP协议对交互消息进行SOAP封装，还用于对数据包加密，以及利用 专有的加密协议将加密it据包还原成可解析的XML文件;格式消息体，使 HTTP模块可以正常接收发送Web Service方式的XML消息体，Web Service 方式的XML消息体发送接收则通过http模块提供的远程访问接口得以实 现；</p>
    <p>SOAP子模块的具体功能体现在：用于接收对端发送的消息并转发给 IMP模块；还用于将IMP模块处理后的消息进行SOAP封装后发送给对端；</p>
    <p>XML子模块用于解析SOAP模块接收的XML消息体；</p>
    <p>统一管理接口 （ Integrate manage port, IMP)模块，设置统一管理配置文件，是一种消息统一集中管理的设计架构，主要实现应用程序与网络通讯</p>
    <p>模块(指HTTP模块)的分离，进行单独开发，解决系统稳定性与扩展性；</p>
    <p>IMP模块的具体功能体现在：用于接收HTTP模块发送的消息，以及才艮 据统一管理配置文件调用XML子模块解析该消息的具体内容并发送给本端 设备；还用于接收本端设备发送的消息，以及根据消息内容及统一管理配置 文件构造XML消息发送给所述HTTP模块；</p>
    <p>统一管理配置文件，是将WSDL文件解析后以XML文件格式定义的文 件，XML的灵活性，简单性，易用性是处理系统兼容性问题的首选目标。</p>
    <p>下发：服务提供者通过中间件确定消息是否下发。</p>
    <p>查找：查找操作由服务请求者和中间件共同完成。服务请求者描述他们 正在寻找的服务类型，而服务中间件发布与请求最匹配的结果。</p>
    <p>绑定：绑定操作发生在服务请求者和服务提供者之间。双方经过适当的 商讨之后，请求者就可以访问和调用提供者所提供的服务。</p>
    <p>WSDL, Web Services Description Language 4笛述Web Service月良务"f言息、 的XML语言，且提供了一种向其它程序描述Web Service服务的统一方法； 主要描述信息内容，定义了描述Web服务接口规范的标准格式。</p>
    <p>本发明还提供一种所述Web Service组件，Web Service组件包括Web Service设备及统一管理接口机，所述统一管理接口机包括相互连接且分别 独立运行超文本传输协议HTTP冲莫块及统一管理接口 IMP冲莫块，所述Web Service设备与所述IMP模块相连接，所说的统一管理接口机的模块结构如 上所迷。</p>

    <p>步骤3:服务中间件的IMP模块接收到以XML构造的消息，根据设置 的统一管理配置文件调用XML子模块解析具体消息内容，转发到消息功能 具体实施模块服务提供者DCS;</p>
    <p>步骤4:服务提供者DCS接收到服务中间件的IMP模块解析后详细的 消息内容参数，根据参数进行相应的功能设置，并返回IMP请求功能消息 的处理结果及响应消息；</p>

    <p>当服务请求者OSP收到响应消息并做响应处理后，至此一个完整web Service方式的请求/响应功能得以完整结束。</p>
    <p>本发明基于web Service方式统一管理接口机，利用基于Web Service方 式的IMP架构结合HTTP与XML技术对DCS( Data Center for Short Message Service,短信业务数据中心）数据处理接口机进行规范统一规划设计，从而 提高DCS接口机系统的可维护性和通用性。</p>
    <p>因为接收/发送消息由SOAP子模块封装SOAP消息并根据HTTP协议 实现，解码工作由XML模块实现，通讯与解码模块二者既有明确分工又同 时保持密切联系且共同封装在一个独立运行的HTTP模块中。本发明统一管 理接口机、Web Service组件及方法，通过合理划分模块功能，可以很好地 解决软件安全且测试人员在发现故障时可以迅速定位问题的关键所在地及时解决故障，及开发人员对程序安全的可控性。由于运营商基于web Service 方式的基础上实现不同功能，在后续相同系统不同功能需求的web Service 服务当中无需再次开发通讯与解析模块，程序员遇到此类问题只需修改 XML文件格式的统一配置文件无需开发相关联模块，他将极大提高程序员 的编程时间，编程质量，同时极大提高程序使用效率，降低开发成本，增强 系统稳定性，扩展性。本发明在不改变原有网络的架构的前提下得以实现， 灵活性，兼容性，稳定性极强。</p>
    <p>总之，本发明为WEB SERVICE功能开发提供了一套简单实用解决方 案，HTTP协议+XML协议+IMP应用程序，就可以进行WEBSRVICE功能 开发，实现开发人员对程序安全隐患的可控性，因为HTTP协议，XML协 议都是通信常用协议，学习运用简单易懂，而本发明就抓住此特点实用这两 种协i义有机组合利用开发实现WEB SERVICE功能。</p>
