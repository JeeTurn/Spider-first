    <div class="abstract">本发明提供了一种异形矩阵式二维条码构建方法，包括以下步骤：获取打印设备的图像打印分辨率；获取文本上待打印的异形矩阵二维条码符号区域的边界尺寸；定义模块尺寸；定义异形二维条码符号的整体尺寸；确定模块数初值M为X4/X3；确定最终的模块数M1以及与该模块数M1对应的版本V；定义完成满足需求的异形矩阵式二维条码参数为：模块数M1、版本V和模块尺寸，并按照该异形矩阵式二维条码参数使用标准生成算法生成所需要的异形矩阵式二维条码符号图像。本发明实现了X方向和Y方向具有不等的图像打印分辨率的打印机对二维条码进行打印的支持和满足了特殊文本对非正方向二维条码的需求。</div>

    <div class="claim"> <div num="1" class="claim">

    <p>异形矩阵式ニ维条码构建方法</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及条码领域，具体而言，涉及�种异形矩阵式ニ维条码构建方法。</p>
    <p>背景技术</p>
    <p>[0002]	矩阵式ニ维条码是由若干边长相等的模块按X和Y方向相同数量拼接组成，整体，矩阵式ニ维条码符号也是�个边长相等的正方形图像符号，该模块通过黑色和白色分别表示信息O和1，矩阵式ニ维条码能表达�个数据包。模块越大，矩阵式ニ维条码符号的图像就越大，模块的尺寸和矩阵式ニ维条码符号的尺寸成正比。如果限制了矩阵式ニ维条码符号的面积，那么如果想要容纳更多的信息，就需要把模块的尺寸定义地比较小，但模块的尺寸太小了又无法识别模块，因此这相当于是矛盾的指标。 [0003]	通用的矩阵式ニ维条码图像符号一般是方的，矩阵式ニ维条码图像符号的印刷使用专用条码打印机打印或者随其所在的文本一起印刷。在�些特殊的应用场合，这种方的矩阵式ニ维条码符号和专用设备打印或印刷无法实施，该应用场合例如：实际应用场合无法使用条码打印机，所使用的打印设备又不支持X方向和Y方向相同的图像打印分辨率，也就是说打不出来正常的方的矩阵式ニ维条码的情况；还有，当文本预留的区域面积，希望打印�个长方形的矩阵式ニ维条码的情&#27841;。上述问题都是通常情况下现有的条码技术无法很好解决和实现的问题。</p>
    <p>发明内容</p>
    <p>[0004]	本发明的主要目的在于提供一种异形矩阵式ニ维条码构建方法，用以实现对X方向和Y方向分辨率不同的打印设备的支持，同时用以实现构建�个非正方形的矩阵式ニ维条码图像符号。</p>
    <p>[0005]	本发明提供了一种异形矩阵式ニ维条码构建方法，包括以下步骤：</p>
    <p>[0006]	步骤I :获取打印设备的图像打印分辨率，井根据该图像打印分辨率计算出单个打印点的尺寸X1XY1，其中，Xl为该单个打印点的宽度，Yl为该单个打印点的长度；</p>
    <p>[0007]	步骤2 :获取文本上待打印的异形矩阵ニ维条码符号区域的边界尺寸为X2XY2，其中，X2为所述异形矩阵ニ维条码符号区域的边界的长度，Y2为所述异形矩阵ニ维条码符号区域的边界的宽度；</p>
    <p>[0008]	步骤3 :定义模块尺寸为X3XY3，其中，X3 = 2XX1, Y3 = Yl ；</p>
    <p>[0009]	步骤4 :定义异形ニ维条码符号的整体尺寸为X4XY4，其中，X4 = X2-2X3XX3，Y4 = Y2-2X3XY3 或 X4 = X2-2X4XX3，Y4 = Y2-2X4XY3 ；</p>
    <p>[0010]	步骤5 :确定模块数初值M为X4/X3 ；</p>
    <p>[0011]	判断MX Y3是否小于或等于Y4，如果是则转至步骤9，否则转至步骤6 ;</p>
    <p>[0012]	步骤6 ：使得模块数初值M为Y4/Y3 ；</p>
    <p>[0013]	步骤7 :判断MXX3是否小于或等于X4，如果是则转至步骤9，否则转至步骤8 ；</p>
    <p>[0014]	步骤8 :将模块数初值M的值减1，转至步骤7 ；[0015]	步骤9 :根据矩阵式ニ维条码标准，找出与模块数初值M最接近的并小于或等于模块数初值M的模块数Ml，以及与该模块数Ml对应的版本V ；</p>
    <p>[0016]	步骤10 :定义完成满足需求的异形矩阵式ニ维条码&#21443;数为：模块数Ml、版本V和模块尺寸X3XY3，并按照该异形矩阵式ニ维条码&#21443;数并使用标准生成算法生成所需要的异形矩阵式ニ维条码符号图像。</p>
    <p>[0017]	实施时，在步骤4中，所述异形ニ维条码符号的整体尺寸为：在所述异形矩阵ニ维条码符号区域的面积固定的情况下，所述异形矩阵ニ维条码符号区域的面积减去需要留出的四周空白的面积后得到的可以实际打印ニ维码的面积。</p>
    <p>[0018]	实施时，在步骤2中，当所述文本上待打印的异形矩阵ニ维条码符号区域为发票上的表格时，该边界尺寸为表格尺寸。</p>
    <p>[0019]	与现有技术相比，本发明所述的异形矩阵式ニ维条码构建方法，实现了 X方向和Y方向具有不等的图像打印分辨率的打印机对ニ维条码进行打印的支持和满足了特殊文本对非正方向ニ维条码的需求， </p>

    <p>[0020]	图I是本发明所述的异形矩阵式ニ维条码构建方法的流程图。</p>
    <p>具体实施方式</p>
    <p>[0021]	本发明提供了一种异形矩阵式ニ维条码构建方法，用以实现对X方向和Y方向分辨率不同的打印设备的支持，同时用以实现构建�个非正方形的矩阵式ニ维条码图像符号。</p>
    <p>[0022]	如图I所示，本发明所述的异形矩阵式ニ维条码构建方法，包括以下步骤：</p>
    <p>[0023]	步骤11 :获取打印设备的图像打印分辨率，井根据该图像打印分辨率计算出单个打印点的尺寸X1XY1，其中，Xl为该单个打印点的宽度，Yl为该单个打印点的长度；</p>
    <p>[0024]	步骤12 :获取文本上待打印的异形矩阵ニ维条码符号区域的边界尺寸为X2XY2，其中，X2为所述异形矩阵ニ维条码符号区域的边界的长度，Y2为所述异形矩阵ニ维条码符号区域的边界的宽度；</p>
    <p>[0025]	所述文本上待打印的异形矩阵ニ维条码符号区域，是指法定的票据表格；</p>
    <p>[0026]	以所述文本上待打印的异形矩阵ニ维条码符号区域为发票上的表格为例，发票上的各个表格都规定了各自的内容，不能越界，相应的对于规定好打印ニ维条码的表格，ニ维条码不能出界，表格尺寸就是边界尺寸；</p>
    <p>[0027]	步骤13 :定义模块尺寸为X3XY3，其中，X3 = 2XX1，Y3 = Yl ；</p>
    <p>[0028]	步骤14 :定义异形ニ维条码符号的整体尺寸为X4XY4，其中，X4 = X2-2X3XX3，Y4 = Y2-2X3XY3 或 X4 = X2-2X4XX3，Y4 = Y2-2X4XY3 ；</p>
    <p>[0029]	根据ニ维条码的国家标准，在二维条码符号的周围需要留出至少3个模块尺寸的空白以便于识别，所述异形矩阵ニ维条码符号区域的面积固定的情况下，步骤4是将所述异形矩阵ニ维条码符号区域的面积减去需要留出的四周空白的面积，得到可以实际打印ニ维码的面积，并将该面积定义为异形ニ维条码符号的整体尺寸；</p>
    <p>[0030]	步骤15 :确定模块数初值M为X4/X3 ；[0031]	判断MXY3是否小于或等于Y4，如果是则转至步骤19，否则转至步骤16 ；</p>
    <p>[0032]	步骤16 :使得模块数初值M为Υ4/Υ3 ；</p>
    <p>[0033]	步骤17 :判断MXΧ3是否小于或等于Χ4，如果是则转至步骤19，否则转至步骤18 ；</p>
    <p>[0034]	步骤18 :将模块数初值M的值减I，转至步骤17 ；</p>
    <p>[0035]	步骤19 :根据矩阵式ニ维条码标准，找出与模块数初值M最接近的并小于或等于模块数初值M的模块数Ml，以及与该模块数Ml对应的版本V ；</p>
    <p>[0036]	步骤110 :定义完成满足需求的异形矩阵式ニ维条码&#21443;数为：模块数Ml、版本V和模块尺寸X3XY3，并按照该异形矩阵式ニ维条码&#21443;数使用标准生成算法生成所需要的异形矩阵式ニ维条码符号图像。 </p>
    <p>[0037]	下面根据�具体的实施例来对本发明进行介绍：</p>
    <p>[0038]	打印机使用型号为SK810的9针式打印机，文本待打印面积尺寸限制为横向20mm，纵向18毫米。</p>
    <p>[0039]	步骤I :SK810打印机图像打印分辨率为120X72dpi，dpi表示每英寸点数，I英寸等于IOOOmiI，计算单个打印点的尺寸Xl = 1000/120 = 8. 33mil, Yl = 1000/72 =13. 89mil ；</p>
    <p>[0040]	步骤2 :需求给出X2 = 20mm, Y2 = 18mm,折算成英制单位的话，X2 =20mmX39. 37mil/mm = 787. 4mil,Y2 = 18mmX39. 37mil/mm = 708. 66mil,折算单位转换依据 Imm = 39. 37mil ；</p>
    <p>[0041]步骤	3 :定义模块尺寸，X3 = 2X8. 33 = 16. 66mil, Y3 = 13. 89mil ；</p>
    <p>[0042]	步骤4 :定义异形ニ维条码符号整体尺寸，X4 = 787. 4-2X3X16. 66 =687. 44mil, Y4 = 708. 66-2X3X13. 89mil = 625. 32mil ；</p>
    <p>[0043]步骤	5 :确定模块数初值 M = 687.44/16.66 = 41，验证 41 X 13. 89 = 569. 49mil,小于或等于625. 32mil，满足条件，可用；</p>
    <p>[0044]	步骤6 :查矩阵式ニ维条码标准例如汉信码国标GB/T21049-2007，版本10的模块数为41，与模块数初值相等，可作为模块数终值；</p>
    <p>[0045]	步骤7 :完成&#21443;数定义，&#21443;数为：模块数41，版本10，模块尺，16.66X13.89mil，由</p>
    <p>这些&#21443;数结合标准里面的生成算法可构建满足本实施例需求的异形矩阵式ニ维条码。</p>
