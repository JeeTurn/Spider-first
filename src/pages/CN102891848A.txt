    <p>利用IPSec安全联盟进行加密解密的方法</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及通信技术领域，特别涉及一种利用IPSec安全联盟进行加密解密的方法。</p>
    <p>背景技术 [0002]	互联网协议安全性（IPSec, Internet Protocol Security)是网络通信中广泛应用的一种安全协议。在该协议的应用中，两个设备之间进行信息传递需要建立IPSec安全联盟用于信息的加密解密过程，以确保信息的安全传递。当多个设备分别向一个设备发送信息，一个设备接收到信息并向多个设备分别返回回应信息时，会造成回应信息与多个设备不对应，各易造成错乱。</p>
    <p>发明内容</p>
    <p>[0003](一）解决的技术问题</p>
    <p>[0004]	本发明解决了多个本端设备分别向一个对端设备发送报文，一个对端设备分别向多个本端设备返回回应报文的技术问题。</p>
    <p>[0005]	(二）技术方案</p>
    <p>[0006]	本发明提出了一种利用IPSec安全联盟进行加密解密的方法，其特征在于，至少一个本端设备向对端设备发送报文，所述方法包括：</p>
    <p>[0007]	A、本端设备向本端的第一防火墙发送报文；</p>
    <p>[0008]	B、第一防火墙接收到报文，利用IPSec安全联盟对报文进行加密，并将加密报文发送给对端的第二防火墙；</p>
    <p>[0009]	C、第二防火墙接收到加密报文后，根据加密报文的安全协议类型和安全参数索引进行IPSec安全联盟的查找，并通过查找到的IPSec安全联盟对加密报文进行解密，得到解密报文；</p>
    <p>[0010]	D、第二防火墙创建映射关系表，并将解密报文发送给所述对端设备。</p>
    <p>[0011]	优选地，步骤A中所述报文包括：本端设备地址和对端设备地址。</p>
    <p>[0012]	优选地，步骤B中所述加密报文包括：第一防火墙地址和第二防火墙地址。</p>
    <p>[0013]	优选地，步骤C中所述解密报文包括：本端设备地址和对端设备地址。</p>
    <p>[0014]	优选地，步骤D中所述映射关系表包括本端设备地址、对端设备地址、第一防火墙地址、第二防火墙地址以及IPSec安全联盟的对应关系。</p>
    <p>[0015]	优选地，步骤D对端设备接收到解密报文后还包括：</p>
    <p>[0016]	E、对端设备向第二防火墙发送回应报文，所述回应报文包括对端设备地址和本端设备地址；</p>
    <p>[0017]	F、第二防火墙将接收到的回应报文与映射关系表进行匹配，获得与回应报文中的对端设备地址和本端设备地址相对应的第一防火墙地址和第二防火墙地址、IPSec安全联盟；[0018]	G、利用所述IPSec安全联盟对回应报文进行加密,加密后回应报文包括第一防火墙地址和第二防火墙地址，并将加密回应报文发送给第一防火墙；</p>
    <p>[0019]	H、第一防火墙接收到加密回应报文后利用IPSec安全联盟对加密回应报文进行解密，并将解密回应报文发送给本端设备。</p>
    <p>[0020]	优选地，步骤F中将接收到的回应报文与映射关系表进行匹配时，将回应报文中对端设备地址和本端设备地址进行对调后再与映射关系表进行匹配。</p>
    <p>[0021](三）有益效果</p>
    <p>[0022]	本发明通过将多个本端设备分别向一个对端设备发送的报文加密，且对端设备的防火墙在将报文解密后，创建映射关系表，实现了当多个本端设备分别向一个对端设备发送报文，对端设备的防火墙根据映射关系分别向多个本端设备返回回应报文。</p>
    <p>附图说明 </p>
    <p>[0023]	图I是本发明提出的利用IPSec安全联盟进行加密解密的方法流程图。</p>
    <p>具体实施方式</p>
    <p>[0024]	下面将结合本发明实施例中的附图，对本发明实施例中的技术方案进行清楚、完整地描述。</p>
    <p>[0025]	实施例I</p>
    <p>[0026]	本发明提出了一种利用IPSec安全联盟进行加密解密的方法，其特征在于，至少一个本端设备向对端设备发送报文，所述方法包括：</p>
    <p>[0027]	A、本端设备向本端的第一防火墙发送报文；</p>
    <p>[0028]	B、第一防火墙接收到报文，利用IPSec安全联盟对报文进行加密，并将加密报文发送给对端的第二防火墙；</p>
    <p>[0029]	C、第二防火墙接收到加密报文后，根据加密报文的安全协议类型和安全参数索引SPI进行IPSec安全联盟的查找，并通过查找到的IPSec安全联盟对加密报文进行解密，得到解密报文；</p>
    <p>[0030]	D、第二防火墙创建映射关系表，并将解密报文发送给所述对端设备。</p>
    <p>[0031]	实施例2</p>
    <p>[0032]	本实施例包括了实施例I的所有内容，另外，将步骤A中所述报文进一步限定为包括：本端设备地址和对端设备地址。</p>
    <p>[0033]	步骤B中所述加密报文包括：第一防火墙地址和第二防火墙地址。第一防火墙将报文加密为封装安全净载（Encapsulating Security Payload, ESP)/认证标头(Authentication Header, AH)报文。</p>
    <p>[0034]	步骤C中所述解密报文包括：本端设备地址和对端设备地址。当第二防火墙接收到ESP/AH报文后，根据报文的安全协议类型（AH协议或ESP协议）和安全参数索引查找服务器上的IPSec SA (IPsec安全联盟)，找到之后，利用IPSec安全联盟将加密报文进行解密，并发送给对端设备。</p>
    <p>[0035]	步骤D中所述映射关系表包括：本端设备地址和对端设备地址、第一防火墙地址和第二防火墙地址、IPSec安全联盟的对应关系。将加密报文解密之后，形成映射关系表，并将解密报文发送给对端设备。</p>
    <p>[0036]	步骤D对端设备接收到解密报文后还包括：</p>
    <p>[0037]	E、对端设备向第二防火墙发送回应报文，所述回应报文包括对端设备地址和本端设备地址；</p>
    <p>[0038]	F、第二防火墙将接收到的回应报文与映射关系表进行匹配，获得与回应报文中的对端设备地址和本端设备地址相对应的第一防火墙地址和第二防火墙地址、IPSec安全联盟；</p>
    <p>[0039]	G、利用所述IPSec安全联盟对回应报文进行加密,加密后回应报文包括第一防火墙地址和第二防火墙地址，并将加密回应报文发送给第一防火墙；</p>
    <p>[0040]	H、第一防火墙接收到加密回应报文后利用IPSec安全联盟对加密回应报文进行 解密，并将解密回应报文发送给本端设备。</p>
    <p>[0041]	步骤F中将接收到的回应报文与映射关系表进行匹配时，将回应报文中对端设备地址和本端设备地址进行对调后再与映射关系表进行匹配。由于是回应报文，因此回应报文中对端设备地址和本端设备地址需要对调，对调后和映射关系表进行匹配。</p>
    <p>[0042]	实施例3</p>
    <p>[0043]	本实施例利用具体的IP地址来更为具体的说明一种利用IPSec安全联盟进行加密解密的方法，具体实施例如下：</p>
    <p>[0044]	Pcl-------------第一防火墙------------第二防火墙------------Pc2</p>
    <p>[0045]	客户端Pcl向客户端Pc2发送报文,其中，经过第一防火墙（Firewall, fw)和第二防火墙。</p>
    <p>[0046]	Pcl的本端设备IP2地址I. I. I. I发送报文到pc2的对端设备IP2地址2. 2. 2. 2。</p>
    <p>[0047]	此时报文结构为：</p>
    <p>[0048]</p>
    <p>I IP2 (原 1.1.1.1/目的 2.2.2.2 ) | data (数据) j</p>
    <p>[0049]	当经过第一防火墙时需要通过IPSec SA进行加密为ESP报文，第一防火墙的IPl地址为192. 168. I. I到第二防火墙的IPl地址为192. 168. I. 2</p>
    <p>[0050]	第一防火墙加密后的报文结构为：</p>
    <p>[0051]</p>
    <p>jiPU srcl92.168.1.l/dstl92.168.1.2) |ESP |1P2( srcl.l.1 ■ l/dst2.2.2.2 ) |data|</p>
    <p>[0052]	第二防火墙接收后并根据报文ESP中的安全参数索引和ESP安全协议类型进行IPSec SA的查找，并通过查找到的IPSec SA进行解密，成功解密后创建映射关系表，映射关系表如下：</p>
    <p>[0053]	I、加密后 IP 地址：IP1 (src 192. 168. I. 1/dst 192. 168. I. 2)</p>
    <p>[0054]	2、IPSec 安全联盟：IPSec SA</p>
    <p>[0055]	3、解密后 IP 地址：IP2 (srcl. I. I. l/dst2. 2. 2. 2)[0056]	解密后将解密后的报文进行转发给Pc2，此时报文结构为：</p>
    <p>[0057]</p>
    <div class="patent-image small-patent-image"> <a href="//patentimages.storage.googleapis.com/CN102891848A/CN102891848AD00061.png"> <img id="idf0001" file="CN102891848AD00061.tif" img-content="drawing" img-format="tif" src="//patentimages.storage.googleapis.com/CN102891848A/CN102891848AD00061.png" class="patent-full-image" alt="Figure CN102891848AD00061"> </a> </div>
    <p>[0058]	Pc2获取了报文之后发出回应报文，报文格式如下：</p>
    <p>[0059]</p>
    <div class="patent-image small-patent-image"> <a href="//patentimages.storage.googleapis.com/CN102891848A/CN102891848AD00062.png"> <img id="idf0002" file="CN102891848AD00062.tif" img-content="drawing" img-format="tif" src="//patentimages.storage.googleapis.com/CN102891848A/CN102891848AD00062.png" class="patent-full-image" alt="Figure CN102891848AD00062"> </a> </div>
    <p>[0060]	第二防火墙接收到回应报文后匹配映射关系表中的解密后IP地址：IP2(srcl. I. I. l/dst2. 2. 2. 2)，匹配时需要将srcl. I. I. I和dst2. 2. 2. 2进行对调匹配，匹配上之后就根据表中的IPSec安全联盟对回应报文进行加密，加密后的回应报文包括第一防火墙地址和第二防火墙地址，将加密后的回应报文发送给第一防火墙，第一防火墙解密后将报文转发给Pci。</p>
