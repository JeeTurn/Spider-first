    <div class="abstract">本发明涉及一种基于RIA的高性能Web传输网管系统及实现方法，该Web传输网管系统包括：设置于客户端的Web浏览器，Web浏览器通过HTTP协议将富客户端应用程序从Web服务器上下载到本地计算机，然后离线或在线运行该程序；Web服务器端包含富客户端应用程序、Web数据服务层、Web&#160;Services层、业务逻辑层、数据访问层及系统集成层。本发明有效地利用RIA架构及富客户端技术，应用于电信传输网管系统中，在主流的Web浏览器上运行功能强大的网管客户端；该Web传输网管客户端具有各种复杂的图形用户界面，支持局部刷新界面，直接与服务器进行同步或异步交互，实时接受服务器端推送的数据，并且能够充分利用本地计算机的计算资源和存储资源。</div>

    <p>一种基于Rl A的高性能Web传输网管系统及实现方法</p>
    <p>技术领域</p>
    <p>[0001]	本发明涉及电信传输网管系统，具体说是一种基于RIA的高性能Web传输网管系 统及实现方法。所述RIA是Rich Internet Application，即：富互联网应用程序。</p>
    <p>背景技术</p>
    <p>[0002]	传统的基于C/S(客户端/服务器）架构的传输网管系统在应用、部署、更新和维 护等方面存在着诸多不便与困难，自然而然人们会想到B/S(浏览器/服务器）架构的Web 传输网管系统，但由于Web技术的限制，Web传输网管系统的发展非常缓慢并存在不少问 题：</p>
    <p>[0003]	1)传统Web传输网管系统的用户操作体验简单，不容易开发丰富的效果来展示数 据；</p>
    <p>[0004]	2)用户的请求是单一、同步的，并且需要刷新整个页面，带来了很多的无需的开 销，影响了响应速度，更让用户感觉有太多的等待；</p>
    <p>[0005]	3)传统Web传输网管系统很难或者需要通过其他技术的辅助才能操作客户端计 算机的资源，服务器的负载过高而客户端计算机的计算能力没有充分使用；</p>
    <p>[0006]	4)传统Web传输网管系统的数据通讯方式比较单一，均是基于HTTP连接，而HTTP 连接是一种非永久连接，基于HTTP协议的Web传输网管系统只能由客户端向服务器端发起 请求，请求响应后，连接也终止，无法实现服务器端主动向客户端推送数据等。</p>
    <p>[0007]	为了利用B/S架构的优势，很多人尝试进行改进，比如混合B/S和C/S而构成B/ C/S架构，即在浏览器中嵌入业务功能组件来弥补缺陷，但最终只是折中这两种架构的优 缺点，而且加大了开发的复杂度，同时由于嵌入的组件作为插件在浏览器上安装时经常被 认为不可信，而被相关安全设置或安全防护软件所屏蔽，给用户带来不少麻烦，可谓得不偿 失。</p>
    <p>[0008]	上述Web传输网管系统的诸多问题，都是传统Web应用程序的共性问题，为了解决 此类问题，一种被称为富互联网应用程序（RIA)、具有高度互动性和丰富用户体验的网络应 用程序出现了，使得开发真正的B/S架构的高性能Web传输网管系统成为可能。</p>
    <p>[0009]	RIA是Web技术的发展的一个里程碑。它具有的桌面应用程序的特点，包括：</p>
    <p>[0010]	1、在消息确认和格式编排方面提供互动用户界面；</p>
    <p>[0011]	2、在无刷新页面之下提供快捷的界面响应时间；</p>
    <p>[0012]	3、提供通用的用户界面特性如拖放式（Drag and Drop)以及在线和离线操作能 力；</p>
    <p>[0013]	4、同时具有Web应用程序的特点，如：立即部署、跨平台、采用逐步下载来检索内 容和数据以及可以充分利用被广泛采纳的互联网标准。</p>
    <p>[0014]	RIA的出现逐步屏蔽了 C/S与B/S之间的隔阂。</p>
    <p>[0015]	RIA开发必须具备两个要素：富客户端（Rich Client)技术和服务器技术。RIA的 服务器技术从现有的服务器技术继承而来，并且技术相差无几，它主要为富客户端提供一</p>
    <p>4种连接机制，支持数据库应用开发、双向数据通信、实时数据通信。而RIA的核心则是富客 户端技术。RIA利用具有很强交互性的富客户端技术来为用户提供一个更高和更全方位的 网络体验。富客户端技术使创建RIA成为可能，它提供一个运行时的环境以承载被编译的 客户端应用程序（该客户端应用程序是一个使用HTTP协议发布的文件）。客户端应用程序 采用一种安全的、可升级的、具有良好适应性的面向Web服务的驱动模型，异步连接到现有 的应用服务器上。</p>
    <p>[0016]	目前，还没有将RIA应用到电信传输网管系统的解决方案，对此进行进一步的研 究很有必要。</p>

    <p>[0017]	针对现有技术中存在的缺陷，本发明的目的在于提供一种基于RIA的高性能Web 传输网管系统及实现方法，所开发的Web传输网管系统具有高度互动性、丰富用户体验，以 及运行于Web浏览器上的强大客户端。</p>

    <p>[0026]	所述的数据访问层，为业务逻辑层提供统一的、透明的数据资源访问接口，该数据 资源访问接口用于实现：对存储于数据库中的数据记录进行增、删、改、查，</p>

    <p>[0058]	图1为本发明所述的基于RIA的高性能Web传输网管系统的系统架构图，该Web 传输网管系统包括：</p>
    <p>[0059]	设置于客户端的Web浏览器（Client Browser)，是富客户端应用程序（Web传输网 管客户端程序）运行时的承载工具，Web浏览器通过HTTP协议将富客户端应用程序从Web 服务器上下载到本地计算机，然后离线或在线运行该程序；</p>
    <p>[0060]	Web服务器端，包含富客户端应用程序、Web数据服务层、Web Services层、业务逻 辑层、数据访问层及系统集成层；其中：</p>
    <p>[0061]	所述的富客户端应用程序（Rich Client Application)，实现Web传输网管系统的 图形化用户界面和客户端数据处理，</p>
    <p>[0062]	所述的Web数据服务层（Web Data Provider Tier)，为富客户端应用程序提供数 据服务，数据的请求可以是同步或异步的，同时该层负责实时数据的推送，</p>
    <p>[0063]	所述的Web Services层（Web Services Tier)，为富客户端应用程序或外部系统提供基于 SOAP Web Service 或 REST Web Service 的数据服务，所述 SOAP 是 Simple Object Access Protocol，简单对象访问协议；所述 REST 是 R印resentational State Transfer, 表述性状态转移；</p>
    <p>[0064]	所述的业务逻辑层（Business Logic Tier)，负责网管系统的业务功能，并为Web 数据服务层和Web Services层提供数据，所述业务功能包括：拓扑管理、告警管理、性能管</p>
    <p>理、配置管理、安全管理，</p>
    <p>[0065]	所述的数据访问层（Data Access Tier)，为业务逻辑层提供统一的、透明的数据 资源访问接口，该数据资源访问接口用于实现：对存储于数据库中的数据记录进行增、删、 改、查，</p>
    <p>[0066]	所述的系统集成层（System Integration Tier)，负责与底层网管系统（网元级网 管系统）进行交互，并为业务逻辑层提供基础数据。</p>
    <p>[0067]	上述基于RIA的高性能Web传输网管系统中，所述的设置于客户端的Web浏览器 支持当前的各大主流浏览器；所述的富客户端应用程序采用了 RIA的富客户端技术进行开 发，该富客户端应用程序预先发布到Web服务器上，然后随用户的请求被设置于客户端的 Web浏览器下载到本地计算机来执行。</p>
    <p>[0068]	本发明还提供一种应用上述传输网管系统的基于RIA的高性能Web传输网管系统 实现方法，包括以下步骤：</p>
    <p>[0069]	a)利用富客户端技术开发富客户端应用程序，并发布到Web服务器上；</p>
    <p>[0070]	b)ffeb浏览器向Web服务器端发送HTTP请求，然后下载富客户端应用程序，并在 Web浏览器中运行富客户端应用程序：如果客户端首次运行富客户端应用程序，则需要安 装相应的插件，如：基于Silverlight的富客户端应用程序需要安装Silverlight插件，基 于Flex的富客户端应用程序需要安装Flash播放器插件，这些插件都是官方提供的，安全 可靠，且能在几分钟内即可下载、安装完成，无需额外的配置；</p>
    <p>[0071]	c)富客户端应用程序在Web浏览器中运行后，显示登录窗口，用户输入正确的用 户名、密码，登录成功后加载并显示Web传输网管系统的主界面，之后用户即可进行网管功 能操作；</p>
    <p>[0072]	d)用户登录成功的同时，该客户端在Web服务器端进行了注册，之后即可实时获 取Web服务器端推送的数据；</p>
    <p>[0073]	e)富客户端应用程序在实现网管功能时，通过多线程技术来充分利用本地计算机 的计算资源，通过内置类操作本地磁盘，通过隔离存储技术创建和维护安全客户端虚拟文 件系统；</p>
    <p>[0074]	f)在客户端运行的富客户端应用程序通过HTTP的请求与Web服务器端进行数据 通讯，请求返回的数据经富客户端应用程序处理后，调用UI (User Interface)线程来局部 刷新网管界面；</p>
    <p>[0075]	g)外部系统通过调用Web Services层的Web Service，来与Web服务器进行数据 通讯，所述外部系统为第三方运营维护系统，如派单系统。</p>
    <p>[0076]	所述步骤a)中，富客户端应用程序的设计过程如图2所示，包含以下步骤：</p>
    <p>[0077]	1)用户选择富客户端技术中的组件，或利用类库构建自定义组件；</p>
    <p>[0078]	2)加载富客户端应用程序设计所需的资源文件；</p>
    <p>8[0079]	3)利用上述组件构建图形用户界面，编写界面的描述文件；</p>
    <p>[0080]	4)编写界面的后台处理文件；</p>
    <p>[0081]	5)设计界面的样式；</p>
    <p>[0082]	6)编译源文件，连接成富客户端应用程序。</p>
    <p>[0083]	所述步骤a)中，设计的富客户端应用程序具有丰富的图形用户界面，支持对界面 元素的相关鼠标、键盘操作，所述鼠标、键盘操作包括拖拽、移动、多选、框选、界面缩放。</p>
    <p>[0084]	所述步骤d)中，实时数据的推送技术采用富客户端技术中的双工通讯和 XMLSocket通讯，或采用Ajax或HTTP长连接技术。所述XMLSocket是XML套接字；所述 Ajax 是 Asynchronous JavaScript and XML,异步 JavaScript 禾口 XML。</p>
    <p>[0085]	所述步骤e)中，富客户端应用程序的多线程是指：富客户端应用程序具有一个主 应用程序线程：UI线程，且富客户端应用程序具备同时使用开发者创建的其他线程或线程 池提供的其他线程的能力。多线程的使用可以让网管客户端享用本地计算机更多的计算资 源。多线程在传统的Web技术中是不具备的，而在富客户端技术中可以很好使用，但也有限 制，多线程的使用原则是需要保证UI线程的安全性。</p>
    <p>[0086]	所述步骤e)中，利用隔离存储技术，富客户端应用程序中的数据都可以保存到虚 拟文件系统的存储区中，这些数据将始终按用户设置在虚拟文件系统中隔离，所述虚拟文 件系统是根目录中的一个文件，或是根目录中的一个目录和文件树。利用隔离存储技术，从 而可以让富客户端应用程序充分、安全地享用本地计算机的存储资源，实现传统Web应用 程序无法实现的客户端个性化定制服务。</p>
    <p>[0087]	所述步骤f)中，在客户端运行的富客户端应用程序通过Web数据服务层或Web Services层来与Web服务器端进行数据通讯：Web数据服务层或Web Services层解析富客 户端应用程序发送的请求，调用业务逻辑层对应的方法来处理请求，然后将处理结果返回 给富客户端应用程序。</p>
    <p>[0088]	所述步骤f)中，在客户端运行的富客户端应用程序优先选择通过Web数据服务层 来访问Web服务器：采用富客户端技术或Ajax技术来同步或异步向Web数据服务层发送请 求。</p>
    <p>[0089]	所述步骤g)中，外部系统调用的Web Service是：SOAP Web Service或REST Web Service，数据的格式为Pox或Json，所述Pox是Plain Old XML，早期的XML格式；所述的 Json是JavaScript Object Notation，一种轻量级的数据交换格式。</p>
    <p>[0090]	本发明的有益效果在于：利用RIA架构和富客户端技术，让真正的B/S架构的高性 能Web传输网管的开发成为了一种可能。所构建的Web传输网管系统具有C/S架构应用程 序的复杂用户界面、高交互性和复杂消息处理能力，兼有B/S架构应用程序的应用、部署、 更新和维护的优点，支持并发处理，能够发送同步或异步的HTTP请求与Web服务器进行数 据通讯，实时接受Web服务器推送的数据，并且能够充分利用本地计算机的计算资源和存 储资源，为用户提供全方位的、丰富多彩的客户体验。</p>
    <p>[0091]	具体实施案例</p>
    <p>[0092]	本具体实施案例采用Silverlight作为富客户端开发技术，采用 Spring+Hibernate的Java架构作为服务器端开发技术，来构建RIA架构。</p>
    <p>[0093]	利用所述的RIA架构来构建Web传输网管系统，系统架构如图3，具体实现如下：[0094]	a) Web浏览器，采用当前主流浏览器，如IE、FireFox、Chrome、Opera等。下载 Web传输网管客户端（即富客户端应用程序)，然后在浏览器上运行和操作Web传输网管 的界面，为用户提供无差异的客户体验。首次运行Silverlight应用程序需要从微软下载 Silverlight 插件；</p>
    <p>[0095]	b) Web传输网管客户端，主要由Silverlight技术来构建丰富的图形化用户界面， 同时支持传统的Web界面。客户端后台数据处理采用多线程技术，由线程池统一管理；用户 定制数据及部分客户端临时数据则利用隔离存储技术进行读写；</p>
    <p>[0096]	c)ffeb数据服务层，采用Silverlight的WebClient调用该层的Servlet异步获取 数据，使用Ajax的XMLHTTPRequest调用该层的Servlet同步或异步获取数据。采用基于 HTTP长连接的DWR框架和ActiveMQ实现告警数据的实时推送；</p>
    <p>[0097]	d)ffeb Service 层，采用基于 Json 格式的 Soap Web Service ；</p>
    <p>[0098]	e)业务逻辑层，利用Spring框架实现传输网管系统的业务功能；</p>
    <p>[0099]	f)数据访问层，采用Hibernate框架提供统一的、透明的数据资源访问接口。</p>
    <p>[0100]	所述Web传输网管系统实施案例，除了本发明具有的特点外，还具有跨Web浏览 器、跨操作系统、跨Web服务器的优势，在只需简单修改Hibernate配置文件的情况下，支持 Oracle、Informix、Sql Server和MySql等数据库间的移植，让本发明在实际应用中变得更 为广泛。</p>
    <p>[0101]	Silverlight开发Web传输网管客户端程序的具体步骤如下：</p>
    <p>[0102]	1)用户选择Silverlight内置的控件，或利用Silverlight类库构建自定义控 件；</p>
    <p>[0103]	2)编译自定义控件工程，并生成DLL文件，便于以后重用；</p>
    <p>[0104]	3)加载Web传输网管客户端设计所需的资源文件（如png、jpg文件等）；</p>
    <p>[0105]	4)添加Silverlight动态库DLL和自定义控件动态库DLL，利用上述控件构建图 形用户界面，利用XAML语言编写界面的描述文件；</p>
    <p>[0106]	5)利用.NET语言(C#、VB. NET、JavaScript等）编写界面的后台处理程序；</p>
    <p>[0107]	6)设计界面的样式；</p>
    <p>[0108]	7)编译源文件，连接成Web传输网管客户端程序（XAP文件）；</p>
    <p>[0109]	8) Web浏览器发送HTTP请求，下载所述的Web传输网管客户端程序（XAP文件)， 并在其内运行。</p>
    <p>[0110]	图4为Web传输网管用户登录流程，具体步骤如下：</p>
    <p>[0111]	l)ffeb浏览器向Web服务器端发送HTTP请求，然后下载上述的Web传输网管客户 端程序，并在浏览器中运行；</p>
    <p>[0112]	2)Web传输网管客户端程序启动登录窗口 ；</p>
    <p>[0113]	3)用户输入用户名、密码；</p>
    <p>[0114]	4) Web传输网管客户端程序首先验证输入的合法性，如果合法，进入下一步，否则， 返回3)；</p>
    <p>[0115]	5)Web传输网管客户端程序将密码加密，然后与用户名及配置数据组合成参数，发 送HTTP请求到上述Web数据服务层；</p>
    <p>[0116]	6) Web数据服务层解析客户端请求，调用上述业务逻辑层的用户验证接口进行用</p>
    <p>10户验证，并将验证结果返回给Web传输网管客户端程序。如果验证成功的话，还将该客户端 在Web服务器上进行注册，之后即可向该客户端推送实时数据；</p>
    <p>[0117]	7) Web传输网管客户端程序获得验证结果，如果验证成功，加载并显示网管主界 面；如果失败，重新进入3)。</p>
    <p>[0118]	当然，本发明还有很多其他实施案例，在不背离本发明精神和实质的情况下，熟悉 不同RIA技术的技术人员可以根据本发明选择相应的富客户端技术与服务器端技术来开 发Web传输网管系统，但这些改变都应属于本发明所附的专利要求的保护范围。</p>
