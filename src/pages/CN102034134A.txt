    <div class="abstract">本发明揭示了一种集成项目管理系统及方法，架构于Web服务器端，并通过局域网络与客户端电脑以网络传输协议通信相连，其特征在于：所述Web服务器端的项目管理系统由项目计划建制模块、项目数据收集模块、项目度量分析模块及组织财富库模块集成化构建。通过用户选择项目规模，系统自动分配控管项和管理流程，按项目计划提示用户及时提交项目信息的数据，并结合历史数据进行项目度量分析，得出项目进度和状态。该项目管理系统能够满足CMMI对于项目计划、项目控管及相关过程域的要求，简化项目管理实施，规范项目管理运作，提高项目成员工作效率。</div>

    <p>集成项目管理系统及方法技术领域[0001]	本发明涉及一种软件能力成熟度集成模型（CMMI)，尤其涉及一种关于该模型中项 目计划\项目控管及其相关过程域的集成项目管理系统，以及利用该系统进行项目管理的 方法。背景技术[0002]	随着IT产业的不断进步，越来越多的软件公司或部门注重项目运作的规范化，所 以很多组织根据自身的状况实施 CMMI (Capability Maturity Model htegration)。CMMI 是目前世界公认的软件产品进入国际市场的通行证，它不仅仅是对产品质量的认证，更是 一种软件过程改善的途径。CMMI有25个过程域，其中项目控管、项目计划过程域对于调整 和控制项目成本、效率都很有帮助。然而这部分过程的具体实施需要收集很多方面的信息， 比如成本、人力、效益、时间控制、缺陷、风险等等，而过程中某些流程的触发也需要人员的 告知或稽核。CMMI本身是一种模型，一套方法，而具体实施每个公司都有所不同，通常都需 要撰写大量的文档，付出较多的人力物力去实施，这就造成CMMI难实施的状况。发明内容[0003]	鉴于上述现有技术的缺陷，本发明的目的旨在提供一种集成项目管理系统及方 法。基于Web服务器端及其通过局域网络通信相连的客户端电脑，并基于用户人员的签合 流程，满足CMMI对项目计划、项目控管及相关过程域的控管要求。[0004]	本发明的第一个目的，将通过以下技术方案来实现：[0005]	集成项目管理系统，架构于Web服务器端，并通过局域网络与客户端电脑以网络 传输协议通信相连，其特征在于：所述Web服务器端的项目管理系统集成化构建，包括：[0006]	一项目计划建制模块，通过客户端电脑访问及显示计划建制界面，并在用户作出 规模选择后自动配置相应的控管项以及管理流程；[0007]	&#8212;项目数据收集模块，根据项目计划通过客户端电脑显示对应项目的数据收集页 面，实时采集项目进度及数据信息；[0008]	一项目度量分析模块，基于Web服务器端的处理器，用于对数据收集模块采集到 的数据进行计算、分析，以度量项目各项指标；[0009]	一组织财富库模块，用于存储管理系统所有进行中或是已完成的完整项目工程文 件以及组织规范和模板。[0010]	进一步地，前述的集成项目管理系统，其中所述Web服务器端与客户端电脑之间 通信连接架构的是基于TCP/IP、NETBEUI或IPX/SPX传输协议的有线局域网，或基于WLAN 的QoS协议的无线局域网。[0011]	进一步地，前述的集成项目管理系统，其中客户端电脑至少为两台，且位于同一网 域的同一工作组内，项目数据收集模块采集所述工作组的项目进度及数据信息；或者两台 客户端电脑位于同一网域的两个相对独立的工作组内，项目数据收集模块采集并汇总所述两个工作组的项目进度及数据信息。[0012]	进一步地，前述的集成项目管理系统，其中所述组织财富库模块为一台用于存放 项目文件的独立服务器。[0013]	本发明的第二个目的，将通过以下技术方案来实现：[0014]	集成项目管理方法，通过权利要求1所述的集成项目管理系统实现，所述项目管 理方法的特征在于包括步骤：[0015]	I、用户在客户端电脑上通过局域网络访问Web服务器端的项目计划建制模块并 选定项目规模后，项目管理系统自动配置相应的控管项及管理流程，显示于客户端电脑的 计划建制页面上；[0016]	II、系统根据项目计划并结合项目管理流程，在客户端电脑上显示对应的项目数 据收集页面，并通信知会并控管对应的用户签合信息；[0017]	III、项目度量分析模块对所收集到的项目信息数据进行度量报表和图表分析，其 中所述度量报表为参照历史数据计算项目各指标的度量值报表；图表分析为根据收集到的 项目信息数据，结合对应度量值绘制分析图表，以度量项目状态并给出强、弱项分析结果；[0018]	IV、将所有进行中或是已完成的完整项目工程文件以及组织规范和模板分类存入 文件服务器中，用户对各文件标注作为查询调用条件的对应属性，形成一项目文件数据库。[0019]	进一步地，前述的集成项目管理方法，其中步骤I中所述项目管理系统自动配置 相应的控管项及管理流程后，还包括用户透过计划建制页面对控管项进行裁减、增加的流 程。[0020]	更进一步地，前述的集成项目管理方法，其中步骤I中所述项目的控管项包括项 目运行所需的成本、人力、时间，以及项目需要克服的缺陷与风险。[0021]	进一步地，前述的集成项目管理方法，其中步骤II中，对于超出管理流程所预设 的时间而未获得用户的签合信息情况，项目数据收集模块以邮件形式向对应用户发出提醒。[0022]	更进一步地，前述的集成项目管理方法，对于所述超出管理流程预设时间达到12 小时&#12316;M小时的情况，项目数据收集模块向对应客户端电脑发送机催信号。[0023]	应用本发明集成项目管理系统及方法，其有益效果体现在：[0024]	该项目管理系统能够满足CMMI对于项目计划、项目控管及相关过程域的要求，简 化了项目管理实施，规范了项目管理运作，提高了项目成员工作效率；项目度量分析有助于 改进项目，以达到项目持续优化的目标；组织财富库有助于积累知识财富，分享优秀经验， 并有助于项目人员参考学习。附图说明[0025]	图1是本发明集成项目管理系统的拓扑结构示意图。 具体实施方式[0026]	为使本发明集成项目管理系统、方法的技术特征及应用后的显著效果，以下便结 合附图，对本发明技术方案作如下详细描述。[0027]	如图1所示，从结构上来看：该集成项目管理系统，架构于Web服务器端1，并通过局域网络与客户端电脑31&#12316;33以网络传输协议通信相连，其主要包括项目计划建制模块、 项目数据收集模块、项目度量分析模块及组织财富数据库四部分组成，集成化构件于Web 服务器端。用户使用客户端电脑通过网络而连结至集成项目管理系统，以建立项目开发计 划，并有效管理项目、任务的运作；基于签合的工作流程实现流程控制；最底层具有一数据 库，供储存资料。[0028]	具体来看，项目计划建制是指用户在系统中选择符合自身条件的项目规模后，系 统会自动配置相应的控管项以及管理流程，这些都会显示于计划建制页面。这里的项目控 管项是指（成本、人力、效益、时间、缺陷、风险等项目因素），控管项可以根据自身需要进行 裁减，需写明原因。用户被要求根据页面提示建制计划，建制后经过对应审核流程后才生 效。项目计划建制模块是项目数据收集模块的前置条件。[0029]	项目数据收集是指对于已经建制的项目，系统会根据项目计划，显示对应的项目 数据收集页面，并结合项目管理流程，Mail知会并控管对应角色的用户签合对应的信息，对 于迟迟未能签合的用户，系统会发机催信息以提醒。[0030]	项目度量分析主要包含度量报表和图表分析两部分，度量报表是根据历史数据定 期或手动计算各个指标的度量值报表；图表分析是根据已收集的各类项目数据，结合对应 度量数据，绘制出分析图表，以度量项目状态，同时还给出强项弱项分析结果。[0031]	组织财富库模块是指保存各种项目文件，就像一个资料库，所以称之为财富库，包 含各类组织规范、模版等规范类文件，以及实际项目的工程文件，用户需对各文件标注对应 属性，这些文件最终是存放在一台独立的文件服务器2，以提供足够大的存储空间。用户可 以用这些属性条件快速查找到所需文件。组织规范、文件模版、重要项目文件等需要经过特 定的签合流程才能提交到组织财富库。[0032]	上述技术方案可以优化设计为：[0033]	一方面，该Web服务器端与客户端电脑之间通信连接架构的是基于TCP/IP、 NETBEUI或IPX/SPX传输协议的有线局域网，或基于WLAN的QoS协议的无线局域网。[0034]	另一方面，该客户端电脑至少为两台，且位于同一网域代表部门一的同一工作组 内，项目数据收集模块采集所述工作组的项目进度及数据信息，或者该客户端电脑至少为 两台，且位于同一网域代表部门一和部门二的两个相对独立的工作组内，项目数据收集模 块采集并汇总所述两个工作组的项目进度及数据信息。[0035]	本发明服务器端需要至少两台服务器，一台作为应用服务器1，一台或多台作为数 据和文件服务器2，均要求装配Microsoft Windows 2000 Server(任何版本）的操作平台。[0036]	本发明的应用软件布署在应用服务器1上以处理前端需求，该应用服务器要求 128M以上内存，250G以上硬盘空间，系统要求安装IIS 5.0以上互联网信息服务和5. 0以 上网络浏览器；另外如果同时作为流程签核服务器，还需要安装IBM Lotus Domino R6。[0037]	文件服务器2存储项目数据和文件，要求64M以上内存，2G以上硬盘空间，并需求 安装数据库软件SQL Server 2000。[0038]本发明客户端要求	Microsoft Windows NT 4. 0、Windows 2000(所有版本）、 Windows Me或Windows 98的操作平台；要求安装IIS 5. 0以上互联网信息服务和5. 0以上 网络浏览器。如果需要流程签核的用户客户端，还需要安装IBM Lotus Notes R6客户端。[0039]	从管理方法来看，实现项目集成管理的步骤包括：6[0040]	I、用户在客户端电脑上通过局域网络访问Web服务器端的项目计划建制模块并 选定项目规模后，项目管理系统自动配置相应的控管项及管理流程，显示于客户端电脑的 计划建制页面上；[0041]	II、系统根据项目计划并结合项目管理流程，在客户端电脑上显示对应的项目数 据收集页面，并通信知会并控管对应的用户签合信息；[0042]	III、项目度量分析模块对所收集到的项目信息数据进行度量报表和图表分析，其 中所述度量报表为参照历史数据计算项目各指标的度量值报表；图表分析为根据收集到的 项目信息数据，结合对应度量值绘制分析图表，以度量项目状态并给出强、弱项分析结果；[0043]	IV、将所有进行中或是已完成的完整项目工程文件以及组织规范和模板分类存入 文件服务器中，用户对各文件标注作为查询调用条件的对应属性，形成一项目文件数据库。[0044]	上述技术方案可以优化设计为：对于所述超出管理流程预设时间达到12小时&#12316;
