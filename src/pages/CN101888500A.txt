    <p>背景技术</p>
    <p>[0002]	数字电视正在全国各地推广并会最终替代模拟电视，但由于历史和国情，各个城 市和地区的数字电视运营商没有在完全统一的标准和规定下进入这一领域，各地发送数字 电视信号的方式并不完全相同，包括数字电视的PSI (Programservice information节目服 务信息）和SI (Service information服务信息）的放置频点和组织形式也可能不一样。</p>
    <p>[0003]	就节目列表的排序而言，因为模拟电视一个频道的带宽内只能放一套节目，其可 能的一种排序方式就是按照物理频道从小到大排列，则对应的接目镜列表中的节目也就是 按照这个顺序排列了 ；而数字电视一个频点可以放置多套节目，这些节目对应节目列表的 排序方式都由各个运营商自行定义，为接收终端提供了一种有序的节目列表排序方式，但 这仅适合机顶盒产业一批订单就针对一个运营商的研、产、销模式。对于数字电视一体机 而言，产品面向全国市场，且通过零售市场销售，因此在产品销往不同的城市、不同地区时， 面对国内上百家有线电视网络运营商，无法适应各个有线电视网络的不同节目列表排序方 法。</p>
    <p>[0004]	因此，需要提供一种数字电视一体机节目列表排序的方案，能够适应不同有线电 视网络服务信息的发送方式的区别，给用户提供一种适合于各个有线电视网络的节目列表 排序方式。</p>
    <p>发明内容</p>
    <p>[0005]	本发明的目的在于提供一种数字电视一体机及其实现节目列表的方法，旨在解决 数字电视一体机节目列表中的节目排序的问题。</p>
    <p>[0006]	本发明是这样实现的，一种数字电视一体机实现节目列表的方法，所述的方法包 括：</p>
    <p>[0007]	a、在数字电视节目信息中查找节目列表的排序信息，如果查找到节目列表的排序 信息，转步骤b，否则转步骤C;</p>
    <p>[0008]	b、根据所查找到的节目列表排序信息生成节目列表；</p>
    <p>[0009]	C、按照用户选中的节目列表排序方式生成节目列表。</p>
    <p>[0010]	其中步骤a之前还包括：</p>
    <p>[0011]	aO、启动数字电视一体机搜台功能，从有线电视网络获取所述的节目信息。</p>
    <p>[0012]	其中步骤a具体包括：</p>
    <p>[0013]	al、在所述在节目信息中查找Logical_channel_number_descriptor描述符，如 果查找到所述Logical_channel_number_descriptor描述符，从中获取LCN作为节目列表 的排序信息，转步骤b，否则转步骤C。[0014]	所述的在所述节目信息中查找Logical_channel_number_descriptor描述符具 体包括：</p>
    <p>[0015]	查找描述符值为0xE2或者0x83的描述符，确认其对应的描述符是否为Logical channel_number_descriptor 描述符中。</p>
    <p>[0016]	其中步骤b具体包括：</p>
    <p>[0017]	bl、根据所述的LCN生成节目列表，所述的节目列表中各个节目按照LCN从小到大 或者从大到小的方式排列。</p>
    <p>[0018]	本发明还提供了 一种数字电视一体机，所述的数字电视一体机包括MCU、排序信息 查找模块、存储器和节目列表生成模块，其中：</p>
    <p>[0019]	所述的排序信息查找模块，用来从节目信息中查找节目列表排序信息，将查找到 的节目列表排序信息传输到所述MCU ；</p>
    <p>[0020]	所述的MCU用来将所述的节目列表排序信息传输到所述节目列表生成模块，控制 所述节目列表生成模块按照查找到的节目列表排序信息生成节目列表；</p>
    <p>[0021]	所述节目列表生成模块用来从所述存储器中调用所述节目列表排序信息对应的 排序算法参数，根据所述MCU传输的节目列表排序信息及其对应的算法参数生成节目列 表；</p>
    <p>[0022]	所述的存储器用来存储所述节目列表排序信息对应的排序算法参数。</p>
    <p>[0023]	所述MCU还用来在所述排序信息查找模块查找不到节目列表排序信息时，控制所 述节目列表生成模块根据用户选中的节目列表排序方式生成节目列表。</p>
    <p>[0024]	所述的电视机还包括与MCU连接的显示模块、所述的MCU还用来从所述存储器中 调用节目列表排序方式选择菜单送所述显示模块显示，根据用户从所述节目列表排序方式 选择菜单中选中的排序方式控制所述节目列表生成模块生成节目列表。</p>
    <p>[0025]	所述MCU还用来在所述排序信息查找模块查找不到节目列表排序信息时，控制所 述节目列表生成模块按照默认的节目列表排序方式生成节目列表。</p>
    <p>[0026]	所述的节目列表排序信息具体为Logical_channel_number_descriptor描 述符中携带的LCN，所述节目列表生成模块用来根据所述L0gical_channel_number_ descriptor描述符中携带的LCN生成节目列表，所述的节目列表中各个节目按照LCN从小 到大或者从大到小的方式排列。</p>
    <p>[0027]	本发明克服现有技术的不足，针对数字电视一体机的特点，在系统搜台时在信号 中检测排序信息，如果找到对应的排序信息，在搜台后直接按照这种排序方式组织节目列 表即可；如果没有找到对应的排序信息，则提供几种可选的节目列表排序方式供用户选择， 按照用户选择的排序方式组织节目列表。本发明提供的技术方案使得数字电视一体机可以 自动检测当地有线电视网络运营商所使用的节目列表排序方式，根据该节目列表排序方式 组织节目列表，如果无法检测到当地有线电视网络运营商所使用的节目列表排序方式，也 可以提供多种排序方式给用户选择，从而使得系统可以建立一种有序的节目列表，使得用 户更加快捷、方便的使用数字电视一体机产品。</p>

    <p>[0028]	图1是本发明实施例提供的节目列表排序方式选择菜单示意图；[0029]	图2是本发明实施例流程图；</p>
    <p>[0030]	图3是本发明实施例提供数字电视一体机原理图。</p>
    <p>具体实施方式</p>
    <p>[0031]	本发明提供的技术方案是：针对数字电视一体机的特点，在系统搜台时在信号中 检测排序信息，如果找到对应的排序信息，在搜台后直接按照排序信息所确定的排序方式 组织节目列表即可；如果没有找到对应的排序信息，则提供可选的节目列表排序方式供用 户选择，按照用户选择的排序方式组织节目列表。</p>
    <p>[0032]	为了使本发明的目的、技术方案及优点更加清楚明白，以下结合附图及实施例，对 本发明进行进一步详细说明。应当理解，此处所描述的具体实施例仅仅用以解释本发明，并 不用于限定本发明。</p>
    <p>[0033]	本发明实施例中，数字电视一体机搜台功能启动后，系统将从有线电视网络获取 节目信息（PSI和Si)，对节目信息进行解析，在解析后的节目信息中查找节目列表的排序 信息，如果能够找到节目列表的排序信息，则在搜台后直接按照排序信息对应的排序方式 组织节目列表即可；如果没有找到排序信息，则提供可选的节目列表排序方式供用户选择， 然后按照用户选择的排序方式组织节目列表。</p>
    <p>[0034]	具体来说，系统可以根据对NIT (网络信息表）的解析结果，因为Logical_ channel_number_descriptor(LCN逻辑频道序号)描述符的值可能为0xE2或者0x83，也 可能是其他的值，则在解析后的服务信息中查找描述符值为0xE2或者0x83的描述符是 否为Logical_channel_number_descriptor(LCN逻辑频道序号)描述符，如果能够查找到 Logical_channel_number_descriptor 描述符，该描述符中包括了每个 Service id(业务 识别号）及其对应的Logicalcharmelnumber (LCN逻辑频道序号），则说明有线电视网络 运营商确定的节目列表的排序方式为按照LCN排序，由于Service id是用来标识各个节目 的，根据Service id可以查找到各个节目的具体信息，如节目名称、播放频道、播放起始时 间等等，则可以按照逻辑频道号从小到大的顺序排列节目，形成一个有规律的节目列表。</p>
    <p>[0035]	如：LCN 为 1,对应的 service id = 3, service name = cctv-1 ；</p>
    <p>[0036]	LCN 为 2,对应的 service id = 5, service name = cctv-2 ；</p>
    <p>[0037]	LCN 为 3,对应的 service id = 10, service name = sztv-1。</p>
    <p>[0038]	LCN从小到大排序后为：1、2、3，则节目列表就排列为：</p>
    <p>[0039]	cctv-1 ；</p>
    <p>[0040]	cctv-2 ；</p>
    <p>[0041]	sztv-lo</p>
    <p>[0042]	如果系统根据描述符值为0xE2或者0x83没有找到L0giCal_Channel_number_ descriptor描述符，则可以按照其他可能的描述符值进行查找，如果根据各种可能的描述 符值都没有查找到描述符Logical_channel_number_descriptor，此时可以提供如图1所 示的节目列表排序方式选择菜单，该菜单中提供了三种排序方式选项：按照Service id排 序；按照物理频道排序；按照搜索顺序排序。用户可以通过数字电视一体机遥控器或者控 制面板从图1所示的菜单中选中一个选项，则系统将根据用户选中的选项组织节目列表， 该节目列表种各个节目按照用户选中排序方式排列。[0043]	在提供如图1所示的节目列表排序方式选择菜单给用户选择之前，系统可以先按 照Service id从小到大的顺序来组织节目列表，将按照这种排序方式组织的节目列表作为 系统默认的节目列表。如果用户不调用如图1所示的菜单重新选择节目列表排序方式，则 直接采用默认的节目列表，如果用户重新选择，则按照用户选择的排序方式组织节目列表。</p>
    <p>[0044]	本发明实施例流程图如图2所示，包括如下步骤：</p>
    <p>[0045]	1、启动数字电视一体机搜台功能；</p>
    <p>[0046]	2、系统将从有线电视网络获取节目信息，对节目信息进行解析；</p>
    <p>[0047]	3、根据有可能采用的描述符值，在解析后的节目信息中查找Logicalcharmel number_descriptor描述符，从中获取对应的排序信息；</p>
    <p>[0048]	4、所有可能采用的描述符值搜索完毕？如果是，转步骤5，否则转步骤2 ；</p>
    <p>[0049]	5、是否找到排序信息？如果是，转步骤6，否则转步骤7 ；</p>
    <p>[0050]	6、根据查找到的排序信息确定有线电视网络运营商的节目列表排序方式，按照该 排序方式组织节目列表；</p>
    <p>[0051]	7、按照Service id从小到大的顺序来组织节目列表，将按照这种排序方式组织的 节目列表作为系统默认的节目列表；</p>
    <p>[0052]	8、用户调用节目列表排序方式选择菜单，从该菜单中选择节目列表排序方式；</p>
    <p>[0053]	9、根据用户选择的节目列表排序方式组织节目列表。</p>
    <p>[0054]	本发明实施例提供的数字电视一体机的原理图如图3所示，除了现有数字电视一 体机所具备的功能模块之外（图未示出），还包括节目信息解析模块、MCU(微处理器）、控 制信号接收模块、存储器、排序信息查找模块、节目列表生成模块和显示模块。</p>
    <p>[0055]	其中，控制信号接收模块与MCU相连，用来接收数字电视一体机遥控器或者控制 面板发送的控制信号，如搜台控制信号、调用节目列表排序方式选择菜单控制信号以及从 中选中一个选项的控制信号等，并将接收的控制信号送MCU ；</p>
    <p>[0056]	MCU与存储器、节目信息解析模块、排序信息查找模块、显示模块和节目列表生模 块相连，用来控制系统的运行；如用来对控制信号接收模块传输的控制信号进行解析，根据 解析结果启动数字电视一体机搜台功能，从存储器中调用存储的节目列表排序方式选择菜 单送显示模块显示，接收用户从该菜单中选中某一选项的控制指令，控制节目列表生成模 块按照用户选中的排序方式生成节目列表；控制节目信息解析模块对有线电视网络传输的 节目信息进行解析，控制节目列表生成模块按照指定的排序方式生成节目列表；将排序信 息查找模块查找到的节目列表排序信息传输到节目列表生成模块；</p>
    <p>[0057]	节目信息解析模块与排序信息查找模块相连，用来对有线电视网络传输的节目信 息进行解析，将解析结果传输到排序信息查找模块；</p>
    <p>[0058]	排序信息查找模块与MCU相连，用来从节目信息解析模块传输的解析结果中查找 节目列表的排序信息，将查找到的节目列表排序信息传输到MCU，如用来从NIT中查找描 述符值为0xE2或者0x83的Logical_channel_number_descriptor描述符，从查找到的 Logical_channel_number_descriptor描述符中获取LCN及其对应的Service id后传输到 MCU ；</p>
    <p>[0059]	节目列表生成模块与存储器相连，用来生成节目列表，当排序信息查找模块查找 到排序信息时，MCU向节目列表生成模块传输节目列表排序信息，节目列表生成模块从存储器中调用对应的排序算法参数，根据该排序信息组织节目列表，生成与该排序信息对应的 节目列表；当排序信息查找模块没有查找到排序信息时，MCU指示节目列表生成模块根据 用户选择的节目列表排序方式生成节目列表，节目列表生成模块从存储器中调用对应的排 序算法参数，根据用户选择的节目列表排序方式组织节目列表，生成与用户选择的节目排 序方式对应的节目列表；</p>
    <p>[0060]	存储器用来存储与各种节目列表排序方式对应的算法参数，供节目列表生成模块 调用。</p>
    <p>[0061]	以上所述仅为本发明的较佳实施例而已，并不用以限制本发明，凡在本发明的精 神和原则之内所作的任何修改、等同替换和改进等，均应包含在本发明的保护范围之内。</p>
