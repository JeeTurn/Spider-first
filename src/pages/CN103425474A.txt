    <p>用于获取屏幕快照中的内容的方法和设备</p>
    <p>技术领域</p>
    <p>[0001]	本发明的实施例总体上涉及计算机技术，更具体地，涉及一种用于获取屏幕快照中的内容的方法和设备。</p>
    <p>背景技术</p>
    <p>[0002]	在网络会议中，桌面共享、应用共享和文档共享等等是网络会议系统的常见功能。通过这些功能，演讲者和参与者之间能够实现直观、及时的交互。通常，演讲者会将自己的演讲内容通过网络会议服务器向参与者进行共享和传达，参与者在自己附近的屏幕上呈现的屏幕快照来观看演讲者共享和传达的内容。</p>
    <p>[0003]	演讲者传达的内容通常都是以屏幕快照或者图片的表现形式向参与者进行展示，但是现有的网络会议软件或者网站没有为参与者提供获取屏幕快照中的内容的功能。目前，无论采用哪种网络会议工具，当参与者对屏幕快照中的某些内容很感兴趣并且希望得到这些内容时，参与者一般是无法对共享的屏幕快照中的内容进行复制、保存等操作的。在演讲者进行讲解的过程中，屏幕快照可以是随时更新的。在这种情况下，参与者往往从演讲者那里通过直接拷贝或者电子邮件发送等手段来得到演讲者传达的内容，或者自己对屏幕快照进行截屏、然后手工记录下自己需要的内容，或者通过对截屏后的屏幕快照进行识别(例如采用光学字符识别(OCR)等方式)来得到自己需要的内容，等等。</p>
    <p>[0004]	然而，无论采用上述哪种方法来获取屏幕快照中的内容，都会给网络会议的参与者(下文也称为“用户”)带来很多额外的工作量和不便，降低了参与者参会即时性和效率，显著影响了参与者的与会体验。</p>
    <p>发明内容</p>
    <p>[0005]	鉴于现有技术中存在的上述问题，本领域中需要一种能够获取屏幕快照中的内容的解决方案。为此，本发明的实施例提供一种获取屏幕快照中的内容的方法和设备。</p>
    <p>[0006]	根据本发明的一个方面，提供了一种用于获取屏幕快照中的内容的方法。该方法可以包括:获取与屏幕快照相关联的内容组织信息，其中内容组织信息包括与屏幕快照中的内容相关联的一个或多个数据对象的结构和数据；以及基于内容组织信息确定屏幕快照中的内容。</p>
    <p>[0007]	根据本发明的另一个方面，提供了一种用于获取屏幕快照中的内容的设备。该设备可以包括:获取单元，配置用于获取与屏幕快照相关联的内容组织信息，其中内容组织信息包括与屏幕快照中的内容相关联的一个或多个数据对象的结构和数据；以及确定单元，配置用于基于内容组织信息确定屏幕快照中的内容。</p>

    <p>[0116]	表I示出了根节点或中间节点的数据结构的一种形式:</p>
    <p>[0117]	表I根节点或中间节点的数据结构</p>
    <p>[0118]</p>
    <p>ID X Y Z Height Width FirstChild NextSibling UpdateFlag</p>
    <p>[0119]其中:</p>
    <p>[0120]	ID是数据对象标识；</p>
    <p>[0121 ] X是数据对象在快照提供方屏幕上左上角点的横坐标值；</p>
    <p>[0122]	Y是数据对象在快照提供方屏幕上左上角点的纵坐标值；</p>
    <p>[0123]	Z是数据对象在快照提供方屏幕上左上角点的层次坐标值，用于在存在窗口覆盖的情况下指定窗口层次；</p>
    <p>[0124]	Height是数据对象所属区域的高度；</p>
    <p>[0125]	Width是数据对象所属区域的宽度；</p>
    <p>[0126]	FirstChild是此根节点的第一个孩子节点,或者是中间节点的第一个数据对象孩子节点；</p>
    <p>[0127]	NextSibling是此中间节点的下一个相邻的兄弟节点,根节点此字段为空。</p>
    <p>[0128]	UpdateFlag是在两个相邻的刷新周期间此节点是否发生过变化的标记,取值可以用2个比特来表示，00表示无变化，01表示只发生了大小和位置变化，10表示此节点内容发生了变化，11表示位置和内容均发生变化。</p>
    <p>[0129]	表2示出了叶节点的数据结构的一种形式:</p>
    <p>[0130]	表2叶节点的数据结构</p>
    <p>[0131]</p>
    <p>  ID X Y Z Height Width DataType Data NextSibling UpdateFlag</p>
    <p>[0132]其中:[0133]	ID是数据对象标识；</p>
    <p>[0134]	X是数据对象对应区域，左上角点相对于应用窗口的横坐标值；</p>
    <p>[0135]	Y是数据对象对应区域，左上角点相对于应用窗口的纵坐标值；</p>
    <p>[0136]	Z是数据对象对应区域，左上角点在应用窗口中所属的层次值；</p>
    <p>[0137]	Height是数据对象所属区域的高度；</p>
    <p>[0138]	Width是数据对象所属区域的宽度；</p>
    <p>[0139]	DataType是数据对象的数据类型，比如文字、图片等；</p>
    <p>[0140]	Data是数据对象的数据，例如构成文字、图片等的数据，后续将按照DataType指定的数据类型实现编解码。</p>
    <p>[0141]	Next Sibling是此数据对象节点的下一个相邻的兄弟节点。</p>
    <p>[0142]	UpdateFlag是两个相邻的刷新周期间此数据对象是否发生过变化的标记，取值可以用2个比特来表示，00表示无变化，01表示只发生了大小和位置变化，内容不变，10表示只是数据对象里面的内容发生了变化，大小和位置不变11表示位置和内容均发生变化。</p>
    <p>[0143]	根据本发明的实施例，内容组织信息的至少一部分可以包括所述内容组织信息中的一个或多个数据对象的结构和数据。此时，内容组织信息的至少一部分是内容组织信息的全部，既包括具有如表I所示的数据结构的根节点或中间节点的所有信息，也包括具有如表2所示的数据结构的叶节点的所有信息。</p>
    <p>[0144]	根据本发明的实施例，内容组织信息的至少一部分也可以仅包括该内容组织信息中的一个或多个数据对象的结构，而不包括该一个或多个数据对象的数据。此时，内容组织信息的至少一部分不是内容组织信息的全部，而是仅包括数据对象的结构，也即包括:具有如表I所示的数据结构的根节点或中间节点的所有信息、以及具有如表2所示的数据结构的叶节点的除“Data”字段之外或者除“Data”和“DataType”字段之外的所有信息。可见，内容组织信息的至少一部分不包括数据对象的数据，即，关于“Data”字段或者“Data”和“DataType”字段的信息。通常来说，由于数据对象的数据&#8212;&#8212;即“Data”字段的信息&#8212;&#8212;对应的数据量较大，因此在仅向用户传送内容组织信息的至少一部分的情况下，可以有效降低通信带宽占用率并提高传输速率。</p>
    <p>[0145]	在步骤S902，向用户发送内容组织信息中的一个或多个数据对象的结构和屏幕快照。</p>
    <p>[0146]	在本实施例中，向用户发送的除了屏幕快照之外，还要发送内容组织信息的至少一部分，也即内容组织信息中的一个或多个数据对象的结构。如上所述，内容组织信息中的一个或多个数据对象的结构例如可以包括:具有如表I所示的数据结构的根节点或中间节点的所有信息、以及具有如表2所示的数据结构的叶节点的除“Data”字段之外或者除“Data”和“DataType”字段之外的所有信息。</p>
    <p>[0147]	在步骤S903，接收与用户希望得到的屏幕快照中的内容有关的选择信息。</p>
    <p>[0148]	用户可以通过多种方式来从所看到的屏幕快照中选择希望得到的内容。例如，用户可以利用点击鼠标并进行拖拉来选定屏幕快照中的一块区域，该区域中有用户希望得到的内容；用户也可以通过首先利用鼠标点击来选中一个图片或一段音频，来指定该图片或音频为自己希望得到的内容。可以理解，上述示例仅仅是示意性的，并不应作为对本发明的限制。[0149]	在用户通过点击鼠标然后拖拉来选定在屏幕快照图片上所要获取的区域的情况下，用户处的设备可以记录选定区域的原始位置信息(例如，横坐标Ox、纵坐标Oy)和尺寸信息(例如，高度Oh、宽度Ow)。根据本发明的实施例，选择信息可以包括用户希望得到的内容的位置信息，例如上述原始位置信息和尺寸信息。此时，在步骤S903可以接收到包括原始位置信息和尺寸信息的选择信息。</p>
    <p>[0150]	在步骤S904，基于选择信息，从一个或多个数据对象的数据中确定与用户希望得到的屏幕快照中的内容相关联的数据。</p>
    <p>[0151]	如上所述，步骤S903接收到的选择信息包括原始位置信息和尺寸信息。然后，可以在步骤S904将原始位置和尺寸信息可以被换算为与快照提供者的屏幕相对应的更新位置和尺寸信息，并根据更新位置和尺寸信息确定与用户希望得到的内容相关联的数据对象。</p>
    <p>[0152]	具体而言，可以从内容组织信息中确定快照提供者桌面的高度Dh、宽度Dw，并读取屏幕快照在用户的桌面显示的高度PH、宽度Pw。接着，可以利用以下公式计算用户选定的区域在快照提供者的屏幕中的更新位置:</p>
    <p>[0153]	O，Y = (Dh*0y)/Ph (I)</p>
    <p>[0154]	O’ x = (Dw*0x)/Pw (2)</p>
    <p>[0155]	可以利用以下公式计算用户选定的区域在快照提供者的屏幕中的更新尺寸:</p>
    <p>[0156]	O’η = (Dh*0h)/Ph (3)</p>
    <p>[0157]	O’w = (Dw*0w)/Pw (4)</p>
    <p>[0158]	然后，可以根据上述的更新位置O’ x、0’ Y和更新尺寸O’ w、0’ H来在内容组织信息中找出用户选定区域所涉及到的应用窗口。</p>
    <p>[0159]	根据本发明的实施例，用户选定区域可能属于一个应用，也可能涉及一个或多个应用。当用户选定区域属于一个应用时，可以根据内容组织信息来确定该选定区域所涉及的数据对象，以及该数据对象的数据对象标识和版本标识，继而可以确定这些数据对象的数据。当用户选定区域涉及若干应用时，可以根据内容组织信息来确定该选定区域分别涉及到这若干应用中的每个应用中的哪个或哪些数据对象，然后确定这些涉及到的数据对象的数据对象标识和版本标识，继而可以确定这些数据对象的数据。</p>
    <p>[0160]	此外，应该注意的是，根据本发明的备选实施例，选择信息可以包括:与用户希望得到的内容相关联的数据对象的数据对象标识和版本标识。在这样的实施例中，如在步骤S904中所述的将原始位置和尺寸信息换算为与快照提供者的屏幕相对应的更新位置和尺寸信息并根据更新位置和尺寸信息确定数据对象的过程可以在用户侧的设备处完成，此时从用户向快照服务器发送的选择信息中包含的是在用户侧的设备处确定出的与用户希望得到的内容相关联的数据对象的数据对象标识和版本标识，并且可以不包括用户希望得到的内容的位置信息。</p>
    <p>[0161]	在步骤S905，向用户传送所确定的数据。</p>
    <p>[0162]	通过将根据上述过程所确定的数据发送给用户，可以使得用户得到自己希望得到的屏幕快照中的内容，从而有效改善了用户体验，解决了用户的资料获取需求。</p>

    <p>[0170]	根据本发明的实施例，在接收到的内容组织信息的至少一部分包括内容组织信息中的一个或多个数据对象的结构，而不包括该一个或多个数据对象的数据的情况下，用户侧的设备可以根据用户对于希望得到的所述屏幕快照中的内容的选择以及一个或多个数据对象的结构，确定与用户希望得到的屏幕快照中的内容有关的选择信息；向快照服务器发送所述选择信息；以及从快照服务器接收与用户希望得到的屏幕快照中的内容相关联的数据。</p>
