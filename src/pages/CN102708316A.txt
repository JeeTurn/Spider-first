    <div class="abstract">一种用于多租户架构中数据隔离的方法，其中为租户远程创建数据库和数据库表，将远程数据库表发布为Web服务描述语言(WSDL)服务，通过在云计算服务器中注册该服务，以便租户调用。当多个租户同时访问服务器时，服务器首先根据域名的唯一性以及相关权限配置，在数据库表中加以字段区别，每个租户的基础数据可以由此达到部分隔离。然后服务器会为每个租户创建独立的数据库，这样租户的数据将会完全隔离，租户之间的操作互不影响，实现良好的安全性。</div>

    <p>[0001]	本发明涉及云计算技术领域，尤其涉及一种在云计算中用于多租户架构中数据隔离的方法。</p>
    <p>背景技术</p>
    <p>[0002]	传统的企业级应用交付和运营模式是根据客户自己的需求，定制开发�套完全属于自己的系统，客户需要购买并维护各种所需的硬件和软件资源。近年来，随着互联网技术的发展和云计算引发的全球信息化革命浪潮，多租户软件架构作为一种新的软件架构模式开始兴起的。它通过Internet提供软件服务，服务商将应用软件统�部署在服务器上，租户可以根据自己的需求，向服务商定购所需的软件服务，并按定购服务的功能和使用时间，向厂商支付费用，并通过互联网获得支持。它的出现彻底改变了系统交付和用户使用的方 式，更加灵活可扩展，易于快速定制和集成以便适应用户需求的变化。由于多租户技术可以让多个租户共用一个应用程序或运算环境，且租户大多不会使用太多运算资源的情况下，对供应商来说多租户技术可以有效的降低环境建设的成本，包含硬件本身的成本，操作系统与相关软件的授权成本等。</p>
    <p>[0003]	在多租户系统中多个租户共用�个实例，租户的数据既有隔离又有共享，保护租户数据的隐私与安全也成为多租户技术的关键之一。多租户技术可以通过许多不同的方式来切割用户的应用程序环境或数据，包括数据层、程序层、系统层。</p>
    <p>[0004]	在数据层方面，供应商可以利用切割数据库，切割存储区，切割结构描述或是表格来隔离租户的数据，必要时会需要进行对称或非&#23550;称加密以保护敏感数据，但不同的隔离作法有不同的实现复杂度与风险。</p>
    <p>[0005]	在程序层方面，供应商可以利用应用程序挂载环境，于进程上切割不同租户的应用程序运行环境，在无法跨越进程通信的情况下，保护各租户的应用程序运行环境，但供应商的运算环境要够强。</p>
    <p>[0006]	在系统层方面，供应商可以利用虚拟化技术，将实体运算单元切割成不同的虚拟机，各租户可以使用其中一至数台的虚拟机来作为应用程序与数据的保存环境，但对供应商的运算能力要更高要求。</p>
    <p>发明内容</p>
    <p>[0007]	鉴于现有技术中存在的问题，本发明的目的在于提供�种用于多租户架构中数据隔离的方法，以达到安全地隔离每个租户。</p>
    <p>[0008]	本发明所述的用于多租户架构中数据隔离的方法，其特征在于包括如下步骤：步骤I、启动云计算服务器网络，租户向服务器发送注册请求；步骤2、服务器收到注册请求后通过域名名称验证唯一性，如果验证通过则可以登录服务器；步骤3、在服务器端设置租户的角色，授权给不同用户对平台的不同操作权限；步骤4、租户远程创建数据库和数据库表，配置数据库服务和数据源，建立数据库连接；步骤5、配置SQL语句和字段对应关系，用于对远程数据库的数据进行增删改查操作；步骤6、将远程数据库表发布为Web服务描述语言（WSDL)服务，通过在云计算服务器中注册该服务，以便租户调用。</p>
    <p>[0009]	进�步，本发 明所述的用于多租户架构中数据隔离的方法，其特征在于所述注册请求内容包括域名、用户名、密码、邮件地址。</p>
    <p>[0010]	进�步，本发明所述的用于多租户架构中数据隔离的方法，其特征在于包括授权给不同用户对平台的不同操作权限包括查询、更改、访问、监控权限。</p>
    <p>[0011]	进�步，本发明所述的用于多租户架构中数据隔离的方法，其特征在于当多个租户同时访问服务器时，服务器首先根据域名的唯一性以及相关权限配置，在数据库表中加以字段区&#21029;，以隔离每个租户的基础数据，然后服务器会为每个租户创建独立的数据库，以完全隔离租户的数据。</p>
    <p>[0012]	由以上技术方案可知，本发明提供了用于多租户架构中数据隔离的方法，该发明具有以下优势：隔离租户数据采用为不同的租户建立独立的数据库，这样有助于简化数据模型的扩展设计，满足不同租户的独特需求，而且如果出现故障，恢复数据比较简&#21336;。隔离系统数据通过域名的唯一性以及相关配置，在数据库表中加以字段区别，这样使用结构描述的方式操作简单，易于扩展。</p>

    <p>[0013]	图I为本发明用于实现多租户架构中数据隔离方法之系统的架构图。</p>
    <p>[0014]	图2为本发明用于多租户架构中数据隔离的方法之流程图。</p>
    <p>具体实施方式</p>
    <p>[0015]	为使本发明的上述目的、特征和优点更加明显易懂，下面结合附图和具体实施方式对本发明作进�步详细的说明。</p>
    <p>[0016]	如图I所示为本发明用于实现多租户架构中数据隔离方法之系统的架构图，它是建立在Carbon (企业中间件平台）之上，数据服务处理器采用DSDL (数据服务描述语言）的XML格式，Apache AXIOM (Axis对象模型)提供XML快速响应，Apache Axis2 (开源Web服务运行引擎）负责读取数据描述和创建数据服务，把数据集成到业务流程、小工具、商务应用中去。数据库方面支持Oracle、DB2、MYSQL、SQLServer、HSQLDB、Sysbase等主流数据库，以及JNDI (Java命名和目录接ロ )数据源。</p>
    <p>[0017]	图I所示多租户架构用于租户把数据发布为Web服务，它可以快速发布数据，而且几乎不用编程。它不但能发布关系数据库中的数据，而且还能发布非关系数据库比如Excel、Txt等扁平文件中的数据。具体配置发布流程详见图2。</p>
    <p>[0018]	如图2所示为本发明用于多租户架构中数据隔离的方法之流程图，包括：</p>
    <p>[0019]	步骤I、启动云计算服务器网络，租户向服务器发送注册请求，注册内容包括域名、用户名、密码、邮件地址等；</p>
    <p>[0020]	步骤2，服务器收到注册请求后通过域名名称验证唯一性，如果验证通过则可以登录服务器，如果不唯一，则注册失败，重新申请；</p>
    <p>[0021]	步骤3、在服务器端设置租户的角色，分为管理员和普通用户两种角色，授权给不同用户对平台的操作，包括查询、更改、访问、监控等不同的权限。[0022]	步骤4、租户远程创建数据库和数据库表，并接着配置数据库服务和数据源，在平台选择数据库，配置数据库驱动、访问地址，输入访问数据库的用户名、密码，建立数据库连接。数据库是由租户结合自己的业务需求而创建，以便租户自己调用。我们用mysql数据库为例：</p>
    <p>[0023]	mysql &gt; create database employeedb ；</p>
    <p>[0024]	mysql &gt; use employeedb ；</p>
    <p>[0025]	mysql &gt; create table employee(id VARCHAR(10)N0T NULL PRIMARY KEY，</p>
    <p>[0026]	name VARCHAR(100), address VARCHAR(IOO))；</p>
    <p> [0027]	mysql &gt; insert into employee values ( ‘1，，，john，，，Boston，)；</p>
    <p>[0028]	mysql &gt; insert into employee values ( ‘2，，，Micheal，，，Dallas，)；</p>
    <p>[0029]	mysql &gt; insert into employee values ( ‘3，，，richard，，，Chicago，)；</p>
    <p>[0030]	mysql &gt; exit ；</p>
    <p>[0031]	步骤5、配置SQL操作，具体为配置SQL语句和字段对应关系，用于对远程数据库的数据进行增删改查操作。创建SQL如下：</p>
    <p>[0032]	select id, name，address form employee</p>
    <p>[0033]	步骤6、利用图I中的Axis2架构把远程数据库表发布为WSDL(Web服务描述语言）服务，通过在云计算服务器中注册该服务，以便租户调用。</p>
    <p>[0034]	当多个租户同时访问服务器时，服务器首先根据域名的唯一性以及相关权限配置，在数据表中加以字段区&#21029;，利用关系型数据库（如mysql)的多版本并发控制技术和行加锁机制关联，以达到隔离租户的基础数据。然后服务器会为每个租户创建独立的数据库，这样租户的数据将会完全隔离，租户之间的操作互不影响，实现良好的安全性。</p>
    <p>[0035]	由以上技术方案可知，本发明提供了用于多租户架构中数据隔离的系统和方法，该发明具有以下优势：隔离租户数据采用为不同的租户建立独立的数据库，这样有助于简化数据模型的扩展设计，满足不同租户的独特需求，而且如果出现故障，恢复数据比较简&#21336;。隔离系统数据通过域名的唯一性以及相关配置，在数据表中加以字段区别，这样使用结构描述的方式操作简单，易于扩展。数据库方面支持Oracle、DB2、MYSQL、SQLServer、HSQLDB, Sysbase等主流数据库，以及JNDI数据源。</p>
    <p>[0036]	最后应当说明的是：以上实施例仅用以说明本发明的技术方案而非对其限制；尽管&#21443;照较佳实施例对本发明进行了详细的说明，所属领域的普通技术人员应当理解：依然可以对本发明的具体实施方式进行修改或者对部分技术特征进行等同替换；而不脱离本发明技术方案的精神，其均应涵盖在本发明请求保护的技术方案范围当中。</p>
