    <div class="abstract">本发明实施例公开了一种通用业务架构下业务调用的方法、装置与系统，以实现端到端的USI调用。所述方法包括：通用业务接口USI接收来自应用业务提供商ASP的USI调用请求消息，所述请求消息中携带ASP标识和业务信息；USI向具有网络开放能力的功能实体发送网络功能调用请求消息，所述请求消息中携带ASP标识和业务信息，触发所述功能实体执行相应的网络功能业务。所述装置包括接收模块和发送模块。所述系统包括USI和与之以可通信的方式连接的具有网络开放能力的功能实体。</div>

    <p>一种通用业务架构下业务调用的方法、装置与系统</p>
    <p>技术领域</p>
    <p>本发明属于通信技术领域，尤其涉及一种通用业务架构下业务调用的方 法、装置与系统。</p>
    <p>背景技术</p>
    <p>WiMAX(Worldwide Interoperability for Microwave Access), 中文全称是 全球接入微波互操作性，是一种基于IEEE 802.16标准的无线城域网技术。 WiMAX网络主要由三个部分组成即客户端（MSS/SS)、接入业务网（ASN) 和连接业务网（CSN) 。 WiMAX网络无线侧是基于IEEE 802.16d/e标准的无 线城域网接入技术。现在主要遵循的是IEEE 802.16-2004 (S02.16d)标准。 该标准用于城域网中的非视距点对多点技术，主要是固定和游牧网络的形式， IEEE 802.16e中增加了支持移动通信的技术。</p>
    <p>WiMAX论坛在IT厂家的推动下，启动了 WiMAX网络通用业务接口 (USI)的研究，试图打破WiMAX网络仅作为无线宽带接入网络的地位，而 可以直接面对应用提供商/互联网应用提供商，从而作为一种端到端的无线宽 带网络解决方案，提供接近与无线宽带互联网的服务，并拥有优于互联网的 一些特性：统一账户管理、QoS、位置服务、终端管理、安全、广播和多播 等；同时又实现了目前无线通信网络的优点：漫游、无缝切换等。应用服务 器通过调用USI接口 ，将提供针对个人用户的个性化服务。</p>
    <p>图1所示为USI网络参考模型，USI系统位于CSN中，也可能是CSN之夕卜，多个CSN共用 一个USI系统。</p>
    <p>在目前通用业务架构的讨论中，对于USI的动态QoS调用以及用户信息查 询等业务调用^f又给出了系统的架构。该架构如图2所示，USI系统分别通过U1 接口与ASP/iASP网络相连，通过U2接口与功能实体相连。所述功能实体是 WiMAX网络通过USI向ASP提供网络开放能力的网元，可位于CSN或ASN中。</p>
    <p>发明人发现，在现有技术中，业务调用的架构没有考虑用户在USI系统 中的位置，对于业务调用的过程如何实现，接口参数如何定义，各网元执行 的功能等也都没有具体方案，导致端到端的功能调用无法实现。</p>

    <p>具体实施方式</p>
    <p>为使本发明的目的、技术方案和优点更加清楚，下面结合附图对本发明 具体实施例作进一步的详细描述。</p>
    <p>图3为本发明实施例公开的通用业务架构下业务调用的方法流程图。</p>
    <p>本发明实施例公开的USI调用的方法为MS调用自身的USI能力的场景下 的调用过程。具体步骤如下：</p>
    <p>步骤301、终端MS向ASP发送应用业务请求消息，所述请求消息中携带 有MS的ASP用户标识、业务信息，所述业务信息包括业务参数和业务类型； 如果是涉及到其它用户的业务，例如MS要通过调用USI来定位另 一个用户， 则所述业务参数中还需携带所涉及的业务相关用户的用户标识，通常为ASP 用户标识或用户当前的IP地址或用户的USI用户标识或用户的NAI(网络接入 标识，network access identity)。</p>
    <p>步骤302、 ASP根据与用户的签约关系授权此次业务请求，授权成功后向 USI发送USI调用请求消息，所述请求消息中携带ASP标识、MS的USI用户标 识、业务信息，所述业务信息包括业务类型和业务参数，所述业务参数中携 带业务相关用户的用户标识。在需要计费的场景下，所述请求消息中还携带 ASP计费标识。</p>
    <p>所述MS的USI用户标识是ASP将MS的ASP用户标识映射得到的；所述 ASP计费标识是ASP为此次调用生成的；所述业务相关用户的用户标识是步 骤301中的业务相关用户的USI用户标识或IP地址或NAI，或者由ASP根据业务相关用户的ASP用户标识映射得到的USI用户标识或IP地址或NAI。</p>
    <p>步骤303、 USI根据ASP、 MS、 USI三者之间的签约关系对所述USI调用 请求进行授权，授权成功后向具有网络开放能力的功能实体发送网络功能调 用请求消息，所述请求消息中携带ASP标识、MS的NAI、业务信息，所述业 务信息包括业务类型和业务参数，所述业务参数中携带业务相关用户的NAI 或IP地址。在计费的场景下，还携带USI计费标识和/或ASP计费标识。可选 的还包括U2会话标识。</p>
    <p>所述MS的NAI是USI将MS的USI用户标识映射得到的；所述USI计费标识 是USI为此次调用生成的；所述业务相关用户的NAI或IP地址是步骤302中的 业务相关用户的IP地址或NAI ，或者由USI根据业务相关用户的USI用户标识 映射得到的IP地址或NAI。所述USI计费标识和ASP计费标识的具体发送情形 取决于计费系统自身的策略。</p>
    <p>所述功能实体是WiMAX网络通过USI向ASP提供网络开放能力的网元， 可位于CSN或ASN中。针对不同的业务它可以是不同的实例，例如用于PCC (Policy Control and Charging,策略控制与计费）控制的PCRF( policy charging rule function,策略计费规则功能）、用于MBS( Multicast and Broadcast Service, 多播广播业务）控制的MBS Server ( MBS服务器）、用于LBS ( Location Based Service,定位业务）控制的LBS Server ( LBS服务器）、用于非PCC的Dynamic QoS (动态QoS)控制的PF (PolicyFunction,策略功能）、用于用户或终端 信息（包括终端IP地址、用户或终端状态等信息）操作的AAA Server (认证 授权和计费服务器）、用于计费相关的计费系统等等，此处不再一一列举。</p>
    <p>所述网络功能调用请求可以存在并发的多次调用请求，即同时进行多次</p>
    <p>调用，所述多次调用可以针对不同的功能实体，也可以针对同一个功能实体。为了实现并发的多次调用的功能，需要在各调用请求中携带一个U2接口会话 标识。所述U2会话标识可以是USI直接转发的由ASP生成并包含在步骤303中 的USI调用请求中一同下发给USI的U1会话标识，也可以是USI根据所述ASP 下发的U1会话标识对应生成的，也可以是由USI单独生成的。当USI向功能实 体发送的调用请求为单次调用时，即同 一时间只对一个功能实体进行一次调 用时，也可以不在调用请求中携带所述U2接口会话标识。</p>
    <p>步骤304、具有网络开放能力的功能实体（图示功能实体）根据USI的网 络功能调用请求，执行相应的网络功能业务。</p>
    <p>步骤305、具有网络开放能力的功能实体向USI发送网络功能调用响应消 息，在计费场景下，所述响应消息中携带本次业务的承载网计费标识，如果 步骤303中携带有U2会话标识，则所述功能调用响应消息中也携带所述U2会 话标识。</p>
    <p>所述响应消息用于向USI报告网络功能业务执行的结果，所述承载网计 费标识由承载网络的计费功能实体在执行本次业务时生成。所述U2会话标识 用于帮助USI标识本次响应消息对应的请求消息。</p>
    <p>步骤306、 USI向ASP发送USI调用响应消息，在计费场景下，所述响应 消息中携带承载网计费标识和/或USI计费标识。</p>
    <p>所述承载网计费标识和USI计费标识的具体发送情形取决于计费系统自 身的策略。</p>
    <p>步骤307、可选的，ASP向MS发送应用业务响应消息。本步骤是否执行， 取决于ASP网络自己的策略。</p>
    <p>通过应用本发明实施例所公开的方法，MS可以向ASP发送应用业务请求 消息，通过ASP、 USI的授权和映射，实现MS对自己的USI能力的调用。通过</p>
    <p>12在USI向功能实体发送的网络功能调用请求中携带U2会话标识，所述调用还 可以实现并发的多次调用。</p>

    <p>的场景下，如其他方通过ASP和USI查询该MS的位置或信息、动态QoS、》务 改MS签约信息、缴费/扣费等，ASP调用自己或用户的USI能力的调用过程。 所述其他方是相对于MS而言，可以是ASP网络本身、其他网络功能实体，也 可以是其它用户。具体步骤如下：</p>

    <p>步骤403、 USI根据ASP、 MS、 USI三者之间的签约关系或者ASP、 USI 之间的签约关系对所述USI调用请求进行授权，授权成功后向具有网络开放 能力的功能实体发送网络功能调用请求消息，所述请求消息中携带ASP标识、 业务信息，所述业务信息包括业务类型和业务参数。在计费场景下，所述请 求信息还携带USI计费标识和/或ASP计费标识。可选的还包括U2会话标识。 在ASP调用用户的USI能力的场景下，所述请求消息中还包括所述被操作的 MS的IP地址或NAI;在ASP调用自己的USI能力的场景下，所述被操作的MS 的IP地址或NAI携带在所述业务参数中。</p>
    <p>所述ASP计费标识是ASP为此次调用生成的；所述被操作的MS的NAI或 IP地址是步骤402中的被操作的MS的IP地址或NAI，或者由USI根据被操作的 MS的USI用户标识映射得到的IP地址或NAI。所述U2会话标识用于帮助USI 标识本次调用请求。所述USI计费标识和ASP计费标识的具体发送情形取决于 计费系统自身的策略。</p>
    <p>步骤404、具有网络开放能力的功能实体根据USI的网络功能调用请求， 执行相应的网络功能业务。</p>
    <p>步骤405、功能实体向USI发送网络功能调用响应消息，在计费场景下， 所述响应消息中携带本次业务的承载网计费标识，如果步骤403中携带有U2 会话标识，则所述功能调用响应消息中也携带所述U2会话标识。</p>

    <p>步骤406、 USI向ASP发送USI调用响应消息，在计费场景下，所述响应消息中携带承载网计费标识和/或USI计费标识。所述承载网计费标识和USI</p>
